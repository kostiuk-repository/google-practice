# Algorithm Visualizer — як додавати навчальні анімації

`AlgorithmVisualizer` перетворює fenced JSON-блок `algoviz` у покрокову SVG-анімацію. Для цього не потрібно змінювати React-код: опис живе безпосередньо у Markdown поруч із поясненням алгоритму.

## Мінімальний приклад

````markdown
```algoviz
{
  "type": "array",
  "title": "Один прохід масивом",
  "values": [7, 12, -3, 8, 5],
  "steps": [
    {"label": "Починаємо з індексу 0", "pointer": 0},
    {"label": "Переходимо до індексу 1", "pointer": 1, "visited": [0]}
  ]
}
```
````

## Підтримувані режими

| `type` | Для чого використовувати |
|---|---|
| `array` | звичайний прохід, prefix/suffix, порівняння елементів |
| `two-pointers` | left/right, read/write, slow/fast pointers |
| `sliding-window` | фіксоване або змінне неперервне вікно |
| `binary-search` | `lo`, `mid`, `hi` і звуження області пошуку |

Усі режими використовують одну схему даних. `type` задає навчальний контекст і дозволить надалі додавати спеціалізовані оформлення без зміни Markdown.

## Поля верхнього рівня

- `type` — один із чотирьох режимів вище;
- `title` — коротка назва в заголовку віджета;
- `values` — від 1 до 40 чисел або коротких рядків;
- `steps` — непорожній масив станів анімації.

## Поля кроку

- `label` — обов’язкове пояснення того, що зараз відбувається;
- `note` — необов’язкове пояснення, чому цей рух правильний;
- `pointer` — скорочений запис одного pointer-а з назвою `i`;
- `pointers` — кілька іменованих pointers, наприклад `{"L": 0, "R": 5}`;
- `visited` — індекси, які вже відкинуті або оброблені;
- `active` — додатково підсвічені індекси;
- `compare` — елементи, які порівнюються на цьому кроці;
- `range` — включні межі активного фрагмента `[left, right]`;
- `values` — новий стан значень тієї самої довжини, корисний для sorting-візуалізацій.

Усі індекси мають бути в межах `0..values.length-1`. Невалідний JSON або індекс не ламає сторінку: замість анімації відображається пояснення помилки й початковий блок.

## Рекомендації для навчального тексту

1. Один крок має пояснювати одну зміну стану.
2. У `label` пишіть **що** сталося, у `note` — **чому це безпечно або корисно**.
3. Не робіть більше 8–12 кроків в одному віджеті: довгу трасу краще розділити на кілька ключових фаз.
4. Позначайте вже відкинуту область через `visited`, а живий простір пошуку — через `range`.
5. Залишайте коротке текстове пояснення після віджета: анімація допомагає зрозуміти інваріант, але не замінює його формулювання.

Віджет має Play/Pause, покрокову навігацію, Reset, вибір швидкості, fullscreen, підтримку dark/light theme, `prefers-reduced-motion` і розгортану текстову таблицю для accessibility.
