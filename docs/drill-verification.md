# Покриття перевірок дрілів

У репозиторії є 1 600 дрілів. `npm run verify:runner` обходить кожен із них і класифікує фактичний рівень перевірки — без припущення, що успішна компіляція означає правильний алгоритм.

Поточний аудит:

| Рівень | Кількість | Що доводить |
|---|---:|---|
| Explicit contract | 5 | У тестовому класі є окремий `testDrillNN` |
| Generated example contract | 344 | Runner перетворив конкретні JavaDoc `Input` / `Output` на executable assertion |
| Compile-only | 1 251 | Код компілюється, але correctness цього дрілу ще не доведено |

Generated contract створюється лише тоді, коли parser однозначно розуміє сигнатуру, аргументи й очікуваний результат. Описи на кшталт «graph», «list modified in-place» або сценарії з потоками навмисно не перетворюються на фіктивні тести. Compile-only запуск не позначає дріл чи основну задачу завершеними.

Щоб підвищити строгість конкретного дрілу, додайте до відповідного класу тестів метод `testDrill01`, `testDrill02` тощо. Наступний аудит автоматично перенесе його до Explicit contract.

> Цифри вище зафіксовані аудитом під час додавання системи. Актуальний результат завжди друкує `npm run verify:runner`.
