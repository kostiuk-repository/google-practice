# Hard_01_EditDistance — Довідка

## Техніка: Двовимірне DP редагування рядків (Levenshtein Distance DP)

**Суть:** Пошук мінімальної кількості операцій (вставка, видалення, заміна) для перетворення одного рядка в інший. Таблиця dp[i][j] зберігає відстань редагування для префіксів рядків довжиною i та j.

### Концепт
```java
// dp[i][j] = min(dp[i-1][j]+1, dp[i][j-1]+1, dp[i-1][j-1]+(word1[i-1]==word2[j-1]?0:1))
int m = word1.length(), n = word2.length();
int[][] dp = new int[m + 1][n + 1];
for (int i = 0; i <= m; i++) dp[i][0] = i;
for (int j = 0; j <= n; j++) dp[0][j] = j;
for (int i = 1; i <= m; i++) {
    for (int j = 1; j <= n; j++) {
        if (word1.charAt(i-1) == word2.charAt(j-1)) {
            dp[i][j] = dp[i-1][j-1];
        } else {
            dp[i][j] = 1 + Math.min(dp[i-1][j-1], Math.min(dp[i-1][j], dp[i][j-1]));
        }
    }
}
```

**Коли застосовувати:** При порівнянні ступеня схожості двох послідовностей (текстів, ДНК, шляхів тощо) з можливістю додавання/видалення елементів.

**Складність:** Часова: O(M * N) — заповнення таблиці. Просторова: O(M * N) для матриці або O(min(M, N)) з використанням двох рядків.

## Вправи (Drills)
- **01 (minDistanceRecursive)**: Знайдіть відстань редагування за допомогою рекурсії з мемоізацією.
- **02 (minDistanceTab)**: Реалізуйте відстань редагування за допомогою класичної двовимірної таблиці табуляції.
- **03 (minDistanceSpace)**: Оптимізуйте пам'ять алгоритму Edit Distance до використання лише двох рядків.
- **04 (minDistanceCost)**: Знайдіть мінімальну вартість редагування, якщо кожна операція має свою ціну (вставка = insertCost, видалення = deleteCost, заміна = replaceCost).
- **05 (getEditScript)**: Поверніть рядок операцій (наприклад: 'Insert a, Delete b, Replace c with d'), що описує оптимальний шлях трансформації.
