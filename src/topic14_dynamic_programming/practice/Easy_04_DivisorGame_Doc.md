# Easy_04_DivisorGame — Довідка

## Техніка: Теорія ігор з мемоізацією (Game DP / Winning-Losing States)

**Суть:** Визначення виграшних та програшних станів у грі з повною інформацією. Стан виграшний (true), якщо існує перехід хоча б в один програшний стан супротивника; і програшний (false), якщо всі ходи ведуть у виграшні стани супротивника.

### Концепт
```java
// Визначення виграшного стану через переходи
boolean[] dp = new boolean[n + 1];
dp[1] = false; // програш
for (int i = 2; i <= n; i++) {
    for (int x = 1; x < i; x++) {
        if (i % x == 0 && !dp[i - x]) {
            dp[i] = true;
            break;
        }
    }
}
```

**Коли застосовувати:** Для розв'язання ігор для двох гравців, які роблять ходи по черзі, де результат гри залежить від поточного числа чи стану.

**Складність:** Часова: O(N * sqrt(N)) або O(N^2) при простому переборі дільників. Просторова: O(N) для збереження станів.

## Вправи (Drills)
- **01 (divisorGameBase)**: Реалізуйте гру з дільниками за допомогою динамічного програмування (або мемоізації) без використання математичного скорочення n % 2 == 0.
- **02 (countWinningMoves)**: Підрахуйте кількість можливих початкових ходів x (де 0 < x < n і n % x == 0), які гарантують перемогу Алісі при оптимальній подальшій грі обох сторін.
- **03 (divisorGameThreePlayers)**: Припустимо, що у гру грають троє гравців (Аліса, Боб, Чарлі) по колу. Аліса виграє, якщо гра завершується на її ходу (немає дільників). Визначте, чи виграє Аліса.
- **04 (divisorGameWithSubtract)**: Гра, де замість дільників гравець може відняти будь-яку кількість очок, що є у масиві moves. Визначте, чи виграє перший гравець.
- **05 (maxGameMoves)**: Знайдіть максимальну кількість ходів, яку може тривати гра з дільниками, якщо обидва гравці грають з метою максимізувати тривалість гри.
