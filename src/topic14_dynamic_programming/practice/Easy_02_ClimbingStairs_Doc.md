# Easy_02_ClimbingStairs — Довідка

## Техніка: Лінійне динамічне програмування (переходи Фібоначчі)

**Суть:** Зведення задачі підрахунку кількості шляхів до лінійного рекурентного співвідношення, де стан dp[i] залежить від суми кількох попередніх станів.

### Концепт
```java
// Кількість способів піднятися на i-ту сходинку
dp[i] = dp[i-1] + dp[i-2];
```

**Коли застосовувати:** Коли кількість способів досягти стану є сумою кількості способів досягти попередніх станів, з яких дозволений перехід.

**Складність:** Часова: O(N) — один прохід по станах. Просторова: O(N) для масиву або O(1) при оптимізації пам'яті.

## Вправи (Drills)
- **01 (climbStairsBase)**: Обчисліть кількість способів піднятися на n-ту сходинку, якщо за раз можна зробити 1 або 2 кроки (чиста рекурсія).
- **02 (climbStairs3Steps)**: Кількість способів піднятися на n-ту сходинку, якщо за раз можна зробити 1, 2 або 3 кроки.
- **03 (climbStairsMaxKSteps)**: Кількість способів піднятися на n-ту сходинку, якщо за раз можна зробити від 1 до k кроків.
- **04 (climbStairsForbidden)**: Кількість способів піднятися на n-ту сходинку, якщо деякі сходинки позначені в масиві forbidden як true (на них не можна ставати).
- **05 (climbStairsCost)**: Кожна сходинка має певну вартість cost[i]. Знайдіть мінімальну вартість підйому на вершину n, якщо можна робити 1 або 2 кроки.
