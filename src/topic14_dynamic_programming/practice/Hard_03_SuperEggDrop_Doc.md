# Hard_03_SuperEggDrop — Довідка

## Техніка: Оптимізація DP з бінарним пошуком та оберненим мисленням

**Суть:** Пошук мінімальної кількості спроб для знаходження критичного поверху. Пряме DP рівняння використовує оптимізацію переходу за допомогою бінарного пошуку. Альтернативний (обернений) підхід рахує максимальну висоту будівлі, яку можна протестувати за m кроків з k яйцями.

### Концепт
```java
// Обернене DP: dp[m][k] = dp[m-1][k-1] + dp[m-1][k] + 1
int[][] dp = new int[n + 1][k + 1];
int m = 0;
while (dp[m][k] < n) {
    m++;
    for (int j = 1; j <= k; j++) {
        dp[m][j] = dp[m - 1][j - 1] + dp[m - 1][j] + 1;
    }
}
return m;
```

**Коли застосовувати:** При знаходженні мінімальної кількості спроб у найгіршому випадку (Minimax) з обмеженими ресурсами (яйцями), коли класичний бінарний пошук неможливий через ризик розбити всі ресурси.

**Складність:** Часова: O(K * N log N) при прямому DP з бінарним пошуком, або O(K * log N) при оберненому DP. Просторова: O(K * log N) додаткової пам'яті.

## Вправи (Drills)
- **01 (eggDropRecursive)**: Реалізуйте базовий рекурсивний підхід з мемоізацією за формулою: dp(k, n) = 1 + min_{1<=x<=n}(max(dp(k-1, x-1), dp(k, n-x))).
- **02 (eggDropBS)**: Оптимізуйте пошук точки переходу x у прямому DP за допомогою бінарного пошуку за O(K * N log N).
- **03 (eggDropInverse)**: Реалізуйте обернений підхід (DP по кроках m): знайти мінімальне m, при якому dp[m][k] >= n.
- **04 (eggDropInfiniteEggs)**: Визначте кількість тестів, необхідних, якщо у вас є нескінченна кількість яєць (k = infinity).
- **05 (eggDropWithBreakProb)**: Знайдіть математичне сподівання кількості спроб, якщо яйце розбивається на поверху x з фіксованою ймовірністю p.
