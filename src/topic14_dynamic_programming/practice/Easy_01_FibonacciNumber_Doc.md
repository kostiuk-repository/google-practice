# Easy_01_FibonacciNumber — Довідка

## Техніка: Мемоізація та оптимізація пам'яті (Bottom-up DP)

**Суть:** Збереження результатів попередніх обчислень для уникнення повторних розрахунків у рекурсивному дереві, або ітеративне заповнення таблиці знизу вгору з можливим зменшенням пам'яті до O(1) за рахунок збереження лише необхідних останніх станів.

### Концепт
```java
// Базовий шаблон оптимізованого по пам'яті Фібоначчі
int a = 0, b = 1;
for (int i = 2; i <= n; i++) {
    int c = a + b;
    a = b;
    b = c;
}
return n == 0 ? a : b;
```

**Коли застосовувати:** Коли задача розбивається на підзадачі, які перекриваються (overlapping subproblems), і структура оптимальності підзадач дозволяє будувати рішення для більших задач на основі менших (optimal substructure).

**Складність:** Часова: O(N) — кожен стан обчислюється один раз. Просторова: O(N) для табуляції/мемоізації, або O(1) при оптимізації пам'яті двома змінними.

## Вправи (Drills)
- **01 (fibBase)**: Обчисліть n-те число Фібоначчі за допомогою чистої рекурсії без мемоізації. Це базовий навчальний випадок для розуміння дерева рекурсії.
- **02 (fibMemo)**: Реалізуйте обчислення n-го числа Фібоначчі з використанням переданого масиву мемоізації для уникнення повторних обчислень.
- **03 (fibTab)**: Визначте n-те число Фібоначчі за допомогою табуляції (bottom-up DP масиву).
- **04 (fibSpace)**: Знайдіть n-те число Фібоначчі з оптимізацією просторової складності до O(1) додаткової пам'яті.
- **05 (fibEvenSum)**: Обчисліть суму всіх парних чисел Фібоначчі до n-го включно (тобто суму тих F_i, які є парними значеннями і i <= n).
