# Medium_05_CoinChange — Довідка

## Техніка: Необмежений рюкзак (Unbounded Knapsack DP)

**Суть:** Пошук оптимальної комбінації елементів з необмеженою кількістю повторень кожного типу. dp[j] зберігає мінімальну кількість монет для формування суми j.

### Концепт
```java
// dp[j] = min(dp[j], dp[j - coin] + 1)
int[] dp = new int[amount + 1];
Arrays.fill(dp, amount + 1);
dp[0] = 0;
for (int coin : coins) {
    for (int j = coin; j <= amount; j++) {
        dp[j] = Math.min(dp[j], dp[j - coin] + 1);
    }
}
return dp[amount] > amount ? -1 : dp[amount];
```

**Коли застосовувати:** При виборі комбінацій з фіксованих номіналів/ваг для набору цільової суми, коли кожен номінал доступний у необмеженій кількості.

**Складність:** Часова: O(N * A), де N — кількість типів монет, A — сума (amount). Просторова: O(A) додаткової пам'яті для одновимірного масиву.

## Вправи (Drills)
- **01 (coinChangeBase)**: Знайдіть найменшу кількість монет, необхідних для формування суми amount. Якщо це неможливо, поверніть -1.
- **02 (coinChangeWays)**: Знайдіть кількість унікальних комбінацій монет для утворення суми amount (Coin Change II). Order doesn't matter.
- **03 (coinChangeWaysPermutations)**: Знайдіть кількість способів скласти суму amount, якщо різні порядки вважаються окремими способами (наприклад, [1, 2] та [2, 1] — різні).
- **04 (coinChangeSpace)**: Реалізуйте пошук мінімальної кількості монет з обмеженням пам'яті O(amount) (1D DP).
- **05 (getCoinsUsed)**: Поверніть масив номіналів монет, використаних у мінімальній комбінації для досягнення суми amount. Якщо суму набрати неможливо, поверніть порожній масив.
