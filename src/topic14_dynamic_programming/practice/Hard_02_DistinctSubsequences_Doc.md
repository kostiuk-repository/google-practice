# Hard_02_DistinctSubsequences — Довідка

## Техніка: Двовимірне DP на зіставленні символів (Subsequence Count DP)

**Суть:** Підрахунок кількості способів утворити рядок t як підпослідовність s. Стан dp[i][j] вказує кількість відповідностей префікса t[0..j-1] у префіксі s[0..i-1].

### Концепт
```java
// dp[i][j] = dp[i-1][j] + (s[i-1] == t[j-1] ? dp[i-1][j-1] : 0)
int[][] dp = new int[m + 1][n + 1];
for (int i = 0; i <= m; i++) dp[i][0] = 1;
for (int i = 1; i <= m; i++) {
    for (int j = 1; j <= n; j++) {
        if (s.charAt(i - 1) == t.charAt(j - 1)) {
            dp[i][j] = dp[i - 1][j] + dp[i - 1][j - 1];
        } else {
            dp[i][j] = dp[i - 1][j];
        }
    }
}
```

**Коли застосовувати:** Коли потрібно підрахувати точну кількість комбінацій символів одного рядка, що відповідають іншому рядку за збереження черговості.

**Складність:** Часова: O(M * N) — повний прохід по матриці. Просторова: O(M * N) або O(N) при одновимірній оптимізації.
