# Easy_09_IsSubsequence — Довідка

## Техніка: DP узгодження та два вказівники (Matching DP / Two Pointers)

**Суть:** Пошук символів підпослідовності у цільовому рядку. Жадібний підхід двома вказівниками перевіряє символи лінійно. DP підхід зводить задачу до пошуку найдовшої спільної підпослідовності (LCS) або використовує масиви наступних символів.

### Концепт
```java
// Перевірка двома вказівниками
int i = 0, j = 0;
while (i < s.length() && j < t.length()) {
    if (s.charAt(i) == t.charAt(j)) {
        i++;
    }
    j++;
}
return i == s.length();
```

**Коли застосовувати:** Коли потрібно перевірити, чи є один рядок підпослідовністю іншого (порядок символів має зберігатися, але не обов'язково неперервно).

**Складність:** Часова: O(N) для перевірки одного рядка довжиною N. Просторова: O(1) додаткової пам'яті для двох вказівників.

## Вправи (Drills)
- **01 (isSubsequenceRecursive)**: Перевірте, чи є s підпослідовністю t, використовуючи чисту рекурсію.
- **02 (isSubsequenceIterative)**: Реалізуйте ітеративну перевірку за допомогою двох вказівників.
- **03 (isSubsequenceDP)**: Реалізуйте перевірку через знаходження довжини LCS (якщо LCS(s, t) == s.length()).
- **04 (countSubsequences)**: Підрахуйте, скільки разів рядок s зустрічається як підпослідовність у рядку t.
- **05 (isSubsequenceBatch)**: Оптимізуйте перевірку багатьох рядків-джерел (sources) на те, чи є вони підпослідовностями великого рядка t, використовуючи попередню індексацію символів t.
