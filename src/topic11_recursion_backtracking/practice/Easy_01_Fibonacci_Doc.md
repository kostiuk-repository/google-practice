# Easy_01_Fibonacci — Довідка

## Техніка: Linear Recurrence with Memoization

**Суть:** значення на кроці `n` виражається через значення на
попередніх кроках (`f(n) = f(n-1) + f(n-2)`). Наївна рекурсія
рахує одні й ті самі підзадачі експоненційну кількість разів, тому
результати кешують (мемоізація через `HashMap`/масив), і кожна
підзадача обчислюється рівно один раз.

```java
Map<Integer, Integer> memo = new HashMap<>();
int f(int n) {
    if (n <= 1) return n;
    if (memo.containsKey(n)) return memo.get(n);
    int result = f(n - 1) + f(n - 2);
    memo.put(n, result);
    return result;
}
```

**Коли застосовувати:** будь-яка задача, де відповідь для `n` виражається
через відповіді для меншого(их) `n` за фіксованою формулою — підрахунок
шляхів, послідовності з рекурентним співвідношенням, DP-задачі, що
"розгортаються" в дерево рекурсії з дублюванням підзадач.

**Складність:** без мемоізації — O(2^n) (дерево рекурсії дублює
піддерева); з мемоізацією — O(n) час і O(n) пам'ять (кеш + глибина
стека рекурсії).
