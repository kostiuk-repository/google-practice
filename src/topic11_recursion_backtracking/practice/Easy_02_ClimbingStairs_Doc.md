# Easy_02_ClimbingStairs — Довідка

## Техніка: Linear Recurrence / DP with Memoization

**Суть:** Значення на кроці `n` залежить від попередніх кроків. Щоб уникнути експоненційного дублювання викликів, результати проміжних підзадач записуються в масив (мемоізація), зводячи складність до O(n).

```java
int climb(int n, int[] memo) {
    if (n <= 1) return 1;
    if (memo[n] > 0) return memo[n];
    memo[n] = climb(n - 1, memo) + climb(n - 2, memo);
    return memo[n];
}
```

**Коли застосовувати:** Для підрахунку кількості способів досягти певної позиції (або сумарної вартості), коли наступний стан є лінійною комбінацією попередніх.

**Складність:** O(n) час, O(n) додаткова пам'ять для мемоізації та стека викликів.

## Drills

1. **Easy_02_01** — Climbing Stairs with 3-Step Maximum: Ви можете піднятися на 1, 2 або 3 сходинки за раз. Поверніть кількість способів досягти вершини для n сходинок.
2. **Easy_02_02** — Climbing Stairs with Variable Step Sizes: Ви можете піднятися на будь-ку кількість сходинок, вказану в масиві allowedSteps. Поверніть кількість способів дійти до n.
3. **Easy_02_03** — Minimum Cost Climbing Stairs: Кожна сходинка має певну вартість. Оплативши її, ви можете зробити 1 або 2 кроки. Знайдіть мінімальну вартість досягнення вершини.
4. **Easy_02_04** — Climbing Stairs with Broken Steps: Деякі сходинки зламані (broken[i] == true), на них не можна ставати. Поверніть кількість способів досягти сходинки n.
5. **Easy_02_05** — Climbing Stairs with Limited Twos: Ви можете робити кроки по 1 або 2 сходинки, але загалом кроків по 2 сходинки може бути не більше ніж k. Знайдіть кількість способів.

Розв'язуй у порядку 01 → 05, не підглядаючи в Easy_02_ClimbingStairs.java.
