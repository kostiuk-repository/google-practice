# Medium_03_WordSearch — Довідка

## Техніка: 2D Grid DFS / Backtracking

**Суть:** Пошук шляху в двовимірній матриці символів. Для запобігання повторному використанню клітинок у поточному слові, клітинка тимчасово позначається як відвідана (наприклад, міняється на `#`), після чого рекурсивно перевіряються 4 сусіди, а при поверненні символ відновлюється.

```java
boolean dfs(char[][] board, int r, int c, int index, String word) {
    if (index == word.length()) return true;
    if (r < 0 || r >= board.length || c < 0 || c >= board[0].length || board[r][c] != word.charAt(index)) return false;
    char temp = board[r][c];
    board[r][c] = '#';
    boolean found = dfs(board, r + 1, c, index + 1, word) ||
                    dfs(board, r - 1, c, index + 1, word) ||
                    dfs(board, r, c + 1, index + 1, word) ||
                    dfs(board, r, c - 1, index + 1, word);
    board[r][c] = temp;
    return found;
}
```

**Коли застосовувати:** Пошук шляхів, зв'язаних областей чи слів у сітках/графах із можливістю відкоту.

**Складність:** O(R * C * 4^L) час, O(L) пам'ять.

## Drills

1. **Medium_03_01** — Word Search Count: Знайдіть, скільки разів задане слово можна побудувати з сусідніх клітинок у сітці без повторного використання клітинок у межах одного шляху.
2. **Medium_03_02** — Longest Alphabetical Path: Знайдіть довжину найдовшого шляху в сітці, де символи йдуть у строго зростаючому алфавітному порядку.
3. **Medium_03_03** — Word Search with Max Turns: Перевірте, чи існує слово в сітці, якщо шлях може змінювати напрямок не більше ніж maxTurns разів.
4. **Medium_03_04** — Word Search from Center: Перевірте, чи існує слово в сітці, за умови, що пошук може починатися лише з центральних клітинок (середній рядок чи стовпець).
5. **Medium_03_05** — Word Search with Wildcard: Перевірте, чи існує слово в сітці, якщо воно може містити символ '?', що підходить до будь-якої літери.

Розв'язуй у порядку 01 → 05, не підглядаючи в Medium_03_WordSearch.java.
