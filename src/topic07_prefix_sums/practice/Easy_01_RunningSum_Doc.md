# Easy_01_RunningSum — Довідка

## Техніка: Prefix Sum Array (побудова префіксного масиву)

**Суть:** проходимо масив зліва направо один раз і накопичуємо суму в
змінній-акумуляторі, записуючи проміжний результат на кожному кроці.
На відміну від "running accumulator" з topic01 (де нас цікавило лише
best-so-far значення), тут нас цікавить сам МАСИВ проміжних сум — він
стає структурою даних, яку можна перевикористовувати для подальших
запитів (наприклад, сум діапазонів за O(1)).

```java
int[] prefix = new int[nums.length];
int running = 0;
for (int i = 0; i < nums.length; i++) {
    running += nums[i];
    prefix[i] = running;
}
```

**Коли застосовувати:** будь-яка задача, де треба один раз побудувати
масив накопичених сум (або добутків, або іншої асоціативної операції),
а потім багато разів ним користуватись — для запитів діапазону, для
відновлення оригінального масиву через різницю сусідніх елементів,
або як проміжний крок для складніших технік (hashmap + prefix sum).

**Складність:** O(N) час і O(N) пам'ять на побудову префіксного масиву;
кожен подальший запит діапазону — O(1).

## Drills

1. **Easy_01_01** — суфіксна сума (той самий накопичувач, але справа наліво).
2. **Easy_01_02** — префіксна сума з ненульовим стартовим балансом.
3. **Easy_01_03** — префіксний добуток замість префіксної суми (інша операція).
4. **Easy_01_04** — обернена задача: відновити оригінальний масив з його префіксних сум.
5. **Easy_01_05** — побудувати префіксний масив один раз і відповісти на кілька запитів суми діапазону за O(1).

Розв'язуй у порядку 01 → 05, не підглядаючи в Easy_01_RunningSum.java.
