# Easy_01_InorderTraversal — Довідка

## Техніка: Recursive DFS Traversal

**Суть:** обхід дерева рекурсією, де порядок відвідування вузла
(до/між/після рекурсивних викликів у ліве й праве піддерево) визначає
тип обходу — preorder, inorder чи postorder. Базовий рекурсивний
"скелет" той самий, змінюється лише МІСЦЕ, де ми додаємо значення
поточного вузла до результату.

```java
void inorder(TreeNode node, List<Integer> result) {
    if (node == null) return;
    inorder(node.left, result);
    result.add(node.val);      // <-- позиція візиту визначає тип обходу
    inorder(node.right, result);
}
```

**Коли застосовувати:** будь-яка задача, що вимагає відвідати кожен
вузол дерева рівно один раз у певному порядку — збір значень, підрахунок,
пошук k-го елемента, фільтрація листків. Ключова ознака: результат
залежить від ПОРЯДКУ відвідування вузлів, а не лише від їх множини.

**Складність:** O(N) час (кожен вузол відвідується один раз), O(H)
додаткової пам'яті на стек рекурсії (H — висота дерева), O(N) для
самого результату.
