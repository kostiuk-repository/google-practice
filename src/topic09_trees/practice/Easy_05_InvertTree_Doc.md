# Easy_05_InvertTree — Довідка

## Техніка: Recursive Node Mutation (Swap Children)

**Суть:** на кожному вузлі міняємо місцями `left` і `right`, а потім
рекурсивно спускаємось у (вже переставлені) піддерева. Порядок —
pre-order чи post-order — не важливий, бо swap і рекурсія в ліве/праве
піддерево незалежні одна від одної.

```java
TreeNode invertTree(TreeNode root) {
    if (root == null) return null;
    TreeNode tmp = root.left;
    root.left = invertTree(root.right);
    root.right = invertTree(tmp);
    return root;
}
```

**Коли застосовувати:** будь-яка задача "модифікувати кожен вузол
дерева за певним правилом" — дзеркальне відображення, побудова копії,
зміна значень за умовою залежно від глибини. Ключова ознака: зміна
застосовується ЛОКАЛЬНО в кожному вузлі, а рекурсія лише поширює цю
зміну по всьому дереву.

**Складність:** O(N) час (кожен вузол обробляється один раз), O(H)
пам'яті на стек рекурсії.

## Drills

1. **Easy_05_01** — swap лише на рівні root, без рекурсії в піддерева.
2. **Easy_05_02** — пошук вузла за значенням + інверсія лише його піддерева.
3. **Easy_05_03** — побудова НОВОГО інвертованого дерева без мутації оригіналу.
4. **Easy_05_04** — інверсія лише на парних рівнях глибини (умова через параметр).
5. **Easy_05_05** — та сама інверсія, але ІТЕРАТИВНО через чергу (BFS).

Розв'язуй у порядку 01 → 05, не підглядаючи в Easy_05_InvertTree.java.
