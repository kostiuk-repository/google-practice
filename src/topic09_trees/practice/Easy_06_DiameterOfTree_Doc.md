# Easy_06_DiameterOfTree — Довідка

## Техніка: Post-Order Height With Side-Effect Max

**Суть:** обчислюємо висоту дерева post-order рекурсією (спочатку
ліве й праве піддерево, потім комбінуємо), і НА КОЖНОМУ вузлі, попутно,
оновлюємо глобальний максимум `leftHeight + rightHeight` — бо
найдовший шлях може "згинатись" через будь-який вузол, не обов'язково
через корінь.

```java
int diameter = 0;
int height(TreeNode node) {
    if (node == null) return -1;
    int left = height(node.left);
    int right = height(node.right);
    diameter = Math.max(diameter, left + right + 2); // або відповідна формула
    return 1 + Math.max(left, right);
}
```

**Коли застосовувати:** будь-яка задача "знайти найкращу характеристику
шляху/під'єднання, що проходить через якийсь вузол дерева" — діаметр,
максимальна сума шляху, ширина. Ключова ознака: відповідь формується НЕ
у кореневому виклику, а може "зафіксуватись" у будь-якому проміжному
вузлі під час post-order обходу.

**Складність:** O(N) час (кожен вузол відвідується один раз завдяки
об'єднанню висоти й діаметра в один прохід), O(H) пам'яті на стек.
