# Medium_05_InsertDeleteGetRandom — Довідка

## Техніка: Поєднання ArrayList та HashMap для операцій за O(1) (Combination of ArrayList & HashMap for O(1) operations)

**Суть:**
Для побудови структури даних, яка підтримує додавання, видалення та вибір випадкового елемента за середній час $O(1)$, необхідно поєднати дві структури:
1. **`ArrayList` (динамічний масив):** забезпечує вибір випадкового елемента за $O(1)$, оскільки елементи розташовані в пам'яті неперервно (ми можемо просто згенерувати випадковий індекс від `0` до `size - 1`). Також підтримує додавання в кінець за амортизоване $O(1)$.
2. **`HashMap` (хеш-таблиця):** відображає значення елемента на його індекс в `ArrayList`. Це дозволяє за $O(1)$ визначати, чи є елемент у наборі, та знаходити його індекс для видалення.

**Ключовий прийом при видаленні:**
Видалення довільного елемента з масиву зазвичай вимагає $O(N)$ через зсув елементів. Щоб зробити це за $O(1)$:
1. Знаходимо індекс елемента, який потрібно видалити, через `HashMap`.
2. Міняємо цей елемент місцями з **останнім** елементом в `ArrayList`.
3. Оновлюємо індекс останнього елемента в `HashMap`.
4. Видаляємо останній елемент з `ArrayList` за $O(1)$ за допомогою методу `remove(list.size() - 1)`.

```java
// Базовий прийом swap-and-pop
int idxToRemove = map.get(val);
int lastVal = list.get(list.size() - 1);

// Записуємо останній елемент на місце видаленого
list.set(idxToRemove, lastVal);
map.put(lastVal, idxToRemove);

// Видаляємо останній елемент
list.remove(list.size() - 1);
map.remove(val);
```

**Коли застосовувати:**
- При розробці контейнерів та кешів із випадковим доступом та швидким оновленням.
- У алгоритмах рандомізованого тестування чи вибірки без повторень (наприклад, тасування карт).

**Складність:**
- **insert, remove, getRandom:** $O(1)$ у середньому.
