# Medium_01_GroupAnagrams — Довідка

## Техніка: Canonical Key -> HashMap<Key, List<String>>

**Суть:** кожній анаграмі призначаємо один і той самий "канонічний
ключ" — представника всього класу еквівалентності. Найпростіший ключ —
відсортовані символи рядка (усі анаграми після сортування дають
однаковий рядок). Альтернативний ключ — сигнатура частот символів
(масив з 26 чисел, перетворений на рядок), яка будується за O(L) без
сортування. Далі просто накопичуємо рядки в `HashMap<String,
List<String>>` за цим ключем.

```java
Map<String, List<String>> groups = new HashMap<>();
for (String s : strs) {
    char[] chars = s.toCharArray();
    Arrays.sort(chars);
    String key = new String(chars);
    groups.computeIfAbsent(key, k -> new ArrayList<>()).add(s);
}
return new ArrayList<>(groups.values());
```

**Коли застосовувати:** будь-яка задача "згрупуй елементи за спільною
властивістю/формою" — анаграми, ізоморфні рядки, дублікати з точністю
до перестановки/нормалізації.

**Складність:** O(N * L log L) із сортуванням ключа (N — кількість
рядків, L — середня довжина) або O(N * L) із частотним ключем; O(N *
L) пам'яті.
