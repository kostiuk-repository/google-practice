# Easy_03_ValidAnagram — Довідка

## Техніка: Frequency Map Comparison

**Суть:** будуємо мапу "символ -> кількість" для першого рядка,
збільшуючи лічильники, а потім проходимо другий рядок, зменшуючи ті ж
самі лічильники. Якщо в кінці всі лічильники нульові (і довжини
рядків рівні) — рядки є анаграмами. Для алфавіту з 26 малих латинських
літер мапу можна замінити на `int[26]` для O(1) без хешування.

```java
Map<Character, Integer> freq = new HashMap<>();
for (char c : s.toCharArray()) {
    freq.merge(c, 1, Integer::sum);
}
for (char c : t.toCharArray()) {
    freq.merge(c, -1, Integer::sum);
}
for (int count : freq.values()) {
    if (count != 0) return false;
}
return s.length() == t.length();
```

**Коли застосовувати:** будь-яке порівняння "чи складаються дві
колекції символів/елементів з однакового набору" — анаграми,
перевірка мультимножин, перевірка "чи можна переставити один рядок в
інший".

**Складність:** O(N) час (N — сумарна довжина рядків), O(1) пам'яті
для фіксованого алфавіту або O(K) для K унікальних символів.
