# Easy_07_IsomorphicStrings — Довідка

## Техніка: Two-Way Character Mapping (Bijection via Two HashMaps)

**Суть:** щоб перевірити біекцію (взаємно однозначну відповідність)
між символами двох рядків, потрібні ДВІ мапи: `mapST` (s -> t) і
`mapTS` (t -> s). На кожній позиції i перевіряємо, що якщо `s[i]` вже
мапилось раніше — воно мапиться на те саме `t[i]`, і одночасно що
`t[i]` вже мапилось раніше — на те саме `s[i]`. Одної мапи недостатньо:
без зворотної перевірки два різних символи джерела можуть помилково
відобразитись на один символ призначення.

```java
Map<Character, Character> mapST = new HashMap<>();
Map<Character, Character> mapTS = new HashMap<>();
for (int i = 0; i < s.length(); i++) {
    char cs = s.charAt(i), ct = t.charAt(i);
    if (mapST.containsKey(cs) && mapST.get(cs) != ct) return false;
    if (mapTS.containsKey(ct) && mapTS.get(ct) != cs) return false;
    mapST.put(cs, ct);
    mapTS.put(ct, cs);
}
return true;
```

**Коли застосовувати:** будь-яка задача про взаємно однозначну
відповідність між двома послідовностями токенів (символи, слова,
числа) — ізоморфізм рядків, патерни, кодування "формою" замість
конкретних значень.

**Складність:** O(N) час, O(K) пам'яті, де K — кількість унікальних
токенів.

## Drills

1. **Easy_07_01** — лише одностороннє мапування (без зворотної перевірки) — показує хибний позитив.
2. **Easy_07_02** — повернути саму мапу відповідності, а не boolean.
3. **Easy_07_03** — порахувати, скільки рядків зі списку ізоморфні заданому.
4. **Easy_07_04** — узагальнення токена з символу на ціле число (int[] масиви).
5. **Easy_07_05** — капстоун: згрупувати список рядків за патерном ізоморфізму (канонічний ключ).

Розв'язуй у порядку 01 → 05, не підглядаючи в Easy_07_IsomorphicStrings.java.
