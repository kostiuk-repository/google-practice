# Medium_03_LongestConsecutiveSequence — Довідка

## Техніка: Пошук послідовностей за допомогою HashSet (Sequence Building with HashSet)

**Суть:**
Ця техника дозволяє знаходити найдовшу послідовність послідовних цілих чисел (наприклад, $1, 2, 3, 4$) у несортованому масиві за лінійний час $O(N)$ без використання сортування, яке вимагає $O(N \log N)$.

Основна ідея:
1. Записати всі числа з масиву в `HashSet`. Це дозволяє перевіряти наявність будь-якого числа за $O(1)$.
2. Для кожного числа `x` перевірити, чи є воно **початком послідовності**. Число `x` є початком, якщо `x - 1` відсутнє в наборі.
3. Якщо `x` є початком послідовності, ми інкрементуємо значення (`x + 1`, `x + 2`, ...), доки наступні елементи є в `HashSet`, підраховуючи довжину.
4. Оскільки ми розширюємо послідовність лише від її початкового елемента, кожне число перевіряється вкладеним циклом щонайбільше двічі (один раз під час лінійного обходу і один раз під час підрахунку послідовності). Це гарантує загальну складність $O(N)$.

```java
// Базовий шаблон для пошуку найдовшої послідовності
Set<Integer> set = new HashSet<>();
for (int num : nums) set.add(num);

int longest = 0;
for (int num : set) {
    // Перевіряємо, чи є num початком послідовності
    if (!set.contains(num - 1)) {
        int currentNum = num;
        int currentStreak = 1;
        
        while (set.contains(currentNum + 1)) {
            currentNum += 1;
            currentStreak += 1;
        }
        longest = Math.max(longest, currentStreak);
    }
}
```

**Коли застосовувати:**
- Коли потрібно знайти послідовні діапазони чи підмножини елементів, де порядок у вихідному масиві не має значення.
- Коли потрібно розв'язати задачу за $O(N)$ за допомогою додаткової пам'яті.

**Складність:**
- **Часова складність:** $O(N)$ — побудова `HashSet` займає $O(N)$, і обхід елементів також $O(N)$.
- **Просторова складність:** $O(N)$ — для зберігання унікальних елементів масиву в `HashSet`.
