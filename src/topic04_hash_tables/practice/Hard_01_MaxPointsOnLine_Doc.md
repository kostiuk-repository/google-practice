# Hard_01_MaxPointsOnLine — Довідка

## Техніка: Групування за нахилом прямої (Slope Grouping with HashMap)

**Суть:**
Будь-які дві точки на площині задають унікальну пряму. Будь-яка третя точка лежить на тій самій прямій, якщо нахил (slope) між першою і другою точкою дорівнює нахилу між першою і третьою точкою.

Основна ідея для кожної точки $P_i$ (як опорної точки / pivot):
1. Обчислити нахил від $P_i$ до всіх інших точок $P_j$.
2. Згрупувати ці точки за нахилом у хеш-таблиці `Map<Slope, Integer>`, підраховуючи частоту кожного нахилу.
3. Максимальне значення в карті плюс 1 (сама точка $P_i$) дає найбільшу кількість точок на прямій, що проходить через $P_i$.
4. Оновити глобальний максимум.

**Важлива математична деталь:**
Нахил обчислюється як $m = \frac{\Delta y}{\Delta x} = \frac{y_2 - y_1}{x_2 - x_1}$.
Використання типу `double` для представлення нахилу може призвести до помилок точності (rounding errors). Замість цього нахил представляють у вигляді **нескоротного дробу** за допомогою алгоритму Евкліда для знаходження найбільшого спільного дільника (НСД / GCD):
- $\Delta x' = \frac{\Delta x}{\text{GCD}(\Delta y, \Delta x)}$
- $\Delta y' = \frac{\Delta y}{\text{GCD}(\Delta y, \Delta x)}$
У хеш-таблиці ключ зберігається як рядок `"dy/dx"`.

```java
// Обчислення НСД
int gcd(int a, int b) {
    return b == 0 ? a : gcd(b, a % b);
}
```

**Коли застосовувати:**
- У задачах обчислювальної геометрії для пошуку колінеарних точок, паралельних прямих або прямих, що перетинаються.

**Складність:**
- **Часова складність:** $O(N^2)$ — оскільки ми перебираємо кожну пару точок. Обчислення GCD виконується за логарифмічний час $O(\log(\min(\Delta x, \Delta y)))$.
- **Просторова складність:** $O(N)$ — для збереження нахилів щодо однієї опорної точки в хеш-таблиці.

## Тренування (Drills):

1. **Hard_01_01_MaxPointsOnLine (Calculate Slope as Double)**: Обчислити нахил як `double` з урахуванням ділення на нуль (вертикальні лінії).
2. **Hard_01_02_MaxPointsOnLine (Calculate Slope as Coprime Fraction String)**: Реалізувати обчислення нескоротного дробу нахилу для запобігання похибкам округлення.
3. **Hard_01_03_MaxPointsOnLine (Max Points Passing Through One Point)**: Знайти максимальну кількість точок на одній прямій, що обов'язково проходить через задану опорну точку.
4. **Hard_01_04_MaxPointsOnLine (Count Collinear Triplets)**: Підрахувати кількість трійок точок, що лежать на одній прямій, використовуючи косий добуток векторів.
5. **Hard_01_05_MaxPointsOnLine (Max Points on a Line with Duplicates)**: Знайти максимальну кількість точок на прямій, якщо точки вхідного масиву можуть збігатися.
