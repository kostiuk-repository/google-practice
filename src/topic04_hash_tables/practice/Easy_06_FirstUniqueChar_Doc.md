# Easy_06_FirstUniqueChar — Довідка

## Техніка: Two-Pass Frequency Map

**Суть:** прохід 1 — будуємо `HashMap<Character, Integer>` частот
кожного символу. Прохід 2 — знову йдемо рядком зліва направо і
повертаємо перший символ, чия частота дорівнює 1. Два проходи потрібні,
бо частота символу стає відомою лише після повного проходу рядка.

```java
Map<Character, Integer> freq = new HashMap<>();
for (char c : s.toCharArray()) {
    freq.merge(c, 1, Integer::sum);
}
for (int i = 0; i < s.length(); i++) {
    if (freq.get(s.charAt(i)) == 1) {
        return i;
    }
}
return -1;
```

**Коли застосовувати:** "перший/останній елемент з певною
властивістю частоти" — унікальні символи, слова, елементи. Для
потокових даних (де не можна зробити другий прохід) використовуйте
`LinkedHashMap`, щоб підтримувати порядок і оновлювати відповідь
інкрементально.

**Складність:** O(N) час (два проходи — все одно лінійно), O(K)
пам'яті, де K — кількість унікальних символів.

## Drills

1. **Easy_06_01** — повернути сам символ, а не індекс.
2. **Easy_06_02** — знайти ОСТАННІЙ унікальний символ (скан у зворотному напрямку).
3. **Easy_06_03** — порахувати кількість символів, що зустрічаються рівно один раз.
4. **Easy_06_04** — узагальнення з символів на СЛОВА в реченні.
5. **Easy_06_05** — капстоун: потокова версія через LinkedHashMap (add + firstUnique за O(1) амортизовано).

Розв'язуй у порядку 01 → 05, не підглядаючи в Easy_06_FirstUniqueChar.java.
