# Medium_02_SubarraySumEqualsK — Довідка

## Техніка: Префіксні суми та Хеш-таблиця (Prefix Sum + HashMap)

**Суть:**
Ця техніка поєднує концепцію **префіксних сум** (сума всіх елементів масиву від початку до поточного індексу) з **хеш-таблицею** для пошуку підмасивів із заданою сумою за лінійний час $O(N)$.

Нехай $S[i]$ — це сума елементів від $nums[0]$ до $nums[i]$. Тоді сума підмасиву від індексу $j+1$ до $i$ дорівнює:
$$\text{Sum}(j+1, i) = S[i] - S[j]$$

Якщо ми шукаємо підмасив із сумою $k$, то має виконуватися:
$$S[i] - S[j] = k \implies S[j] = S[i] - k$$

Зберігаючи частоту або індекси раніше зустрінутих префіксних сум у хеш-таблиці, ми можемо за $O(1)$ перевірити, чи існує таке $S[j]$, яке задовольняє цю рівність.

```java
// Базовий шаблон для підрахунку підмасивів із сумою k
Map<Integer, Integer> seen = new HashMap<>();
seen.put(0, 1); // Базовий випадок для префіксної суми 0
int sum = 0, count = 0;
for (int num : nums) {
    sum += num;
    if (seen.containsKey(sum - k)) {
        count += seen.get(sum - k);
    }
    seen.put(sum, seen.getOrDefault(sum, 0) + 1);
}
```

**Коли застосовувати:**
- Коли потрібно знайти кількість, довжину (найдовшу/найкоротшу) або існування підмасивів із заданою сумою чи властивістю, пов'язаною із сумою (наприклад, кратність $k$, рівна кількість 0 і 1 тощо).
- Масив може містити як додатні, так і від'ємні числа (якщо числа лише додатні, часто краще підходить sliding window без додаткової пам'яті).

**Складність:**
- **Часова складність:** $O(N)$ — один прохід по масиву, кожна операція з хеш-таблицею виконується в середньому за $O(1)$.
- **Просторова складність:** $O(N)$ — у найгіршому випадку всі префіксні суми унікальні й будуть збережені в хеш-таблиці.
