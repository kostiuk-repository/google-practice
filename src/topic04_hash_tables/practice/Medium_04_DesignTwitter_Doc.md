# Medium_04_DesignTwitter — Довідка

## Техніка: Проектування класів та злиття K відсортованих списків (Class Design & Merge K Sorted Lists)

**Суть:**
Задача проектування спрощеного Twitter вимагає координації кількох структур даних для забезпечення ефективного виконання операцій публікації, підписки/відписки та генерації стрічки новин.

Компоненти:
1. **Зв'язки підписок:** Зберігаються за допомогою `Map<Integer, Set<Integer>>`, де ключ — це `followerId`, а значення — множина унікальних ID користувачів, на яких він підписаний.
2. **Публікації твітів:** Зберігаються за допомогою `Map<Integer, List<Tweet>>`, де кожен твіт має унікальний `tweetId` та глобальний `timestamp`.
3. **Генерація стрічки новин (getNewsFeed):** Це класична задача **злиття K відсортованих списків (Merge K Sorted Lists)**. Кожен список твітів (власних або підписок) уже відсортований за спаданням часу публікації. Ми можемо використати чергу з пріоритетом (`PriorityQueue`), щоб витягти 10 найсвіжіших твітів із сукупності списків.

```java
// Елемент для черги з пріоритетом, що зберігає посилання на твіт, індекс у списку та сам список
class FeedIterator {
    List<Tweet> tweets;
    int index;
    
    FeedIterator(List<Tweet> tweets) {
        this.tweets = tweets;
        this.index = 0;
    }
    
    boolean hasNext() {
        return index < tweets.size();
    }
    
    Tweet next() {
        return tweets.get(index++);
    }
}
```

**Коли застосовувати:**
- При проектуванні систем із великою кількістю зв'язків типу "багато до багатьох" (соціальні мережі, системи рекомендацій).
- Коли потрібно об'єднати результати з декількох джерел, кожне з яких уже відсортоване.

**Складність:**
- **postTweet, follow, unfollow:** $O(1)$ у середньому.
- **getNewsFeed:** $O(K \log K + N \log K)$, де $K$ — кількість підписок, $N$ — ліміт стрічки (10). За допомогою черги з пріоритетом ми робимо щонайбільше 10 кроків вилучення.

## Тренування (Drills):

1. **Medium_04_01_DesignTwitter (Simple Follow Tracker)**: Реалізувати збереження зв'язків фоловерів на базі хеш-таблиці множин.
2. **Medium_04_02_DesignTwitter (User Tweet Store)**: Реалізувати збереження твітів користувача з можливістю повернення списку від найновішого до найстарішого.
3. **Medium_04_03_DesignTwitter (Merge Two Sorted Tweet Lists)**: Об'єднати два відсортовані списки твітів у один топ-10 за допомогою двох вказівників.
4. **Medium_04_04_DesignTwitter (Merge K Sorted Tweet Streams)**: Злити K відсортованих потоків твітів у топ-10 найновіших за допомогою `PriorityQueue`.
5. **Medium_04_05_DesignTwitter (News Feed Cache)**: Інтегрувати компоненти в єдиний трекер стрічки новин з ручним встановленням міток часу.
