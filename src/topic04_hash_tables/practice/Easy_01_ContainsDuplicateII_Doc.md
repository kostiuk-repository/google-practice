# Easy_01_ContainsDuplicateII — Довідка

## Техніка: HashMap value -> lastSeenIndex (Sliding Window через мапу)

**Суть:** проходимо масив зліва направо і зберігаємо в `HashMap<Integer,
Integer>` останній побачений індекс кожного значення. На кожному кроці
`i`, якщо значення `nums[i]` вже є в мапі і `i - lastIndex <= k` —
знайдено "близький дублікат". Після перевірки мапа оновлюється новим
індексом. Це еквівалентно ковзному вікну розміру `k`, реалізованому не
масивом, а мапою "значення -> позиція останньої появи".

```java
Map<Integer, Integer> lastIndex = new HashMap<>();
for (int i = 0; i < nums.length; i++) {
    Integer prev = lastIndex.get(nums[i]);
    if (prev != null && i - prev <= k) {
        return true;
    }
    lastIndex.put(nums[i], i);
}
```

**Коли застосовувати:** будь-яка задача "чи повторюється значення в
межах відстані k" — детекція дублікатів у ковзному вікні, rate
limiting (чи траплялась подія недавно), дедуплікація потоку подій за
часовим вікном.

**Складність:** O(N) час (один прохід, O(1) операції з мапою), O(N)
пам'яті у гіршому випадку (усі значення різні).

## Drills

1. **Easy_01_01** — повернути самі індекси [i,j] близького дубліката, а не boolean.
2. **Easy_01_02** — знайти мінімальну відстань між будь-якими дублікатами (без фіксованого k).
3. **Easy_01_03** — порахувати ВСІ пари близьких дублікатів, а не зупинятись на першій.
4. **Easy_01_04** — той самий пошук, але між ДВОМА масивами замість одного.
5. **Easy_01_05** — капстоун: додається допуск по значенню (valueDiff), потрібна структура, впорядкована за значенням.
