# Easy_09_DesignHashSet — Довідка

## Техніка: Separate Chaining (масив бакетів + зв'язні списки)

**Суть:** створюємо масив фіксованого розміру (`buckets`), де кожен
елемент — список (`LinkedList<Integer>`). Хеш-функція (зазвичай
`key % buckets.length`) визначає, у який бакет потрапляє ключ. Усі
ключі з однаковим хешем (колізії) зберігаються в одному списку —
"ланцюжку". `add` / `contains` / `remove` спочатку обчислюють індекс
бакета за O(1), а потім лінійно сканують список у цьому бакеті.

```java
private static final int SIZE = 769; // просте число зменшує колізії
private List<Integer>[] buckets = new LinkedList[SIZE];

private int hash(int key) { return key % SIZE; }

public void add(int key) {
    int idx = hash(key);
    if (buckets[idx] == null) buckets[idx] = new LinkedList<>();
    if (!buckets[idx].contains(key)) buckets[idx].add(key);
}
```

**Коли застосовувати:** будь-яка задача "реалізуй хеш-таблицю з
нуля" — розуміння того, що ховається за `HashSet`/`HashMap`, важливе
для пояснення внутрішньої роботи на співбесіді, а також для сценаріїв
з обмеженим API.

**Складність:** O(1) середній час для add/contains/remove за умови
рівномірного розподілу хешів і достатньої кількості бакетів; O(N) у
гіршому випадку (усі ключі в одному бакеті).
