# Hard_03_LRUCache — Довідка

## Техніка: Поєднання Двобічно Зв'язаного Списку та Хеш-карти (HashMap + Doubly Linked List)

**Суть:**
Для розробки кешу з витісненням найменш недавно використаних елементів (Least Recently Used) за час $O(1)$ для операцій `get` та `put` стандартної хеш-таблиці недостатньо, оскільки вона не підтримує порядок елементів. Зв'язаний список підтримує порядок, але пошук у ньому займає $O(N)$.

Рішення полягає в їх поєднанні:
1. **`HashMap<Integer, Node>`**: відображає ключ на вузол двобічно зв'язаного списку. Забезпечує доступ до будь-якого вузла за $O(1)$.
2. **`Doubly Linked List` (двобічно зв'язаний список)**: містить самі ключі та значення. Найновіші елементи розташовуються ближче до голови (head), а застарілі — ближче до хвоста (tail).

**Основні операції:**
- **`get(key)`**: Знаходимо вузол через `HashMap`, переміщуємо його в голову списку (як найновіший) та повертаємо значення.
- **`put(key, value)`**: Якщо ключ уже є, оновлюємо значення та переміщуємо вузол у голову. Якщо ключа немає:
  - Створюємо новий вузол і додаємо його в голову списку та в `HashMap`.
  - Якщо розмір кешу перевищив ліміт (capacity), видаляємо вузол з хвоста списку (як найменш використовуваний) та вилучаємо його з `HashMap`.

```java
// Базова структура вузла
class Node {
    int key;
    int val;
    Node prev;
    Node next;
    
    Node(int key, int val) {
        this.key = key;
        this.val = val;
    }
}
```

**Коли застосовувати:**
- При розробці систем кешування даних у пам'яті (in-memory caching), керуванні сторінками пам'яті в ОС (page replacement algorithms).

**Складність:**
- **Часова складність:** $O(1)$ для `get` та `put`.
- **Просторова складність:** $O(C)$, де $C$ — ємність (capacity) кешу.
