# Easy_08_NextGreaterElementI — Довідка

## Техніка: Монотонний стек + Хеш-мапа (Next Greater Element)

**Суть:** Використання монотонно спадного стеку для пошуку наступного більшого елемента для всіх чисел у nums2, та збереження результатів у HashMap (key: число, value: наступне більше число) для швидкого O(1) пошуку під час обробки nums1.

```java
Map<Integer, Integer> map = new HashMap<>();
Stack<Integer> stack = new Stack<>();
for (int num : nums2) {
    while (!stack.isEmpty() && stack.peek() < num) {
        map.put(stack.pop(), num);
    }
    stack.push(num);
}
// lookup for nums1...
```

**Коли застосовувати:** Коли потрібно знайти наступні більші елементи для підмножини елементів іншого масиву.

**Складність:** O(N + M) час, де N та M — довжини масивів; O(N) пам'ять для збереження мапи та стека.

## Drills

   1. **Easy_08_01** — Наївний пошук наступного більшого елемента для всього масиву за O(N^2).
   2. **Easy_08_02** — Знаходження наступних більших елементів за допомогою спадного стека.
   3. **Easy_08_03** — Побудова асоціативної карти наступних більших елементів за допомогою стека.
   4. **Easy_08_04** — Обробка запитів пошуку для підмасиву nums1 за допомогою карти переходів.
   5. **Easy_08_05** — Знаходження наступного більшого елемента в кільцевому масиві (циклічний обхід).

Розв'язуй у порядку 01 → 05, не підглядаючи в Easy_08_NextGreaterElementI.java.
