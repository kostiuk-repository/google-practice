# Hard_03_WordSearchII — Довідка

## Техніка: Trie + DFS (Backtracking on Grid)

**Суть:** Пошук набору слів на двовимірній символьній сітці. Замість того, щоб запускати DFS пошук для кожного слова окремо (що дає O(W * R * C * 4^L)), ми вставляємо всі слова у Trie і робимо один обхід DFS по сітці. Ми переходимо до сусідніх клітинок лише тоді, коли відповідний символ є дитиною поточного вузла Trie.

```java
// Обхід DFS з Trie
void dfs(char[][] board, int r, int c, TrieNode node, List<String> res) {
    char ch = board[r][c];
    if (ch == '#' || node.children[ch - 'a'] == null) return;
    node = node.children[ch - 'a'];
    if (node.word != null) {
        res.add(node.word);
        node.word = null; // уникнення дублікатів
    }
    board[r][c] = '#';
    dfs(board, r+1, c, node, res);
    // ... інші напрямки ...
    board[r][c] = ch;
}
```

**Коли застосовувати:** Пошук великої кількості слів на сітці символів, гра Boggle, словникові ігри.

**Складність:** Час: O(R * C * 4^L), де L — довжина найдовшого слова (але на практиці набагато швидше завдяки Trie pruning). Пам'ять: O(W * L) для Trie.

## Drills

   2. **Hard_03_02** — Пошук слів на сітці, які починаються з простого префікса (без Trie).
   3. **Hard_03_03** — Побудова Trie дерева для списку шуканих слів.
   4. **Hard_03_04** — DFS пошук слів за допомогою Trie нод.
   5. **Hard_03_05** — Реалізація DFS обходу з видаленням знайдених слів з Trie (pruning).
