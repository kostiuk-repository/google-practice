# Easy_09_DailyTemperaturesEasy — Довідка

## Техніка: Монотонний стек для відстаней індексів (Daily Temperatures)

**Суть:** Збереження індексів у монотонно спадному стеці. Коли поточна температура більша за температуру на індексі з вершини стека, ми виштовхуємо індекс та записуємо різницю індексів (current_index - stack_index) як кількість днів очікування.

```java
int[] ans = new int[T.length];
Stack<Integer> stack = new Stack<>();
for (int i = 0; i < T.length; i++) {
    while (!stack.isEmpty() && T[i] > T[stack.peek()]) {
        int prev = stack.pop();
        ans[prev] = i - prev;
    }
    stack.push(i);
}
return ans;
```

**Коли застосовувати:** Пошук відстані (у кроках/індексах) до найближчого більшого чи меншого елемента в послідовності.

**Складність:** O(N) час, O(N) пам'ять.

## Drills

   1. **Easy_09_01** — Наївне обчислення днів до теплішої температури за O(N^2).
   2. **Easy_09_02** — Оптимальне обчислення днів очікування за допомогою стека індексів.
   3. **Easy_09_03** — Обчислення днів до найближчої холоднішої температури.
   4. **Easy_09_04** — Пошук максимальної кількості днів очікування теплішого дня.
   5. **Easy_09_05** — Пошук днів до дня, який тепліший щонайменше на задану величину limit.
