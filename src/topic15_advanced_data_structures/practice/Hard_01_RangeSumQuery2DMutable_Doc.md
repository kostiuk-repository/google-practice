# Hard_01_RangeSumQuery2DMutable — Довідка

## Техніка: 2D Segment Tree / 2D Binary Indexed Tree

**Суть:** Узагальнення дерева сегментів або Fenwick tree на дві розмірності. Кожна нода по одній осі (рядки) містить дерево сегментів по іншій осі (стовпці). Це дозволяє оновлювати клітинки matrix[row][col] за O(log R * log C) та запитувати суму прямокутної області за O(log R * log C).

```java
// Концептуальний запит 2D BIT
void update(int row, int col, int val) {
    for (int r = row; r <= R; r += r & -r) {
        for (int c = col; c <= C; c += c & -c) {
            tree[r][c] += val;
        }
    }
}
```

**Коли застосовувати:** Часті точкові зміни в матриці та запити суми підматриць.

**Складність:** Оновлення/Запит: O(log R * log C) за час, O(R * C) пам'ять.

## Drills

   1. **Hard_01_01** — Обчислення суми елементів у конкретному рядку row від col1 до col2.
   2. **Hard_01_02** — Побудова статичного 2D префіксного масиву для незмінної матриці.
   3. **Hard_01_03** — Оновлення значення клітинки в матриці та відповідне коригування рядкових префіксних сум.
   4. **Hard_01_04** — Додавання значення delta до клітинки (row, col) у 2D дереві Фенвіка.
   5. **Hard_01_05** — Запит префіксної суми області від (0,0) до (row, col) у 2D дереві Фенвіка.

Розв'язуй у порядку 01 → 05, не підглядаючи в Hard_01_RangeSumQuery2DMutable.java.
