# Easy_02_ImplementQueueUsingStacks — Довідка

## Техніка: Емуляція FIFO через LIFO (Two-Stack Transfer)

**Суть:** один стек (`input`) приймає всі нові елементи (push = O(1)).
Коли потрібно прочитати/видалити елемент з фронту черги, а `output`
порожній — усі елементи переливають зі `input` в `output` через
`pop`/`push`, що ОДИН РАЗ розвертає їхній порядок на протилежний. Після
цього `output.pop()` завжди повертає найстаріший елемент — саме те, що
потрібно для FIFO. Поки `output` не спорожніє, нові перекачування не
потрібні — звідси амортизована O(1) складність.

```java
Deque<Integer> input = new ArrayDeque<>(), output = new ArrayDeque<>();
void push(int x) { input.push(x); }
int pop() {
    if (output.isEmpty()) {
        while (!input.isEmpty()) output.push(input.pop());
    }
    return output.pop();
}
```

**Коли застосовувати:** будь-яка задача "реалізуй ADT X через примітиви
ADT Y", де потрібно розвернути порядок доступу — черга через стек(и),
перевірка паліндрому через стек, k-та позиція від початку структури, що
підтримує лише LIFO-доступ.

**Складність:** push O(1); pop/peek — амортизовано O(1) (кожен елемент
переливається зі input в output рівно один раз за весь час життя),
хоча одна конкретна операція може коштувати O(n).
