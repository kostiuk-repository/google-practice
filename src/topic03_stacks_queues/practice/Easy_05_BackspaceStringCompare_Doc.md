# Easy_05_BackspaceStringCompare — Довідка

## Техніка: Stack Simulation of Editing / Backwards Two-Pointer

**Суть:** щоб змоделювати введення тексту з backspace, є два
еквівалентні підходи. (1) Стек: кожен звичайний символ — `push`,
кожен '#' — `pop` (якщо стек не порожній). (2) O(1) пам'яті: йдемо з
КІНЦЯ рядка до початку, лічильником рахуємо кількість "відкладених"
backspace; коли лічильник > 0 — пропускаємо символ (це той, що буде
стерто); коли лічильник == 0 — це символ, що "вижив", і саме його
порівнюємо з відповідним символом іншого рядка.

```java
// Stack-based
Deque<Character> st = new ArrayDeque<>();
for (char c : s.toCharArray()) {
    if (c == '#') { if (!st.isEmpty()) st.pop(); }
    else st.push(c);
}
```

**Коли застосовувати:** будь-яка задача про симуляцію редагування
тексту (backspace, undo, delete) — ключова ознака: операція видалення
залежить від ІСТОРІЇ введення (що вже надруковано), а не лише від
поточного символу, тому потрібна структура з пам'яттю про попередні
кроки (стек) АБО зворотний прохід, що уникає явного зберігання цієї
історії.

**Складність:** стековий підхід — O(N) час, O(N) пам'ять; підхід з
двома вказівниками з кінця — O(N) час, O(1) додаткової пам'яті.

## Drills

1. **Easy_05_01** — ізольований крок: побудова фінального тексту через стек (без порівняння).
2. **Easy_05_02** — підрахунок довжини фінального тексту зворотним проходом без побудови рядка.
3. **Easy_05_03** — дзеркальний напрям стирання: forward-delete замість backspace.
4. **Easy_05_04** — явна реалізація O(1)-пам'яті підходу з двома вказівниками з кінця.
5. **Easy_05_05** — capstone: узагальнення з 2 рядків на N рядків.
