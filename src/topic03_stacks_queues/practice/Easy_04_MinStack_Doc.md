# Easy_04_MinStack — Довідка

## Техніка: Augmented Stack (стек з допоміжним агрегатом)

**Суть:** поруч з основним стеком значень тримаємо ДРУГИЙ стек, який на
кожному `push` записує поточне значення агрегату (мінімум, максимум,
сума тощо) з урахуванням щойно доданого елемента. На кожному `pop` обидва
стеки знімають по одному елементу синхронно, тож вершина допоміжного
стеку завжди відображає правильний агрегат для того, що залишилось.
Це дає O(1) доступ до агрегату без повторного сканування.

```java
Deque<Integer> values = new ArrayDeque<>(), mins = new ArrayDeque<>();
void push(int val) {
    values.push(val);
    mins.push(mins.isEmpty() ? val : Math.min(val, mins.peek()));
}
void pop() { values.pop(); mins.pop(); }
int getMin() { return mins.peek(); }
```

**Коли застосовувати:** будь-яка задача "стек із O(1) доступом до
агрегату" (мінімум, максимум, сума, добуток) — ключова ознака: агрегат
має бути "скасовуваним" при видаленні (можна відновити попереднє
значення), інакше потрібне складніше кодування (Drill 3) або зовсім
інша структура (Drill 5).

**Складність:** push/pop/top/getAggregate — усі O(1) час; O(N)
додаткової пам'яті для допоміжного стеку (або O(1) додатково понад
основний стек у варіанті з кодуванням різниці).
