# Easy_03_MakeStringGreat — Довідка

## Техніка: Stack-Based Adjacent Cancellation

**Суть:** проходимо рядок (чи масив) зліва направо, підтримуючи стек
"вже опрацьованих" елементів. Для кожного нового елемента порівнюємо
його з вершиною стеку за певною умовою "скасування" (canceling
condition) — наприклад, та сама буква в іншому регістрі, або числа, що
у сумі дають 9, або точні протилежності (x і -x). Якщо умова
виконується — вершину стеку видаляємо (`pop`) замість додавання нового
елемента; інакше — додаємо (`push`). Фінальний вміст стеку і є
результатом.

```java
Deque<Character> stack = new ArrayDeque<>();
for (char c : s.toCharArray()) {
    if (!stack.isEmpty() && isBadPair(stack.peek(), c)) {
        stack.pop();
    } else {
        stack.push(c);
    }
}
```

**Коли застосовувати:** будь-яка задача "видаляй сусідні елементи, поки
є пари, що задовольняють умову X" — леткі дужки, дублікати, взаємно
протилежні значення. Ключова ознака: скасування може каскадно відкрити
нову пару після видалення (наприклад, "abBa" → після видалення "bB"
лишається "aa", яка теж скасовується) — саме тому потрібен стек, а не
одноразовий прохід.

**Складність:** O(N) час (кожен символ push і pop не більше одного
разу), O(N) додаткової пам'яті для стеку.
