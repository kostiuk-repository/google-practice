# Easy_01_ImplementStackUsingQueues — Довідка

## Техніка: Емуляція LIFO через FIFO (Rotation Trick)

**Суть:** черга підтримує лише операції з одного кінця в інший (`offer`
у кінець, `poll` з початку), тому щоб емулювати стек (LIFO) за допомогою
черги (FIFO), новий елемент після додавання "прокручують" (rotate)
назад до початку черги: `size` разів поспіль виконують `poll`, а
результат одразу `offer`-ять назад, доки щойно доданий елемент не
опиниться спереду. Це змінює логічний порядок черги так, ніби останній
доданий елемент є першим кандидатом на видалення — точно як у стеку.

```java
Queue<Integer> q = new LinkedList<>();
void push(int x) {
    q.offer(x);
    for (int i = 0; i < q.size() - 1; i++) {
        q.offer(q.poll());
    }
}
int pop() { return q.poll(); } // тепер front черги == top стеку
```

**Коли застосовувати:** будь-яка задача "реалізуй ADT X, використовуючи
лише примітиви ADT Y" — стек через чергу(и), черга через стек(и), а
також будь-яка задача, де треба інвертувати порядок доступу до
структури даних, маючи доступ лише до одного її кінця.

**Складність:** push O(n) через ротацію, pop/top/empty O(1) — або
навпаки (push O(1), pop O(n)), залежно від того, яку операцію ви
робите "вартісною", як показано у Drill 3.
