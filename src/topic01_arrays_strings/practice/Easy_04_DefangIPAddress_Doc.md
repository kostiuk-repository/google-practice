# Easy_04_DefangIPAddress — Довідка

## Суть теми / алгоритму простими словами
Техніка **Single-Pass Scan-and-Replace (Однопрохідне сканування та заміна)** за допомогою `StringBuilder` полягає в тому, що ми обходимо рядок зліва направо символ за символом, аналізуємо кожен символ і дописуємо в буфер `StringBuilder` або сам символ, або його модифіковану версію (підрядок-замінник). 
Використання `StringBuilder` замість стандартних методів конкатенації рядків (`+` або `+=`) або багаторазових викликів `String.replace()` запобігає створенню великої кількості тимчасових об'єктів у пам'яті (Garbage Collection overhead), що критично для високої продуктивності.

Базова задача: знешкодити (defang) IP-адресу, замінивши всі крапки `.` на `[.]`.

## Візуалізація та покроковий розбір (Visualization & Walkthrough)

### a) Звичайний випадок: `address = "1.1.1.1"`
Початковий стан: `StringBuilder sb = new StringBuilder()`.

| Індекс `i` | Символ `c` | Умова `c == '.'` | Дія над `StringBuilder` | Поточний вміст `StringBuilder` |
| :--- | :--- | :--- | :--- | :--- |
| 0 | `'1'` | `false` | `sb.append('1')` | `"1"` |
| 1 | `'.'` | `true` | `sb.append("[.]")` | `"1[.]"` |
| 2 | `'1'` | `false` | `sb.append('1')` | `"1[.]1"` |
| 3 | `'.'` | `true` | `sb.append("[.]")` | `"1[.]1[.]"` |
| 4 | `'1'` | `false` | `sb.append('1')` | `"1[.]1[.]1"` |
| 5 | `'.'` | `true` | `sb.append("[.]")` | `"1[.]1[.]1[.]"` |
| 6 | `'1'` | `false` | `sb.append('1')` | `"1[.]1[.]1[.]1"` |

**Результат:** `"1[.]1[.]1[.]1"`.

### b) Граничний випадок: `address = ""` (порожній рядок)
Початковий стан: `StringBuilder sb = new StringBuilder()`.
Цикл по символах порожнього рядка не виконується жодного разу.

**Результат:** `""`.

## Ключова ідея та шаблон коду (Core Concept & Code Template)

Рядки в Java є незмінними (immutable). Кожна конкатенація рядків виду `s = s + c` створює новий об'єкт `String` у пам'яті. Якщо ми робимо це для кожного символу рядка довжини $N$, це призводить до квадратичної складності за часом $O(N^2)$ та створення $O(N)$ непотрібних тимчасових об'єктів. 
`StringBuilder` працює на базі динамічного масиву символів, який автоматично розширюється за потреби. Додавання нового символу в кінець буфера відбувається за амортизований час $O(1)$. 
Також ручний прохід у циклі `for` дозволяє виконати всі необхідні заміни за один прохід (Single-Pass), на відміну від виклику кількох методів `.replace()`, кожен з яких окремо сканує рядок і створює новий масив символів.

### Базовий шаблон коду (Java)
```java
public String defangIPaddr(String address) {
    // Створюємо StringBuilder для ефективного накопичення результату.
    // Заздалегідь можна вказати початкову ємність, оскільки ми знаємо, 
    // що результуючий рядок буде довшим за вхідний на 6 символів (3 крапки * 2 додаткові дужки).
    StringBuilder sb = new StringBuilder(address.length() + 6);
    
    // Проходимо по кожному символу вхідного рядка
    for (int i = 0; i < address.length(); i++) {
        char c = address.charAt(i);
        // Якщо символ є крапкою, додаємо екрановану версію
        if (c == '.') {
            sb.append("[.]");
        } else {
            // Інакше додаємо символ без змін
            sb.append(c);
        }
    }
    
    // Перетворюємо накопичений буфер у звичайний рядок
    return sb.toString();
}
```

### Аналіз складності (Complexity Analysis)
- **Time Complexity (Часова складність):** $O(N)$, де $N$ — довжина рядка `address`. Ми проходимо по кожному символу рівно один раз.
- **Space Complexity (Просторова складність):** $O(N)$ для збереження результуючого рядка в `StringBuilder` та фінального `String`. За кількістю додаткової пам'яті (overhead) це $O(1)$, якщо не враховувати пам'ять під вихідний результат.

## Граничні випадки та підводні камені (Edge Cases & Common Pitfalls)

1. **Неефективна робота з пам'яттю:** Використання конкатенації `+` у циклі — найпоширеніша помилка новачків.
2. **Використання `String.replace()` або регулярних виразів (`replaceAll`):** Хоча `address.replace(".", "[.]")` дасть правильний результат і виглядає лаконічно, під капотом регулярні вирази компілюють патерни, що значно повільніше за простий порівняльний аналіз символів `char`.
3. **Неправильна початкова ємність `StringBuilder`:** За замовчуванням `StringBuilder` ініціалізується з ємністю 16 символів. Якщо результат буде більшим, Java буде кілька разів копіювати внутрішній масив для його розширення. Завжди корисно вказувати очікуваний розмір `new StringBuilder(address.length() + expectedExtra)`.

### 5. [Easy_04_05_DefangIPAddress.java](file:///C:/Users/kosti/Projects/google-practice/src/topic01_arrays_strings/practice/Easy_04_05_DefangIPAddress.java) — Multi-Rule Replacement
- **Мета:** За один прохід виконати два правила заміни: `.` -> `[.]` та `:` -> `[:]`.
- **Відмінність від основної задачі:** Два різні символи замінюються на два різні підрядки. Це демонструє головну перевагу кастомного парсингу перед окремими викликами методів заміни.
- **Підказка щодо модифікації:** Додайте конструкцію `if-else if-else`: `if (c == '.') sb.append("[.]"); else if (c == ':') sb.append("[:]"); else sb.append(c);`.
- **Приклад:**
  - Вхід: `"1.2.3:4"`
  - Вихід: `"1[.]2[.]3[:]4"`
