# Easy_04_DefangIPAddress — Довідка

## Техніка: Single-Pass Scan-and-Replace через StringBuilder

**Суть:** рядок читається зліва направо один раз, символ за символом.
Для кожного символу перевіряється умова (чи це "спеціальний" символ,
що підлягає заміні). Якщо так — у `StringBuilder` дописується довший
рядок-замінник; якщо ні — символ копіюється як є. На відміну від
`String.replace()`, який всередині теж будує новий буфер, ручний
прохід дозволяє застосувати кілька різних правил заміни в одному
проході, без кількох послідовних алокацій.

```java
StringBuilder sb = new StringBuilder();
for (char c : s.toCharArray()) {
    if (c == '.') {
        sb.append("[.]");
    } else {
        sb.append(c);
    }
}
return sb.toString();
```

**Коли застосовувати:** екранування спеціальних символів (HTML-escape,
defang IP/URL, цензура тексту), будь-яка трансформація рядка "символ
-> символ або підрядок", коли потрібен один прохід і O(N) час.

**Складність:** O(N) час (N — довжина вхідного рядка), O(N) пам'яті на
результуючий `StringBuilder`/`String`.

## Drills

1. **Easy_04_01** — замінити кожну "." в доменному імені на "(dot)".
2. **Easy_04_02** — екранувати символ "&" на "&amp;" (HTML-екранування).
3. **Easy_04_03** — замінити кожну голосну (a,e,i,o,u, лише малі літери) на "*".
4. **Easy_04_04** — замінити кожен "-" на " to ".
5. **Easy_04_05** — застосувати ДВА правила заміни в одному проході: "." -> "[.]" та ":" -> "[:]".

Розв'язуй у порядку 01 → 05, не підглядаючи в Easy_04_DefangIPAddress.java.
