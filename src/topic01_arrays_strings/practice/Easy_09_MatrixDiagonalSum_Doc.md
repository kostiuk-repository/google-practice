# Easy_09_MatrixDiagonalSum — Довідка

## Суть теми / алгоритму простими словами
Техніка **Diagonal Index Formula (Формула індексування діагоналей)** дозволяє обходити головну та побічну діагоналі квадратної чи прямокутної матриці без необхідності сканувати кожну клітинку вкладеними циклами (що дало б $O(N^2)$).
У квадратній матриці розміру $N \times N$:
- Елементи **головної діагоналі** мають однакові індекси рядка та стовпця: `mat[i][i]`.
- Елементи **побічної діагоналі** мають суму індексів рядка та стовпця, яка дорівнює $N - 1$. Тобто для рядка `i` відповідний стовпець буде `N - 1 - i`. Формула елемента: `mat[i][N - 1 - i]`.

Базова задача: обчислити суму всіх елементів на головній та побічній діагоналях квадратної матриці, враховуючи перетин (центральний елемент) лише один раз.

## Інженерний контекст: обходити структуру за формулою, а не повним scan

Матриця має $n^2$ комірок, але діагоналі містять лише $O(n)$ релевантних позицій. Формули координат дозволяють генерувати потрібні адреси без перевірки кожної комірки.

### Де зустрічається

- робота з diagonal/banded matrices у numerical computing;
- вибір pixel diagonals і anti-diagonals в image processing;
- аналіз board states у grid-based games;
- sparse traversal, де структура визначає координати потрібних елементів;
- trace-like metrics для квадратних матриць.

Ширший патерн: якщо множину потрібних координат можна описати формулою, генеруйте її напряму. Це часто зменшує $O(n^2)$ scan до $O(k)$, де `k` — фактична кількість корисних комірок.

### Що перевіряє співбесіда

- чи виведете `n - 1 - i`, а не запам’ятаєте формулу без пояснення;
- чи не порахуєте центр двічі для непарного `n`;
- чи поясните відмінність парного й непарного розміру;
- чи не використаєте вкладений цикл із зайвою перевіркою;
- чи перевірите numeric overflow суми для великих значень.

Для парного `n` перетину немає; для непарного — рівно один. Це дозволяє або перевіряти центр у циклі, або наприкінці один раз відняти `mat[n/2][n/2]`.

### a) Звичайний випадок: непарна розмірність `N = 3` (діагоналі перетинаються)
Матриця `mat`:
```
[[1, 2, 3],
 [4, 5, 6],
 [7, 8, 9]]
```
Розмірність `N = 3`. Проходимо `i` від `0` до `2`.
Центр знаходиться на індексі `(N - 1) / 2 = (3 - 1) / 2 = 1`.

| Індекс `i` | Елемент головної `mat[i][i]` | Індекс побічної `N-1-i` | Умова перетину `i == N-1-i` | Елемент побічної `mat[i][N-1-i]` | Додавання до суми | Поточна сума |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **Старт** | — | — | — | — | — | 0 |
| 0 | `mat[0][0] = 1` | `2` | `0 == 2` (Ні) | `mat[0][2] = 3` | `1 + 3` | 4 |
| 1 | `mat[1][1] = 5` | `1` | `1 == 1` (Так) | `mat[1][1] = 5` (ігноруємо) | `5` | `4 + 5 = 9` |
| 2 | `mat[2][2] = 9` | `0` | `2 == 0` (Ні) | `mat[2][0] = 7` | `9 + 7` | `9 + 16 = 25` |

**Результат:** `25`.

## Ключова ідея та шаблон коду (Core Concept & Code Template)

Головна перевага використання формул індексування — зменшення кількості ітерацій. Замість вкладених циклів $O(N^2)$, ми робимо один прохід $O(N)$, де $N$ — кількість рядків. 
На кожній ітерації `i` ми гарантовано беремо елемент `mat[i][i]` з головної діагоналі. Елемент побічної діагоналі `mat[i][n-1-i]` ми додаємо лише за умови, що він не збігається з головним (тобто `i != n-1-i`). Це усуває потребу в окремому відніманні центру після завершення циклу.

### Базовий шаблон коду (Java)
```java
public int diagonalSum(int[][] mat) {
    int sum = 0;
    int n = mat.length;
    
    for (int i = 0; i < n; i++) {
        // Додаємо елемент головної діагоналі
        sum += mat[i][i];
        
        // Якщо поточний стовпець побічної діагоналі не збігається з головним
        if (i != n - 1 - i) {
            // Додаємо елемент побічної діагоналі
            sum += mat[i][n - 1 - i];
        }
    }
    
    return sum;
}
```

### Аналіз складності (Complexity Analysis)
- **Time Complexity (Часова складність):** $O(N)$, де $N$ — розмірність квадратної матриці (кількість рядків). Ми робимо лише $N$ кроків в одному циклі.
- **Space Complexity (Просторова складність):** $O(1)$, оскільки ми використовуємо константну кількість змінних пам'яті для сумування.

## Граничні випадки та підводні камені (Edge Cases & Common Pitfalls)

1. **Подвійний підрахунок центру:** Якщо розмірність матриці є непарною (наприклад, $3 \times 3$), то елемент в центрі (`mat[1][1]`) належить обом діагоналям. Якщо його не відфільтрувати умовою `i != n-1-i`, результат буде завищеним на значення цього центрального елемента.
2. **Непрямокутні або неквадратні матриці:** Якщо матриця не є квадратною, то побічна діагональ може бути невизначена стандартним чином або вимагати додаткового налаштування меж циклу. Формулу `mat.length` слід використовувати з урахуванням того, що довжина рядка та кількість стовпців можуть відрізнятися.
3. **Порожня матриця:** Завжди варто перевіряти вхідні дані на `mat == null || mat.length == 0` для запобігання помилкам доступу.

- **Мета:** Перевірити, чи є квадратна матриця симетричною відносно своєї головної діагоналі.
- **Відмінність від основної задачі:** Замість суми потрібне порівняння симетричних елементів: `mat[i][j] == mat[j][i]`.
- **Підказка щодо модифікації:** Потрібен вкладений цикл. Щоб не перевіряти пари двічі, внутрішній цикл `j` може йти від `0` до `i`. Якщо `mat[i][j] != mat[j][i]`, повертаємо `false`.
- **Приклад:**
  - Вхід: `mat = [[1, 2], [2, 5]]`
  - Вихід: `true` (оскільки `mat[0][1] == mat[1][0]`).
