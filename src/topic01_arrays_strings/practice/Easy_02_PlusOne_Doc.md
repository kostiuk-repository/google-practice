# Easy_02_PlusOne — Довідка

## Суть теми / алгоритму простими словами
Техніка **Carry / Borrow Propagation (Перенесення розряду / запозичення)** використовується для виконання арифметичних операцій над числами, які представлені у вигляді масивів цифр (так звана довга арифметика). Оскільки такі числа можуть бути набагато більшими за ліміти стандартних типів даних (`int` чи `long`), ми моделюємо додавання чи віднімання у стовпчик: починаємо з наймолодшого розряду (кінця масиву) та рухаємося ліворуч, передаючи залишок (carry) або запозичуючи одиницю (borrow) на кожному кроці.

Базова задача: збільшити число, представлене масивом цифр, на 1.

Прохід виконується з кінця масиву: від `i = 2` до `0`.

| :--- | :--- | :--- | :--- | :--- |

### b) Граничний випадок: `digits = [9, 9]` (перенесення розрядів виходить за межі масиву)
Прохід з кінця масиву: від `i = 1` до `0`.

| Крок (Індекс `i`) | Елемент `digits[i]` | Перевірка `digits[i] < 9` | Дія | Зміна масиву |
| :--- | :--- | :--- | :--- | :--- |
| 1 (1) | 9 | `9 < 9` є `false` | `digits[i] = 0` та перехід ліворуч | `[9, 0]` |
| 2 (0) | 9 | `9 < 9` є `false` | `digits[i] = 0` та завершення циклу | `[0, 0]` |

Оскільки цикл завершився і всі цифри стали `0`, потрібно створити новий масив розміром `len + 1` та встановити перший елемент у `1`.

| Новий масив | Встановлення `result[0] = 1` | Фінальний стан |
| :--- | :--- | :--- |
| `[0, 0, 0]` | `result[0] = 1` | `[1, 0, 0]` |

**Результат:** `[1, 0, 0]`.

## Ключова ідея та шаблон коду (Core Concept & Code Template)

Основний трюк підходу "Plus One" полягає в тому, що якщо під час додавання одиниці до цифри вона менша за `9`, ми можемо просто збільшити її на `1` та одразу завершити роботу алгоритму. Якщо ж цифра дорівнює `9`, вона перетворюється на `0`, і ми переносимо переповнення (`carry = 1`) на наступний крок циклу (ліворуч). Якщо цикл завершився, і ми не вийшли з методу — це означає, що число складалося виключно з дев'яток (наприклад, `999`). У цьому випадку довжина числа збільшується на одиницю (буде `1000`), і нам потрібно виділити новий масив, де перша цифра `1`, а решта — `0` (за замовчуванням у Java).

### Базовий шаблон коду (Java)
```java
public int[] plusOne(int[] digits) {
    // Проходимо масив справа наліво (від молодших розрядів до старших)
    for (int i = digits.length - 1; i >= 0; i--) {
        // Якщо цифра менша за 9, додаємо одиницю та одразу завершуємо роботу
        if (digits[i] < 9) {
            digits[i]++;
            return digits;
        }
        // Якщо цифра дорівнює 9, вона стає 0, а carry переноситься далі ліворуч
        digits[i] = 0;
    }
    
    // Якщо цикл завершився, це означає, що всі цифри були 9 (наприклад, 99 -> 100)
    int[] resultArray = new int[digits.length + 1];
    resultArray[0] = 1; // Решта елементів автоматично ініціалізуються нулями
    
    return resultArray;
}
```

### Аналіз складності (Complexity Analysis)
- **Time Complexity (Часова складність):** $O(N)$, де $N$ — кількість цифр у масиві `digits`. У найгіршому випадку (всі дев'ятки) ми переглянемо всі цифри один раз. У середньому випадку алгоритм завершиться набагато раніше.
- **Space Complexity (Просторова складність):** $O(1)$ додаткової пам'яті (in-place модифікація), якщо немає розширення розрядності. У найгіршому випадку — $O(N)$ для створення нового масиву результату.

## Граничні випадки та підводні камені (Edge Cases & Common Pitfalls)

1. **Ігнорування останнього перенесення розряду (carry):** Помилкою є забути розширити масив, коли `carry > 0` після закінчення циклу (наприклад, для `[9, 9]` повернути `[0, 0]` замість `[1, 0, 0]`).
2. **Непотрібне копіювання при збільшенні розрядності:** При створенні нового масиву для випадку `999 -> 1000` немає потреби копіювати старі нулі. Новий масив Java ініціалізує нулями автоматично, достатньо встановити перший елемент у `1`.
3. **Оновлення цифр під час обходу зліва направо:** Додавання завжди починається з найменш значущого розряду (індекс `digits.length - 1`). Якщо почати з `0`, то логіка перенесення розряду повністю зламається.
