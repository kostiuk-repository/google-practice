# Read N Characters Given Read4 II - Call multiple times — Довідка

## Суть теми / алгоритму простими словами
Низькорівневі операції введення-виведення (I/O) часто зчитують дані фіксованими блоками (наприклад, по 4 КБ, або у нашому випадку — по 4 символи за допомогою `read4`). Проте користувачеві може знадобитися довільна кількість символів (наприклад, 1, 3 або 10). 

Головна складність виникає тоді, коли метод `read` викликається **кілька разів поспіль** для одного й того самого файлу. Оскільки `read4` зчитує дані лише «пакетами» по 4, він може зчитати більше символів, ніж попросив користувач у поточному виклику. Ці «зайві» символи (leftovers) не можна викидати, оскільки вони мають бути повернуті під час наступних викликів `read`. Для цього ми створюємо **внутрішній буфер стану** (instance-level buffer), який зберігає невикористані символи між викликами.

### Опис базової задачі
Використовуючи API `read4(char[] buf4)` (яке зчитує до 4 символів з файлу у тимчасовий буфер та повертає кількість реально зчитаних символів), потрібно реалізувати метод `read(char[] buf, int n)`, який зчитує `n` символів у буфер `buf`. Метод може викликатися багаторазово.

## Візуалізація та покроковий розбір (Visualization & Walkthrough)

Нехай вміст файлу: `"abcdefgh"` (8 символів).
Клас зберігає стан у трьох змінних:
- `internalBuffer` (масив розміром 4)
- `internalPtr` (індекс наступного символу для читання з `internalBuffer`)
- `internalCount` (кількість валідних символів у `internalBuffer`)

### a) Звичайний випадок: перший виклик `read(buf, 3)`, а потім другий виклик `read(buf, 3)`

#### **Виклик 1: `read(buf, 3)` (потрібно 3 символи)**
1. Початковий стан: `internalPtr = 0`, `internalCount = 0`.
2. Буфер порожній (`internalPtr == internalCount`). Викликаємо `read4(internalBuffer)`.
   - `read4` зчитує `"abcd"`.
   - Стан: `internalBuffer = ['a', 'b', 'c', 'd']`, `internalCount = 4`, `internalPtr = 0`.
3. Копіюємо 3 символи до `buf`:
   - `buf[0] = 'a'` (`internalPtr` стає 1)
   - `buf[1] = 'b'` (`internalPtr` стає 2)
   - `buf[2] = 'c'` (`internalPtr` стає 3)
4. Повертаємо **3**.
5. Залишок у буфері: на індексі 3 лежить символ `'d'`.

```
internalBuffer: [ 'a', 'b', 'c', 'd' ]
                           ^      ^
                      internalPtr  internalCount
```

#### **Виклик 2: `read(buf, 3)` (потрібно ще 3 символи)**
1. Початковий стан: `internalBuffer = ['a', 'b', 'c', 'd']`, `internalPtr = 3`, `internalCount = 4`.
2. Маємо залишок (`internalPtr < internalCount`). Копіюємо `'d'`:
   - `buf[0] = 'd'` (`internalPtr` стає 4).
3. Потрібно ще 2 символи. Оскільки тепер `internalPtr == internalCount` (4 == 4), буфер вичерпано.
4. Викликаємо `read4(internalBuffer)`.
   - `read4` зчитує `"efgh"`.
   - Стан: `internalBuffer = ['e', 'f', 'g', 'h']`, `internalCount = 4`, `internalPtr = 0`.
5. Копіюємо ще 2 потрібні символи:
   - `buf[1] = 'e'` (`internalPtr` стає 1)
   - `buf[2] = 'f'` (`internalPtr` стає 2)
6. Повертаємо **3**.
7. Залишок у буфері: `'g'` та `'h'` (`internalPtr = 2`, `internalCount = 4`).

---

### b) Граничний випадок (файл коротший за запит): вміст `"ab"`, виклик `read(buf, 5)`

1. Початковий стан: `internalPtr = 0`, `internalCount = 0`.
2. Буфер порожній. Викликаємо `read4(internalBuffer)`.
   - `read4` зчитує `"ab"`. Повертає 2.
   - Стан: `internalBuffer = ['a', 'b', ?, ?]`, `internalCount = 2`, `internalPtr = 0`.
3. Копіюємо 2 доступні символи:
   - `buf[0] = 'a'` (`internalPtr` стає 1)
   - `buf[1] = 'b'` (`internalPtr` стає 2)
4. Потрібно ще 3 символи. Оскільки `internalPtr == internalCount` (2 == 2), викликаємо `read4(internalBuffer)`.
   - Файл закінчився, `read4` повертає 0.
   - Стан: `internalCount = 0`, `internalPtr = 0`.
5. Оскільки `read4` повернув 0 (кінець файлу), ми негайно зупиняємось.
6. Повертаємо **2**.

## Ключова ідея та шаблон коду (Core Concept & Code Template)

Головний концепт полягає в абстрагуванні читання по одному символу. Якщо ми реалізуємо допоміжний метод для отримання наступного символу з файлу (який прозоро працює з буфером та `read4`), то логіка основного методу `read` спрощується до звичайного циклу копіювання символів.

### Базовий шаблон коду (Java)

```java
package topic01_arrays_strings.practice;

public class Hard_02_Read4 extends Reader4 {

    // Внутрішній буфер для збереження "зайвих" зчитаних символів
    private char[] internalBuffer = new char[4];
    // Поточна позиція для читання з внутрішнього буфера
    private int internalPtr = 0;
    // Кількість валідних символів у внутрішньому буфері
    private int internalCount = 0;

    public Hard_02_Read4() {
        super();
    }

    public Hard_02_Read4(String content) {
        super(content);
    }

    /**
     * Зчитує n символів у буфер buf.
     * Може викликатися багаторазово.
     */
    public int read(char[] buf, int n) {
        int copiedChars = 0;
        
        // Копіюємо символи один за одним, поки не досягнемо n або кінця файлу (EOF)
        while (copiedChars < n) {
            // Якщо внутрішній буфер порожній або повністю прочитаний,
            // заповнюємо його новими даними за допомогою read4
            if (internalPtr == internalCount) {
                internalCount = read4(internalBuffer);
                internalPtr = 0;
                
                // Якщо read4 повернув 0, це означає кінець файлу
                if (internalCount == 0) {
                    break;
                }
            }
            
            // Копіюємо один символ із внутрішнього буфера до буфера призначення
            buf[copiedChars] = internalBuffer[internalPtr];
            copiedChars++;
            internalPtr++;
        }
        
        return copiedChars;
    }
}
```

### Аналіз складності
- **Часова складність (Time Complexity):** $O(N)$, де $N$ — кількість символів, які потрібно зчитати. Кожен символ із файлу зчитується за допомогою `read4` рівно один раз і копіюється в буфер `buf` також один раз.
- **Просторова складність (Space Complexity):** $O(1)$, оскільки розмір `internalBuffer` фіксований (завжди 4 елементи) і не залежить від обсягу зчитуваних даних чи кількості викликів.

## Граничні випадки та підводні камені (Edge Cases & Common Pitfalls)

1. **Втрата залишків (Leftover characters):**
   Найпоширеніша помилка — не зберігати стан `internalBuffer`, `internalPtr` та `internalCount` як поля класу (поля екземпляра). Якщо зробити їх локальними змінними методу `read`, то при кожному новому виклику стан буде скидатися, і символи, зчитані «про запас», будуть безповоротно втрачені.

2. **Неправильна обробка кінця файлу (EOF):**
   Якщо `read4` повертає менше 4 символів (або 0), це означає кінець файлу. Потрібно правильно зупиняти цикл читання та не намагатися читати з порожнього буфера.

3. **Скидання вказівника (`internalPtr`):**
   При кожному новому виклику `read4` обов'язково потрібно скидати `internalPtr` в 0, а також оновлювати `internalCount` значенням, яке повернув `read4`.

## Розбір Drills (Drills Breakdown)

### 1. Hard_02_01 — Read Single Call
- **Мета:** Спрощений варіант, де гарантується лише ОДИН виклик методу `read`.
- **Відмінність від основної:** Нам не потрібні змінні класу `internalBuffer`, `internalPtr` та `internalCount`. Усі залишкові символи, які зчитав останній виклик `read4` понад необхідні `n` символів, можна просто проігнорувати.
- **Підказка:** Використовуйте локальний буфер розміром 4. Зчитуйте в циклі за допомогою `read4` і копіюйте тільки потрібну кількість символів до досягнення `n`.
- **Приклад:** Файл = `"abcdef"`, `read(buf, 3)`. Локально зчитуємо 4 символи (`"abcd"`), копіюємо `"abc"` у `buf` і завершуємо роботу. Символ `'d'` втрачається, але оскільки повторних викликів не буде, це не проблема.

### 2. Hard_02_02 — Peek Next Character
- **Мета:** Реалізувати метод `peek()`, який повертає наступний символ у файлі без його споживання (наступний виклик `peek()` або `read()` знову поверне цей символ).
- **Відмінність від основної:** Символ не видаляється з внутрішнього буфера (вказівник `internalPtr` не інкрементується).
- **Підказка:** Якщо буфер порожній (`internalPtr == internalCount`), виконайте `read4` для його наповнення. Після цього поверніть `internalBuffer[internalPtr]`, але **не** збільшуйте `internalPtr`. Якщо файл порожній, повертайте `'\0'`.
- **Приклад:** Файл = `"abc"`. Перший `peek()` завантажує `"abc"` у буфер і повертає `'a'` (`internalPtr = 0`). Другий `peek()` бачить, що буфер не порожній, і знову повертає `'a'`.

### 3. Hard_02_03 — Read Line
- **Мета:** Зчитувати символи доти, доки не зустрінеться символ нового рядка `\n` або кінець файлу.
- **Відмінність від основної:** Замість фіксованої довжини `n`, умовою зупинки є знаходження символу `\n`.
- **Підказка:** Читайте символи по одному (використовуючи буфер). Додавайте їх до `StringBuilder`. Як тільки зустрінете `\n`, припиніть читання та поверніть накопичений рядок. Символи після `\n` мають залишатися у буфері для наступного виклику.
- **Приклад:** Файл = `"line1\nline2"`. Перший виклик повертає `"line1"`, другий — `"line2"`.

### 4. Hard_02_04 — Count Vowels Read
- **Мета:** Зчитати `n` символів та підрахувати кількість голосних літер (`'a', 'e', 'i', 'o', 'u'`) серед них.
- **Відмінність від основної:** Символи аналізуються внутрішньо, буфер користувача `buf` не заповнюється.
- **Підказка:** Логіка просування по файлу та робота з `internalBuffer` повністю ідентичні до основної задачі, але замість копіювання символів у масив ми перевіряємо кожен символ і збільшуємо лічильник, якщо він є голосною.
- **Приклад:** Файл = `"leetcode"`, виклик `countVowels(8)` повертає `4` (літери `'e'`, `'e'`, `'o'`, `'e'`).

### 5. Hard_02_05 — Read With Unread
- **Мета:** Реалізувати `read(buf, n)` та додатково `unread(m)`, який повертає останні `m` зчитаних символів назад у потік.
- **Відмінність від основної:** Можливість «відкотити» читання останніх `m` символів.
- **Підказка:** Оскільки `unread` повертає символи назад, найпростіший спосіб — використовувати додаткову структуру даних (наприклад, стек або двобічну чергу `pushbackBuffer`). При виклику `unread(m)` ми додаємо останні `m` символів у цей буфер. Метод `read` спочатку має перевіряти `pushbackBuffer`, забирати символи звідти, і лише коли він порожній — переходити до стандартного читання з `internalBuffer` та `read4`.
- **Приклад:** Файл = `"abcde"`. `read(buf, 3)` повертає 3, `buf = ['a','b','c']`. Виклик `unread(2)` повертає `'b'` та `'c'`. Наступний `read(buf, 3)` зчитує спочатку `'b'`, потім `'c'`, а потім бере `'d'` з файлу. Повертає `3` з `buf = ['b','c','d']`.
