# Easy_05_ReplaceElements — Довідка

## Техніка: Suffix/Prefix Running Aggregate (прохід з одного краю з накопиченим агрегатом)

**Суть:** замість того, щоб для кожного елемента заново сканувати весь
"хвіст" або "голову" масиву (що дає O(N^2)), масив проходиться ОДИН
РАЗ у певному напрямку (справа наліво для suffix, зліва направо для
prefix), і на кожному кроці підтримується змінна-агрегат (`best`), що
зберігає максимум/мінімум усіх елементів, які вже пройдені. Результат
для поточного елемента записується ДО оновлення агрегата поточним
значенням (бо агрегат має описувати сусідів, а не сам елемент).

```java
int[] result = new int[arr.length];
int best = -1; // або Integer.MIN_VALUE / MAX_VALUE залежно від задачі
for (int i = arr.length - 1; i >= 0; i--) {
    result[i] = best;
    best = Math.max(best, arr[i]);
}
```

**Коли застосовувати:** "замінити кожен елемент на найбільший/
найменший серед сусідів зліва/справа", "порівняти кожен елемент з
екстремумом іншої частини масиву" — типова ознака: потрібна
інформація про ВЕСЬ хвіст/голову масиву відносно поточної позиції, а
наївне рішення дає O(N^2).

**Складність:** O(N) час, O(N) пам'яті на результуючий масив (O(1)
якщо модифікувати масив на місці і не потрібен окремий вихідний
масив).

## Drills

1. **Easy_05_01** — замінити кожен елемент на НАЙМЕНШИЙ праворуч; останній елемент = -1.
2. **Easy_05_02** — замінити кожен елемент на найбільший ЛІВОРУЧ (прохід зліва направо); перший елемент = -1.
3. **Easy_05_03** — замінити кожен елемент на (елемент - найбільший праворуч); останній елемент = -1.
4. **Easy_05_04** — перевірити, чи кожен елемент (крім останнього) <= найбільшого праворуч.
5. **Easy_05_05** — замінити кожен елемент на найбільший ВКЛЮЧНО із собою до кінця масиву (без -1 для останнього).

Розв'язуй у порядку 01 → 05, не підглядаючи в Easy_05_ReplaceElements.java.
