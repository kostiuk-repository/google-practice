# Medium_02_SpiralMatrix — Довідка

## Суть теми / алгоритму простими словами
Обхід матриці по спіралі (Spiral Traversal) — це задача на симуляцію руху по сітці з постійною зміною напрямку за годинниковою (або проти годинникової) стрілкою. Основна складність полягає в тому, щоб правильно звужувати межі обходу та вчасно зупинятися, уникаючи повторного відвідування елементів або виходу за межі масиву.

Для розв'язання використовують **техніку чотирьох меж, що звужуються (Shrinking Boundary Pointers)**. Ми створюємо чотири вказівники: `top`, `bottom`, `left`, `right`. Після проходження кожної сторони прямокутника ми зсуваємо відповідну межу всередину.

## Інженерний контекст: boundary-driven state machine

Спіраль — лише конкретний порядок. Головний патерн полягає в тому, що **необроблена область описується компактними межами**, а кожна фаза споживає одну її сторону.

### Де зустрічається

- perimeter/ring processing у зображеннях і grid simulations;
- обхід concentric layers у matrix algorithms;
- формування print/layout order для двовимірних даних;
- peeling algorithms, де зовнішній шар видаляється перед внутрішнім;
- boundary-based traversal без окремої `visited[][]`.

Чотири bounds є стисненим станом. Вони замінюють $O(rows \cdot columns)$ boolean-матрицю `visited`, бо жорсткий порядок руху сам гарантує, які комірки вже оброблені.

### Межі патерну

- Він розрахований на rectangular dense matrix.
- Для ragged arrays або obstacles простіше перейти до direction vectors + visited set.
- Для дуже великих матриць spiral access має слабшу cache locality, ніж row-major scan.

### Що перевіряє співбесіда

- чи зробите повторні guard checks перед bottom і left pass;
- чи коректно обробите `1×n`, `n×1` і непарний центр;
- чи доведете, що кожна комірка відвідується рівно один раз;
- чи не змішаєте оновлення меж із coordinate movement;
- чи відокремите termination invariant від окремих напрямків.

### a) Звичайний випадок: матриця $3 \times 3$

**Початкова матриця:**
```
1  2  3
4  5  6
7  8  9
```
Ініціалізація меж: `top = 0`, `bottom = 2`, `left = 0`, `right = 2`.
Кількість елементів: 9.

| Крок / Сторона | Межі перед кроком | Напрямок руху | Пройдені елементи | Оновлення меж після кроку |
| :--- | :--- | :--- | :--- | :--- |
| **1. Верхня** | `t=0, b=2, l=0, r=2` | Вправо по рядку `top` | `1, 2, 3` | `top` збільшується на 1 $\rightarrow$ `top = 1` |
| **2. Права** | `t=1, b=2, l=0, r=2` | Вниз по стовпцю `right` | `6, 9` | `right` зменшується на 1 $\rightarrow$ `right = 1` |
| **3. Нижня** | `t=1, b=2, l=0, r=1` | Вліво по рядку `bottom` (якщо `top <= bottom`) | `8, 7` | `bottom` зменшується на 1 $\rightarrow$ `bottom = 1` |
| **4. Ліва** | `t=1, b=1, l=0, r=1` | Вгору по стовпцю `left` (якщо `left <= right`) | `4` | `left` збільшується на 1 $\rightarrow$ `left = 1` |
| **5. Центр** | `t=1, b=1, l=1, r=1` | Вправо по рядку `top` | `5` | `top` збільшується на 1 $\rightarrow$ `top = 2` |

Наступна ітерація: умова `top <= bottom` (`2 <= 1`) стає хибною, цикл завершується.
**Результат:** `[1, 2, 3, 6, 9, 8, 7, 4, 5]`.

---

### b) Граничний випадок: однорядкова матриця $1 \times 3$

**Початкова матриця:**
```
[1, 2, 3]
```
Ініціалізація меж: `top = 0`, `bottom = 0`, `left = 0`, `right = 2`.

1. **Верхня сторона**: йдемо від `left` (0) до `right` (2) по рядку `top` (0).
   - Записані елементи: `1, 2, 3`.
   - Оновлення: `top++` $\rightarrow$ `top = 1`.
2. **Права сторона**: йдемо від `top` (1) до `bottom` (0). Умова циклу `1 <= 0` хибна, нічого не додаємо.
   - Оновлення: `right--` $\rightarrow$ `right = 1`.
3. **Нижня сторона**: перевірка `top <= bottom` $\rightarrow$ `1 <= 0` хибна. Цей крок пропускається.
   *(Якби ми не перевірили `top <= bottom`, ми б знову пройшли по єдиному рядку у зворотному напрямку та додали дублікати `3, 2, 1`!)*
4. **Ліва сторона**: перевірка `left <= right` $\rightarrow$ `0 <= 1` істинна, але цикл `i` від `bottom` (0) до `top` (1) у зворотному напрямку (`i--`) не виконується, бо `0 >= 1` хибно.
   - Оновлення: `left++` $\rightarrow$ `left = 1`.

Цикл закінчується, оскільки `top <= bottom` (`1 <= 0`) хибно.
**Результат:** `[1, 2, 3]`.

## Ключова ідея та шаблон коду

Ми симулюємо рух по колу. Оскільки межі постійно стискаються, ми гарантовано пройдемо всі елементи по спіралі без повторень.

### Java-шаблон (Звуження меж)
Доступ до файлу: [Medium_02_SpiralMatrix.java](file:///C:/Users/kosti/Projects/google-practice/src/topic01_arrays_strings/practice/Medium_02_SpiralMatrix.java)

```java
import java.util.ArrayList;
import java.util.List;

public List<Integer> spiralOrder(int[][] matrix) {
    List<Integer> result = new ArrayList<>();
    if (matrix == null || matrix.length == 0 || matrix[0].length == 0) {
        return result;
    }
    
    int m = matrix.length;
    int n = matrix[0].length;
    
    int top = 0;
    int bottom = m - 1;
    int left = 0;
    int right = n - 1;
    
    while (top <= bottom && left <= right) {
        // 1. Рух вправо по верхній межі
        for (int j = left; j <= right; j++) {
            result.add(matrix[top][j]);
        }
        top++; // звужуємо верхню межу
        
        // 2. Рух вниз по правій межі
        for (int i = top; i <= bottom; i++) {
            result.add(matrix[i][right]);
        }
        right--; // звужуємо праву межу
        
        // 3. Рух вліво по нижній межі (перевірка наявності невикористаного рядка)
        if (top <= bottom) {
            for (int j = right; j >= left; j--) {
                result.add(matrix[bottom][j]);
            }
            bottom--; // звужуємо нижню межу
        }
        
        // 4. Рух вгору по лівій межі (перевірка наявності невикористаного стовпця)
        if (left <= right) {
            for (int i = bottom; i >= top; i--) {
                result.add(matrix[i][left]);
            }
            left++; // звужуємо ліву межу
        }
    }
    
    return result;
}
```

### Аналіз складності:
- **Часова складність (Time Complexity):** $\mathcal{O}(M \times N)$, де $M$ — кількість рядків, $N$ — кількість стовпців. Кожен елемент матриці відвідується рівно один раз.
- **Просторова складність (Space Complexity):** $\mathcal{O}(1)$ додаткової пам'яті (якщо не враховувати список для збереження результату `result`).

## Граничні випадки та підводні камені

1. **Дублювання елементів у вузьких матрицях**:
   - *Проблема:* В однорядкових ($1 \times N$) або одностовпцевих ($M \times 1$) матрицях, після того, як `top` збільшився або `right` зменшився, нижня та ліва межі можуть накластися на вже пройдені ділянки.
   - *Запобігання:* Обов'язково перевіряти `top <= bottom` перед нижнім проходом та `left <= right` перед лівим проходом всередині циклу `while`.
2. **Порожня матриця**:
   - *Проблема:* Якщо `matrix.length == 0` або `matrix[0].length == 0`, звернення до меж викличе вихід за межі масиву.
   - *Запобігання:* Завжди починати з перевірки на порожнечу (guard clause).
3. **Неправильний інкремент/декремент меж**:
   - *Проблема:* Легко помилитися і написати `left--` замість `left++` або навпаки.
   - *Запобігання:* Пам'ятайте геометричну логіку: `top` рухається вниз (`top++`), `bottom` вгору (`bottom--`), `left` вправо (`left++`), `right` вліво (`right--`).

- **Відмінність від основної:** Зміна послідовності обходу меж.
- **Підказка щодо модифікації:** 
  Послідовність руху:
  1. Вниз по лівому стовпцю `left` (`i` від `top` до `bottom`). Збільшити `left++`.
  2. Вправо по нижньому рядку `bottom` (`j` від `left` до `right`). Зменшити `bottom--`.
  3. Вгору по правому стовпцю `right` (`i` від `bottom` до `top`). Зменшити `right--`.
  4. Вліво по верхньому рядку `top` (`j` від `right` до `left`). Збільшити `top++`.
  Не забувати додавати перевірки меж `left <= right` та `top <= bottom` перед 3-м та 4-м кроками.
- **Приклад:**
  - Вхід: `[[1,2,3],[4,5,6],[7,8,9]]`
  - Вихід: `[1,4,7,8,9,6,3,2,5]`.
