# Easy_01_HighestAltitude — Довідка

## Техніка: Running Accumulator / Best-So-Far

**Суть:** один прохід масивом, одна змінна-акумулятор (`current`), яка
накопичує суму зліва направо. Опційно — друга змінна, що зберігає
"найкраще значення на цей момент" (`max`/`min` so far), яка оновлюється
на кожному кроці порівнянням із `current`.

```java
int current = 0, best = 0;
for (int x : arr) {
    current += x;
    best = Math.max(best, current); // або Math.min
}
```

**Коли застосовувати:** будь-яке "яке максимальне/мінімальне значення
досягається під час проходу" — баланс рахунку, рівень води, ціна акції,
префіксна сума. Ключова ознака завдання: результат залежить від
ІСТОРІЇ проходу, а не тільки від фінального стану.

**Складність:** O(N) час, O(1) пам'ять — не треба зберігати весь масив
проміжних сум, бо `best` залежить лише від попереднього кроку.

## Drills

1. **Easy_01_01** — чиста префіксна сума (без max/min, просто накопичення).
2. **Easy_01_02** — трекінг max-so-far (ізоморфний переказ Easy_01).
3. **Easy_01_03** — трекінг "чи впало нижче нуля" (по суті min-so-far з раннім виходом).
4. **Easy_01_04** — трекінг max І min одночасно в одному проході.
5. **Easy_01_05** — два корельовані акумулятори: min-so-far і best-profit-so-far.

Розв'язуй у порядку 01 → 05, не підглядаючи в Easy_01_HighestAltitude.java.
