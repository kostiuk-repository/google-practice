# Easy_01_HighestAltitude — Довідка

## Суть теми / алгоритму простими словами
Техніка **Running Accumulator (Накопичувальний підсумок)** або **Best-So-Far (Найкраще на цей момент)** полягає в тому, що ми проходимося масивом зліва направо один раз, оновлюючи поточний стан (суму або інший показник) і паралельно порівнюючи його з глобальним найкращим (максимальним/мінімальним) значенням, знайденим до цього часу.

Базова задача: знайти найвищу точку (акумульовану суму), яку досягає мандрівник, якщо він починає з висоти 0, а масив містить відносні зміни висоти (чистий приріст або спад).

## Візуалізація та покроковий розбір (Visualization & Walkthrough)

### a) Звичайний випадок: `gain = [-5, 1, 5, 0, -7]`
Початкова висота (точка 0) дорівнює `0`.
Змінні перед початком циклу: `accumulator = 0`, `highest = 0`.

| Крок (Індекс) | Елемент `gain[i]` | Зміна акумулятора (`accumulator += gain[i]`) | Новий `accumulator` | Оновлення `highest` (`Math.max(highest, accumulator)`) |
| :--- | :--- | :--- | :--- | :--- |
| Старт | — | — | 0 | 0 (початкова точка) |
| 1 (0) | -5 | `0 + (-5)` | -5 | `Math.max(0, -5) = 0` |
| 2 (1) | 1 | `-5 + 1` | -4 | `Math.max(0, -4) = 0` |
| 3 (2) | 5 | `-4 + 5` | 1 | `Math.max(0, 1) = 1` |
| 4 (3) | 0 | `1 + 0` | 1 | `Math.max(1, 1) = 1` |
| 5 (4) | -7 | `1 + (-7)` | -6 | `Math.max(1, -6) = 1` |

**Результат:** Найвища висота = `1`.

### b) Граничний випадок: `gain = [-2, -3, -1]` (лише від'ємні значення)
Змінні перед початком циклу: `accumulator = 0`, `highest = 0`.

| Крок (Індекс) | Елемент `gain[i]` | Новий `accumulator` | Оновлення `highest` |
| :--- | :--- | :--- | :--- |
| Старт | — | 0 | 0 |
| 1 (0) | -2 | -2 | `Math.max(0, -2) = 0` |
| 2 (1) | -3 | -5 | `Math.max(0, -5) = 0` |
| 3 (2) | -1 | -6 | `Math.max(0, -6) = 0` |

**Результат:** Найвища висота = `0` (початкова висота, оскільки всі інші були нижчими).

## Ключова ідея та шаблон коду (Core Concept & Code Template)

Поточна висота в будь-якій точці $i$ є сумою початкової висоти ($0$) та всіх попередніх приростів (`gain[0]` до `gain[i-1]`). Замість того, щоб створювати новий масив для збереження кожної проміжної висоти, ми можемо підтримувати лише одну змінну `accumulator`, що відображає поточну висоту на поточному кроці. Інша змінна, `highest`, зберігає максимальне значення серед усіх обчислених значень `accumulator` на даний момент.

### Базовий шаблон коду (Java)
```java
public int largestAltitude(int[] gain) {
    // Акумулятор для накопичення поточної висоти
    int accumulator = 0;
    
    // Зберігає максимальну висоту, досягнуту на даний момент (починаємо з 0)
    int highest = 0;
    
    // Проходимо по кожному приросту висоти в масиві
    for (int value : gain) {
        // Додаємо поточний приріст до акумулятора
        accumulator += value;
        
        // Оновлюємо максимум, якщо поточна висота більша за попередній рекорд
        highest = Math.max(highest, accumulator);
    }
    
    // Повертаємо знайдений максимум
    return highest;
}
```

### Аналіз складності (Complexity Analysis)
- **Time Complexity (Часова складність):** $O(N)$, де $N$ — довжина масиву `gain`. Ми здійснюємо рівно один прохід по масиву.
- **Space Complexity (Просторова складність):** $O(1)$, оскільки ми використовуємо лише дві змінні (`accumulator` та `highest`) незалежно від розміру вхідного масиву.

## Граничні випадки та підводні камені (Edge Cases & Common Pitfalls)

1. **Не врахування стартової висоти (індекс 0 зі значенням 0):** Новачки часто ініціалізують `highest = gain[0]`. Це помилка, тому що якщо всі прирости від'ємні (наприклад, `[-5, -4, -3]`), то найвища точка — це початкова точка `0`, а не `-5`.
2. **Переповнення типів:** У задачах, де суми можуть виходити за межі `Integer.MAX_VALUE` або `Integer.MIN_VALUE`, потрібно використовувати `long`. Але для цієї конкретної задачі обмеження зазвичай вписуються в `int`.
3. **Порожній масив:** Якщо за умовами задачі масив може бути порожнім, слід уточнити очікувану поведінку (зазвичай повертається 0).
