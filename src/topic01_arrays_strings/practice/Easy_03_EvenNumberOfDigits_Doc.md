# Easy_03_EvenNumberOfDigits — Довідка

## Суть теми / алгоритму простими словами
Техніка **Digit Counting (Підрахунок кількості цифр у числі)** є фундаментальною математичною операцією у програмуванні. Щоб дізнатися довжину запису числа (кількість його розрядів), ми можемо або поділити його послідовно на 10, або скористатися математичним логарифмом, або перетворити число на рядок і виміряти його довжину.
В рамках базової задачі ми поєднуємо цей підрахунок із простим обходом масиву (лінійним пошуком або агрегацією), підраховуючи, скільки чисел мають парну кількість цифр.

Вкладений `while` не робить алгоритм квадратичним: для `int` він виконує не більш як 10 ітерацій на число. Формально час — $O(N \cdot D)$, де $D$ — число десяткових розрядів.

### a) Звичайний випадок: `nums = [12, 345, 2, 6, 7896]`
Початкове значення лічильника результату `evenDigitCount = 0`.

| Елемент `nums[i]` | Процес підрахунку цифр (ділення на 10) | Кількість цифр | Парність (Кількість % 2 == 0) | Зміна лічильника `evenDigitCount` |
| :--- | :--- | :--- | :--- | :--- |
| **12** | `12 -> 1 -> 0` (2 ітерації) | 2 | `2 % 2 == 0` (Парне) | `0 + 1 = 1` |
| **345** | `345 -> 34 -> 3 -> 0` (3 ітерації) | 3 | `3 % 2 == 0` (Непарне) | `1` |
| **2** | `2 -> 0` (1 ітерація) | 1 | `1 % 2 == 0` (Непарне) | `1` |
| **6** | `6 -> 0` (1 ітерація) | 1 | `1 % 2 == 0` (Непарне) | `1` |
| **7896** | `7896 -> 789 -> 78 -> 7 -> 0` (4 ітерації) | 4 | `4 % 2 == 0` (Парне) | `1 + 1 = 2` |

**Результат:** `2`.

### b) Граничний випадок: `nums = [0, 9, 99]` (включаючи нуль та границі розрядів)
Початкове значення лічильника результату `evenDigitCount = 0`.

| Елемент `nums[i]` | Кількість цифр | Парність | Оновлення `evenDigitCount` |
| :--- | :--- | :--- | :--- |
| **0** | `1` (нуль має 1 цифру) | Непарне | `0` |
| **9** | `1` | Непарне | `0` |
| **99** | `2` | Парне | `1` |

**Результат:** `1`.

## Ключова ідея та шаблон коду (Core Concept & Code Template)

Існує три основні способи порахувати кількість цифр у числі $N$:
1. **Перетворення на рядок:** `String.valueOf(N).length()`. Дуже просто писати, але неефективно за пам'яттю, бо створює новий об'єкт `String` для кожного числа в купі (heap).
2. **Логарифм (математичний):** `(int) Math.log10(N) + 1` (для $N > 0$). Працює дуже швидко, але вимагає додаткових перевірок для $N \le 0$ та роботи з дробовими числами (floating point), що може викликати проблеми через точність представлення даних.
3. **Послідовне ділення (цикл `while`):** Ділимо число на 10 на кожній ітерації, поки воно не стане `0`. Цей підхід є найбільш надійним та працює без виділення додаткової пам'яті.

### Базовий шаблон коду (Java)
```java
public int findNumbers(int[] nums) {
    int evenDigitCount = 0;
    for (int num : nums) {
        if (hasEvenNumberOfDigits(num)) {
            evenDigitCount++;
        }
    }
    return evenDigitCount;
}

// Допоміжний метод для визначення парності кількості цифр числа
private boolean hasEvenNumberOfDigits(int num) {
    // Якщо число від'ємне, робимо його додатним
    if (num < 0) {
        num = -num;
    }
    
    // Особливий випадок для нуля
    if (num == 0) {
        return false; // Кількість цифр у 0 дорівнює 1 (непарне)
    }
    
    int count = 0;
    while (num > 0) {
        num /= 10;
        count++;
    }
    
    return count % 2 == 0;
}
```

### Аналіз складності (Complexity Analysis)
- **Time Complexity (Часова складність):** $O(N \cdot D)$, де $N$ — розмір масиву `nums`, а $D$ — середня кількість цифр у числах (для 32-бітних `int` $D \le 10$). Отже, часова складність є фактично лінійною — $O(N)$.
- **Space Complexity (Просторова складність):** $O(1)$, оскільки ми використовуємо лише константні змінні для підрахунку цифр і не виділяємо нових структур даних.

## Граничні випадки та підводні камені (Edge Cases & Common Pitfalls)

1. **Неправильна обробка нуля (`0`):** Якщо написати цикл `while (num > 0)`, то для числа `0` цикл не виконається жодного разу, і метод поверне `0` цифр. Проте число `0` має `1` цифру, що є непарною кількістю.
2. **Від'ємні числа:** Якщо вхідний масив може містити від'ємні числа, операція `num /= 10` для від'ємного числа буде коректно зменшувати його за модулем, але перевірка у циклі `while (num > 0)` зламається. Потрібно взяти модуль числа: `num = Math.abs(num)`.
3. **Переповнення при використанні `Math.abs`:** Пам'ятайте, що `Math.abs(Integer.MIN_VALUE)` призводить до переповнення типу `int` і повертає від'ємне число. Якщо такі значення можливі, краще перетворити число в `long` або обробити цей випадок окремо.
