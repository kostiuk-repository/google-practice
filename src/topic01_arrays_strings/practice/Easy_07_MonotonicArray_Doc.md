# Easy_07_MonotonicArray — Довідка

## Суть теми / алгоритму простими словами
Техніка **Single Pass with Boolean Flags (Один прохід з булевими прапорцями)** полягає в перевірці певної властивості послідовності за один цикл. Ми створюємо логічні прапорці-припущення (наприклад, "масив монотонно зростає" та "масив монотонно спадає"), які спочатку встановлюємо в значення `true`. Проходячи масивом та порівнюючи сусідні елементи, мы скидаємо відповідний прапорець у `false`, якщо зустрічаємо порушення умови. Якщо на певному кроці обидва прапорці стали `false`, ми можемо зробити **ранній вихід (early exit)**, оскільки масив точно не є монотонним.

Базова задача: визначити, чи є масив монотонним (тобто або тільки не спадає, або тільки не зростає).

## Візуалізація та покроковий розбір (Visualization & Walkthrough)

### a) Звичайний випадок: `nums = [1, 2, 2, 3]` (монотонно зростає)
Початковий стан: `isIncreasing = true`, `isDecreasing = true`.
Йдемо по парах `nums[i]` та `nums[i+1]`.

| Індекс `i` | Пара `(nums[i], nums[i+1])` | Умова зростання (`nums[i] <= nums[i+1]`) | Умова спадання (`nums[i] >= nums[i+1]`) | Новий стан `isIncreasing` | Новий стан `isDecreasing` | Дія / Ранній вихід? |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **Старт** | — | — | — | `true` | `true` | — |
| 0 | `(1, 2)` | `true` | `false` | `true` | `false` | Продовжуємо |
| 1 | `(2, 2)` | `true` | `true` | `true` | `false` | Продовжуємо |
| 2 | `(2, 3)` | `true` | `false` | `true` | `false` | Продовжуємо |

**Результат:** `isIncreasing || isDecreasing` -> `true || false` -> `true`.

### b) Звичайний випадок з раннім виходом: `nums = [1, 3, 2, 4]` (немонотонний)
Початковий стан: `isIncreasing = true`, `isDecreasing = true`.

| Індекс `i` | Пара `(nums[i], nums[i+1])` | Новий `isIncreasing` | Новий `isDecreasing` | Дія / Ранній вихід? |
| :--- | :--- | :--- | :--- | :--- |
| **Старт** | — | `true` | `true` | — |
| 0 | `(1, 3)` | `true` | `false` (порушення спадання) | Продовжуємо |
| 1 | `(3, 2)` | `false` (порушення зростання) | `false` | **Early exit** (`!isIncreasing && !isDecreasing`) |

**Результат:** `false`.

## Ключова ідея та шаблон коду (Core Concept & Code Template)

Масив монотонний, якщо він не спадає ($A[i] \le A[i+1]$) або не зростає ($A[i] \ge A[i+1]$) для всіх сусідніх елементів. 
Замість того, щоб спочатку повністю перевіряти одну умову, а потім іншу (що вимагало б двох проходів), ми перевіряємо обидва тренди паралельно за один прохід. Якщо `nums[i] > nums[i+1]`, масив не може бути зростаючим (`isIncreasing = false`). Якщо `nums[i] < nums[i+1]`, масив не може бути спадаючим (`isDecreasing = false`). Якщо обидва прапорці стали `false`, ми негайно перериваємо цикл.

### Базовий шаблон коду (Java)
```java
public boolean isMonotonic(int[] nums) {
    // Припускаємо, що масив може бути і зростаючим, і спадаючим
    boolean isIncreasing = true;
    boolean isDecreasing = true;
    
    // Проходимо по сусідніх парах елементів
    for (int i = 0; i < nums.length - 1; i++) {
        // Якщо поточний елемент більший за наступний, тренд зростання порушено
        if (nums[i] > nums[i + 1]) {
            isIncreasing = false;
        }
        // Якщо поточний елемент менший за наступний, тренд спадання порушено
        if (nums[i] < nums[i + 1]) {
            isDecreasing = false;
        }
        
        // Ранній вихід: якщо обидва тренди неможливі, повертаємо false
        if (!isIncreasing && !isDecreasing) {
            return false;
        }
    }
    
    // Масив монотонний, якщо зберігся хоча б один тренд
    return isIncreasing || isDecreasing;
}
```

### Аналіз складності (Complexity Analysis)
- **Time Complexity (Часова складність):** $O(N)$, де $N$ — довжина масиву `nums`. У найгіршому випадку ми робимо один прохід. Якщо масив немонотонний посередині, алгоритм завершиться швидше завдяки ранньому виходу.
- **Space Complexity (Просторова складність):** $O(1)$, оскільки використовуються лише два логічні прапорці.

## Граничні випадки та підводні камені (Edge Cases & Common Pitfalls)

1. **Масиви з 1 або 2 елементів:** Масив з одного елемента завжди монотонний за визначенням. Будь-який масив з двох елементів також завжди монотонний (він або зростає, або спадає, або елементи рівні). Наш код коректно обробляє це: при `nums.length <= 2` цикл виконається щонайбільше один раз і відразу поверне `true`.
2. **Вихід за межі масиву (Off-by-one error):** Оскільки ми порівнюємо сусідні елементи `nums[i]` та `nums[i+1]`, цикл повинен йти до `nums.length - 1`. Якщо піти до `nums.length`, виникне помилка `ArrayIndexOutOfBoundsException`.
3. **Ігнорування rooftops (рівних сусідів):** Рівність сусідніх елементів (`nums[i] == nums[i+1]`) не порушує нестрогу монотонність. Тому умовою скидання прапорців мають бути суворі нерівності (`>` та `<`).
