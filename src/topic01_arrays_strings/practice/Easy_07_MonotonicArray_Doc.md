# Easy_07_MonotonicArray — Довідка

## Техніка: Single Pass with Boolean Flags

**Суть:** один прохід масивом, одна або дві булеві змінні-прапорці
(`isIncreasing`, `isDecreasing`), що стартують як `true` і опускаються
до `false`, щойно на сусідній парі елементів знаходиться порушення
відповідного напрямку. Якщо обидва прапорці стали `false` — можна
одразу вийти з циклу (early exit), бо результат вже відомий.

```java
boolean isIncreasing = true, isDecreasing = true;
for (int i = 0; i < nums.length - 1; i++) {
    if (nums[i] > nums[i + 1]) isIncreasing = false;
    if (nums[i] < nums[i + 1]) isDecreasing = false;
    if (!isIncreasing && !isDecreasing) break; // early exit
}
boolean result = isIncreasing || isDecreasing;
```

**Коли застосовувати:** будь-яка перевірка "чи послідовність не спадає /
не зростає на всьому проміжку" — монотонність масиву, перевірка
відсортованості, пошук порушення тренду. Ключова ознака: рішення
залежить лише від порівняння КОЖНОЇ сусідньої пари, без потреби
дивитись далі за межі i та i+1.

**Складність:** O(N) час (у гіршому випадку; менше при early exit),
O(1) додаткової пам'яті — потрібні лише прапорці, не весь масив.

## Drills

1. **Easy_07_01** — строга монотонність (строгі нерівності, без рівних сусідніх елементів).
2. **Easy_07_02** — лише перевірка зростання (один прапорець `isIncreasing`).
3. **Easy_07_03** — лише перевірка спадання (один прапорець `isDecreasing`, дзеркально до 02).
4. **Easy_07_04** — довжина найдовшого монотонного префікса (прапорець перетворюється на лічильник).
5. **Easy_07_05** — та сама техніка, але застосована до похідного масиву довжин рядків.

Розв'язуй у порядку 01 → 05, не підглядаючи в Easy_07_MonotonicArray.java.
