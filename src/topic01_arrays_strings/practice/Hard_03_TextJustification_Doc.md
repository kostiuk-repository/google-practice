# Hard_03_TextJustification — Довідка

## Техніка: Greedy Line Packing + Even Space Distribution

**Суть:** алгоритм має два незалежні кроки. Крок 1 (пакування) —
жадібно додавати слова в поточний рядок, поки наступне слово ще
влазить у `maxWidth` разом з мінімальними пробілами між уже доданими
словами; щойно наступне слово не влазить — рядок "закривається" і
починається новий. Крок 2 (розподіл пробілів) — маючи фінальний набір
слів рядка, залишкову ширину (`maxWidth` мінус сумарна довжина слів)
потрібно розподілити як пробіли між словами: ділимо порівну на
кількість проміжків, а залишок від ділення розкидаємо по проміжках
зліва направо (лівіші проміжки отримують на один пробіл більше).

```java
// Крок 1: жадібне пакування
int lineLen = 0, start = i;
while (i < words.length && lineLen + words[i].length() + (i - start) <= maxWidth) {
    lineLen += words[i].length();
    i++;
}
// Крок 2: розподіл пробілів (не остання лінія, > 1 слова)
int gaps = i - start - 1;
int totalSpaces = maxWidth - lineLen;
int spacesPerGap = totalSpaces / gaps;
int extra = totalSpaces % gaps; // перші `extra` проміжків отримують +1 пробіл
```

**Коли застосовувати:** будь-яке форматування тексту у рядки фіксованої
ширини — text justification, word wrap з вирівнюванням, генерація
табличного/консольного виводу з рівномірними відступами.

**Складність:** O(N) час, де N — сумарна довжина всіх слів (кожен
символ обробляється O(1) разів при пакуванні й копіюванні в результат),
O(N) додаткової пам'яті на сам результат (масив рядків).

## Drills

1. **Hard_03_01** — жадібне пакування без вирівнювання: один пробіл + trailing padding.
2. **Hard_03_02** — те саме пакування, але padding іде як leading spaces (right-justify).
3. **Hard_03_03** — ізольований крок 2: розподіл пробілів для вже готового рядка слів.
4. **Hard_03_04** — leftover ширина ділиться порівну на ліву/праву частину (center-justify).
5. **Hard_03_05** — крок 1 змінюється: мінімум 2 пробіли на проміжок замість 1.

Розв'язуй у порядку 01 → 05, не підглядаючи в Hard_03_TextJustification.java.
