# Text Justification — Довідка

## Суть теми / алгоритму простими словами
Форматування тексту за шириною (**Text Justification**) — це класична задача на роботу з рядками, яка моделює поведінку текстових процесорів (наприклад, MS Word або LaTeX). 

Алгоритм виконується за два основних кроки:
1. **Жадібне пакування (Greedy Packing):** Ми намагаємося помістити якомога більше слів у поточний рядок. Кожне слово відокремлюється від попереднього щонайменше одним пробілом. Якщо наступне слово не вміщується в ліміт `maxWidth`, ми «закриваємо» поточний рядок і переходимо до кроку вирівнювання.
2. **Розподіл пробілів (Space Distribution):** Ми вираховуємо загальну кількість пробілів, необхідних для заповнення рядка до `maxWidth`. Ці пробіли розподіляються між словами якомога рівномірніше. Якщо пробіли не діляться націло на кількість проміжків, залишок розподіляється по одному додатковому пробілу в проміжки зліва направо.

## Інженерний контекст: layout як двофазний pipeline

Text Justification поєднує дві різні задачі:

1. **line breaking:** вибрати, які слова належать рядку;
2. **rendering:** перетворити вибрану групу на output точної ширини.

Розділення фаз зменшує кількість одночасних інваріантів і робить edge cases локальними.

### Де зустрічається

- text layout у редакторах, terminals і report generators;
- formatting fixed-width records;
- table/column rendering у CLI;
- pagination і line wrapping;
- greedy packing об’єктів із separator cost.

LeetCode використовує просту metric `word.length()` і забороняє hyphenation. Реальні layout engines складніші: Unicode code points не дорівнюють display columns, існують combining marks, proportional fonts, kerning, bidirectional text і language-specific line breaking.

### Чому greedy тут коректний

Контракт прямо вимагає поміщати максимально можливу кількість слів у рядок. Тому локальне рішення «додавай наступне слово, поки воно вміщується» не конкурує з глобальною typography quality metric. В інших задачах line breaking greedy може бути неоптимальним і застосовують dynamic programming.

### Що перевіряє співбесіда

- чи включите мінімальні пробіли під час packing;
- чи окремо обробите один-word і last line;
- чи правильно розподілите quotient/remainder між gaps;
- чи гарантуєте `line.length() == maxWidth`;
- чи уникнете quadratic string concatenation;
- чи зможете пояснити, чому greedy випливає саме з контракту цієї задачі.

### a) Звичайний випадок: перший та другий рядки

#### **Рядок 1:**
- Починаємо з індексу `i = 0` ("This").
- Перевіряємо слова:
  - "This" (довжина 4) — вміщується.
  - "is" (4 + 1 пробіл + 2 = 7) — вміщується.
  - "an" (7 + 1 пробіл + 2 = 10) — вміщується.
  - "example" (10 + 1 пробіл + 7 = 18 > 16) — не вміщується.
- Слова в першому рядку: `["This", "is", "an"]` (індекси 0, 1, 2).
- **Розподіл пробілів:**
  - Сумарна довжина літер: $4 + 2 + 2 = 8$.
  - Загальна кількість пробілів: $16 - 8 = 8$.
  - Кількість проміжків між словами (gaps): $3 - 1 = 2$.
  - Базові пробіли на проміжок: $8 / 2 = 4$.
  - Залишок (extra): $8 \% 2 = 0$.
  - Обидва проміжки отримують по 4 пробіли: `"This    is    an"`.

#### **Рядок 2:**
- Продовжуємо з індексу `i = 3` ("example").
- Перевіряємо слова:
  - "example" (довжина 7) — вміщується.
  - "of" (7 + 1 пробіл + 2 = 10) — вміщується.
  - "text" (10 + 1 пробіл + 4 = 15) — вміщується.
  - "justification." (15 + 1 пробіл + 14 = 30 > 16) — не вміщується.
- Слова в другому рядку: `["example", "of", "text"]` (індекси 3, 4, 5).
- **Розподіл пробілів:**
  - Сумарна довжина літер: $7 + 2 + 4 = 13$.
  - Загальна кількість пробілів: $16 - 13 = 3$.
  - Кількість проміжків (gaps): $3 - 1 = 2$.
  - Базові пробіли на проміжок: $3 / 2 = 1$.
  - Залишок (extra): $3 \% 2 = 1$.
  - Перший проміжок отримує $1 + 1 = 2$ пробіли, другий — $1$ пробіл: `"example  of text"`.

---

### b) Граничні випадки (останній рядок та рядок з одного слова)

#### **Останній рядок:**
- Продовжуємо з індексу `i = 6` ("justification.").
- Більше слів немає. Цей рядок є останнім.
- За правилом, останній рядок вирівнюється **по лівому краю**: слова розділяються рівно одним пробілом, а всі інші пробіли додаються в кінець.
- Рядок: `"justification."` (довжина 14). Додаємо 2 пробіли в кінець.
- Результат: `"justification.  "`.

#### **Рядок з одного слова (наприклад, слово довжиною 14, `maxWidth = 16`):**
- Тільки одне слово: `["justification."]`.
- Кількість проміжків (gaps) = 0. Ділити на 0 не можна!
- За правилом, такий рядок вирівнюється **по лівому краю**: слово пишеться на початку, а решта простору заповнюється пробілами праворуч.
- Результат: `"justification.  "`.

## Ключова ідея та шаблон коду (Core Concept & Code Template)

Алгоритм вимагає чіткої математичної моделі для розподілу пробілів. Якщо ми маємо рядок зі словами від `start` до `end - 1`:
1. `gaps = (end - start) - 1`
2. Якщо `gaps == 0` або це остання лінія (`end == words.length`), вирівнюємо вліво: з'єднуємо слова одним пробілом і додаємо пробіли справа до `maxWidth`.
3. Інакше виконуємо повне вирівнювання:
   - `totalSpaces = maxWidth - sumOfWordLengths`
   - `spacesPerGap = totalSpaces / gaps`
   - `extraSpaces = totalSpaces % gaps`
   - Для кожного проміжку `j` від 0 до `gaps - 1`: додаємо `spacesPerGap` пробілів, а якщо `j < extraSpaces`, додаємо ще 1 додатковий пробіл.

### Базовий шаблон коду (Java)

```java
package topic01_arrays_strings.practice;

import java.util.ArrayList;
import java.util.List;

public class Hard_03_TextJustification {

    public List<String> fullJustify(String[] words, int maxWidth) {
        List<String> result = new ArrayList<>();
        int i = 0;
        int n = words.length;
        
        while (i < n) {
            int start = i;
            int lineWordsLength = words[i].length();
            i++;
            
            // Жадібно визначаємо, скільки слів вміститься в один рядок
            while (i < n && lineWordsLength + words[i].length() + (i - start) <= maxWidth) {
                lineWordsLength += words[i].length();
                i++;
            }
            
            int end = i;
            int gaps = (end - start) - 1;
            StringBuilder sb = new StringBuilder();
            
            // Випадок 1: Останній рядок або рядок з одного слова (вирівнювання вліво)
            if (end == n || gaps == 0) {
                for (int k = start; k < end; k++) {
                    sb.append(words[k]);
                    if (k < end - 1) {
                        sb.append(" ");
                    }
                }
                // Заповнюємо залишок пробілами з правого боку
                while (sb.length() < maxWidth) {
                    sb.append(" ");
                }
            } 
            // Випадок 2: Звичайний рядок (повне вирівнювання)
            else {
                int totalSpaces = maxWidth - lineWordsLength;
                int spacesPerGap = totalSpaces / gaps;
                int extraSpaces = totalSpaces % gaps;
                
                for (int k = start; k < end - 1; k++) {
                    sb.append(words[k]);
                    
                    // Додаємо базову кількість пробілів
                    int currentGapSpaces = spacesPerGap + ( (k - start) < extraSpaces ? 1 : 0 );
                    for (int s = 0; s < currentGapSpaces; s++) {
                        sb.append(" ");
                    }
                }
                // Додаємо останнє слово рядка (після нього пробіли не потрібні)
                sb.append(words[end - 1]);
            }
            
            result.add(sb.toString());
        }
        
        return result;
    }
}
```

### Аналіз складності
- **Часова складність (Time Complexity):** $O(N)$, де $N$ — загальна кількість символів у всіх словах. Ми проходимо по кожному слову один раз для пакування і один раз копіюємо його символи при побудові рядків.
- **Просторова складність (Space Complexity):** $O(N)$ для збереження відформатованого результату. Додаткова пам'ять (якщо не враховувати список результатів) становить $O(\text{maxWidth})$ на тимчасовий `StringBuilder` для поточного рядка.

## Граничні випадки та підводні камені (Edge Cases & Common Pitfalls)

1. **Ділення на нуль (Division by zero):**
   Якщо в рядку лише одне слово, кількість проміжків `gaps = 0`. Формули `totalSpaces / gaps` та `totalSpaces % gaps` призведуть до `ArithmeticException`. Цей випадок потрібно обробляти окремо (вирівнюванням вліво).

2. **Останній рядок:**
   Початківці часто забувають, що останній рядок має вирівнюватися виключно по лівому краю, навіть якщо там багато слів і багато проміжків.

3. **Слова з максимальною довжиною:**
   Якщо довжина слова дорівнює `maxWidth`, воно займає весь рядок. Код повинен коректно обробляти `gaps = 0` і не додавати зайвих пробілів.

4. **Правильний підрахунок довжини при пакуванні:**
   При перевірці умови `lineWordsLength + words[i].length() + (i - start) <= maxWidth` обов'язково враховуйте мінімальний один пробіл на кожен проміжок `(i - start)`.

## Розбір Drills (Drills Breakdown)

### 1. Hard_03_01 — Left Justify Only
- **Мета:** Спростити алгоритм вирівнювання: абсолютно всі рядки вирівнюються по лівому краю.
- **Відмінність від основної:** Немає складного розподілу пробілів. Кожне слово відокремлюється від наступного одним пробілом, а решта ширини заповнюється пробілами в кінці рядка.
- **Підказка:** Жадібно пакуйте слова так само, як в оригіналі. Будуйте рядок через `String.join(" ", lineWords)` і дописуйте пробіли до досягнення `maxWidth`.
- **Приклад:** `words = ["This", "is", "an", "example"]`, `maxWidth = 12`.
  - Рядок 1: `["This", "is", "an"]` -> `"This is an  "`
  - Рядок 2: `["example"]` -> `"example     "`

### 2. Hard_03_02 — Right Justify
- **Мета:** Кожен рядок має бути вирівняний по правому краю.
- **Відмінність від основної:** Залишок ширини додається як пробіли **попереду** слів (leading spaces), а слова відокремлюються рівно одним пробілом.
- **Підказка:** Сформуйте рядок з одинарними проміжками між словами. Нехай його довжина $L$. Додайте `maxWidth - L` пробілів на початку цього рядка.
- **Приклад:** `words = ["This", "is", "an", "example"]`, `maxWidth = 12`.
  - Рядок 1: `"This is an"` ($L = 10$). Вихід: `"  This is an"`
  - Рядок 2: `"example"` ($L = 7$). Вихід: `"     example"`

### 3. Hard_03_03 — Distribute Spaces Given Line
- **Мета:** Ізолювати та реалізувати суто математичну частину рівномірного розподілу пробілів для заданого списку слів (без кроку пакування).
- **Відмінність від основної:** На вхід подається готовий список слів для одного рядка. Гарантується, що слів більше одного.
- **Підказка:** Застосуйте формулу розподілу залишку пробілів: `totalSpaces = maxWidth - sumOfWordsLength`. Поділіть їх на `gaps = words.size() - 1` і додайте залишок `extra` в ліві проміжки.
- **Приклад:** `words = ["This", "is", "an"]`, `maxWidth = 16`.
  - Літери: 8. Пробіли: 8. Проміжків: 2. Базово 4 пробіли на проміжок. Вихід: `"This    is    an"`.

### 4. Hard_03_04 — Center Justify
- **Мета:** Вирівняти текст по центру.
- **Відмінність від основної:** Залишок ширини ділиться порівну на лівий та правий відступи. Якщо залишок непарний, додатковий пробіл додається праворуч.
- **Підказка:** Спочатку об'єднайте слова з одним пробілом між ними (довжина $L$). Обчисліть залишок `diff = maxWidth - L`. Зліва додайте `diff / 2` пробілів, а справа — `diff - (diff / 2)` пробілів.
- **Приклад:** `words = ["This", "is", "an"]`, `maxWidth = 13`.
  - З'єднання: `"This is an"` ($L=10$). Залишок: 3.
  - Лівий відступ: 1 пробіл. Правий відступ: 2 пробіли.
  - Вихід: `" This is an  "`.

### 5. Hard_03_05 — Minimum Two Spaces Between Words
- **Мета:** Повне вирівнювання за шириною, але між будь-якими двома сусідніми словами має бути мінімум 2 пробіли.
- **Відмінність від основної:** Змінюється логіка перевірки при пакуванні слів.
- **Підказка:** Коли перевіряєте, чи поміститься наступне слово, закладайте мінімум 2 пробіли на кожен проміжoк: `lineWordsLength + nextWordLength + (gaps + 1) * 2 <= maxWidth`. Сам розподіл пробілів робиться аналогічно до основної задачі (діленням та остачею), оскільки мінімальна вимога автоматично задовольниться. Для останнього рядка та рядків з одного слова застосовуються стандартні правила (1 пробіл / вирівнювання вліво).
- **Приклад:** `words = ["a", "bb", "ccc"]`, `maxWidth = 10`.
  - Довжина слів: 1 ("a") + 2 ("bb") + 3 ("ccc") = 6. Кількість проміжків: 2. Мінімальна кількість пробілів: $2 \times 2 = 4$. Разом: $6 + 4 = 10 \le 10$. Всі три слова вміщуються в одну лінію.
  - Вихід: `"a  bb  ccc"`.
