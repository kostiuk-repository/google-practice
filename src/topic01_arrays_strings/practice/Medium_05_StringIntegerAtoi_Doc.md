# Medium_05_StringIntegerAtoi — Довідка

## Суть теми / алгоритму простими словами
Задача "Перетворення рядка на ціле число (atoi)" (String to Integer) полягає в симуляції процесу парсингу текстового представлення числа та конвертації його в 32-бітне ціле число зі знаком.
Це класична задача на роботу зі станами (State Machine / Parser State) та обробку виняткових ситуацій (переповнення типу даних).

Алгоритм має виконати наступні кроки послідовно:
1. **Пропуск пробілів**: ігнорувати всі початкові пробіли.
2. **Визначення знаку**: прочитати символ `+` або `-`, якщо він є.
3. **Парсинг цифр**: читати символи цифр (`'0'-'9'`) підряд, доки не зустрінеться перший нецифровий символ або кінець рядка.
4. **Запобігання переповненню (Overflow Guard)**: якщо число перевищує межі 32-бітного знакового цілого $\left[-2^{31}, 2^{31} - 1\right]$, його необхідно обмежити (clamp) до `Integer.MIN_VALUE` ($-2147483648$) або `Integer.MAX_VALUE` ($2147483647$).

Перевірку overflow робіть **до** `value = value * 10 + digit`: після переповнення Java вже втратить правильне математичне значення.

### a) Звичайний випадок: `s = "   -42"`

1. **Пропуск пробілів**:
   - Індекси 0, 1, 2 містять `' '`.
   - `i` збільшується до `3`.
2. **Визначення знаку**:
   - Символ `s.charAt(3)` дорівнює `'-'`.
   - `sign = -1`, збільшуємо `i` до `4`.
3. **Парсинг цифр**:
   - `i = 4`: символ `'4'`. `digit = 4`. 
     - Перевірка переповнення: `result = 0`, що менше ніж `Integer.MAX_VALUE / 10`.
     - Оновлюємо: `result = result * 10 + 4 = 4`. Збільшуємо `i` до `5`.
   - `i = 5`: символ `'2'`. `digit = 2`.
     - Перевірка переповнення: `result = 4`, що менше ніж `Integer.MAX_VALUE / 10`.
     - Оновлюємо: `result = result * 10 + 2 = 42`. Збільшуємо `i` до `6`.
   - `i = 6`: досягнуто кінця рядка.
4. **Застосування знаку та повернення результату**:
   - `finalResult = sign * result = -1 * 42 = -42`.

---

### b) Граничний випадок: Переповнення `s = "2147483648"`
`Integer.MAX_VALUE = 2147483647` (остання цифра `7`).
Ми парсимо перші 9 цифр і отримуємо `result = 214748364`.
Наступний символ — `'8'`, тобто `digit = 8`.

Перед тим, як зробити крок `result * 10 + digit`, виконуємо перевірку:
- Чи `result > Integer.MAX_VALUE / 10` ($214748364$)? Ні, вони рівні.
- Оскільки вони рівні, перевіряємо чи поточна цифра більша за останню цифру `Integer.MAX_VALUE % 10` ($7$).
- `digit (8) > 7`, отже, при наступному кроці відбудеться переповнення!
- Повертаємо граничне значення залежно від знаку. Оскільки `sign = 1`, повертаємо `Integer.MAX_VALUE` ($2147483647$).

## Ключова ідея та шаблон коду

Головна концепція — **Overflow Guard**. Ми не можемо просто накопичувати число в `int` чи `long`, а потім перевіряти, бо:
1. `int` переповниться безшумно (overflow wrap-around) і ми отримаємо неправильне число.
2. Використання `long` допомагає для 32-бітного `int`, але не є універсальним розв'язком, якщо на вхід подадуть число, яке перевищує навіть ліміти `long` (наприклад, 100 цифр підряд).

Тому перевірка переповнення робиться **завжди на кроці перед додаванням нової цифри**:
```java
if (result > Integer.MAX_VALUE / 10 || (result == Integer.MAX_VALUE / 10 && digit > Integer.MAX_VALUE % 10)) {
    return sign == 1 ? Integer.MAX_VALUE : Integer.MIN_VALUE;
}
```

### Java-шаблон
Доступ до файлу: [Medium_05_StringIntegerAtoi.java](file:///C:/Users/kosti/Projects/google-practice/src/topic01_arrays_strings/practice/Medium_05_StringIntegerAtoi.java)

```java
public int myAtoi(String s) {
    if (s == null || s.isEmpty()) {
        return 0;
    }
    
    int n = s.length();
    int i = 0;
    
    // 1. Пропускаємо пробіли
    while (i < n && s.charAt(i) == ' ') {
        i++;
    }
    
    if (i == n) {
        return 0;
    }
    
    // 2. Зчитуємо знак
    int sign = 1;
    char firstChar = s.charAt(i);
    if (firstChar == '+') {
        i++;
    } else if (firstChar == '-') {
        sign = -1;
        i++;
    }
    
    // 3. Зчитуємо цифри та перевіряємо переповнення
    int result = 0;
    while (i < n) {
        char ch = s.charAt(i);
        if (ch < '0' || ch > '9') {
            break; // перший нецифровий символ зупиняє парсинг
        }
        
        int digit = ch - '0';
        
        // Overflow Guard
        if (result > Integer.MAX_VALUE / 10 || 
           (result == Integer.MAX_VALUE / 10 && digit > Integer.MAX_VALUE % 10)) {
            return sign == 1 ? Integer.MAX_VALUE : Integer.MIN_VALUE;
        }
        
        result = result * 10 + digit;
        i++;
    }
    
    return result * sign;
}
```

### Аналіз складності:
- **Часова складність (Time Complexity):** $\mathcal{O}(N)$, де $N$ — довжина рядка. Ми проходимо по рядку зліва направо максимум один раз.
- **Просторова складність (Space Complexity):** $\mathcal{O}(1)$ додаткової пам'яті, оскільки використовуються лише локальні змінні стану.

## Граничні випадки та підводні камені

1. **Багато знаків підряд**:
   - *Помилка:* Якщо рядок містить `"+-12"`, парсер має зупинитися на `-` після парсингу `+`. Знаком може бути лише ОДИН символ на початку. Наш шаблон зчитає `+`, збільшить `i`, а на наступній ітерації побачить `-` (який не є цифрою) і зупиниться, повернувши `0`. Це правильна поведінка.
2. **Лише пробіли або порожній рядок**:
   - Рядок виду `"    "` має повернути `0`. Цикл пропуску пробілів завершиться на `i == n`, після чого guard clause `if (i == n) return 0;` безпечно поверне `0`.
3. **Числа з великою кількістю провідних нулів**:
   - Наприклад, `"00000000000123"`. 
   - Провідні нулі не збільшують значення `result` (оскільки `0 * 10 + 0 = 0`), тому переповнення не виникає, доки не почнуться значущі цифри. Наш алгоритм працює коректно.
4. **Нецифрові символи перед цифрами**:
   - Наприклад, `"words and 987"`. Перший символ після пробілів — `'w'`. Алгоритм має відразу зупинитися та повернути `0`. У нашому шаблоні першим символом буде `'w'`, який не є знаковим та не є цифрою, тому цикл не виконається, повернеться `0`.
