# Medium_04_ProductExceptSelf — Довідка

## Техніка: Prefix Sweep + Suffix Sweep

**Суть:** щоб для кожного індексу `i` отримати агрегат (добуток, суму,
максимум тощо) усіх елементів, КРІМ `nums[i]`, не обов'язково на кожному
кроці перебирати всі N-1 інших елементів. Достатньо двох проходів: перший
зліва направо накопичує "префіксний" агрегат усіх елементів ДО `i`, другий
справа наліво накопичує "суфіксний" агрегат усіх елементів ПІСЛЯ `i`.
Результат для кожного `i` — комбінація префіксного і суфіксного значень.
Щоб уникнути зайвого масиву, суфіксний прохід можна виконати як один
`running`-змінна, що домножується/додається напряму у вихідний масив, який
на першому проході вже містить префіксні значення.

```java
int[] result = new int[n];
result[0] = IDENTITY; // 1 для добутку, 0 для суми
for (int i = 1; i < n; i++)
    result[i] = combine(result[i - 1], nums[i - 1]); // префіксний прохід

int suffix = IDENTITY;
for (int i = n - 1; i >= 0; i--) {
    result[i] = combine(result[i], suffix); // об'єднання з суфіксом
    suffix = combine(suffix, nums[i]);
}
```

**Коли застосовувати:** "для кожного елемента порахуй агрегат усіх ІНШИХ
елементів" без операції, що трудно чи неможливо інвертувати (наприклад,
без ділення — бо нуль у масиві ламає добуток/діленя), і з вимогою O(N)
часу без O(N²) перебору пар.

**Складність:** O(N) час (два лінійні проходи), O(N) пам'яті для
префіксного/суфіксного масивів у наївному варіанті, O(1) додаткової
пам'яті у варіанті з одним running-суфіксом (не рахуючи вихідного масиву).

## Drills

1. **Medium_04_01** — лише префіксний масив добутків (result[i] = добуток усіх елементів ДО i).
2. **Medium_04_02** — лише суфіксний масив добутків (result[i] = добуток усіх елементів ПІСЛЯ i).
3. **Medium_04_03** — та сама техніка, але з сумою замість добутку (sum except self) — перевірка узагальнюваності підходу.
4. **Medium_04_04** — оригінальна задача (product except self) у O(1) додаткової пам'яті: префіксний прохід прямо у вихідний масив + один суфіксний прохід із running-змінною.
5. **Medium_04_05** — та сама техніка, перенесена на максимум (max except self) через prefix-max і suffix-max.

Розв'язуй у порядку 01 → 05, не підглядаючи в Medium_04_ProductExceptSelf.java.
