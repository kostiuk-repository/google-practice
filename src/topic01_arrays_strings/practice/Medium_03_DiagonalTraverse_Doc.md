# Medium_03_DiagonalTraverse — Довідка

## Суть теми / алгоритму простими словами
Обхід матриці по діагоналях (Diagonal Traverse) полягає в переборі елементів двовимірного масиву вздовж косих ліній (антидіагоналей), де сума індексів рядка та стовпця є константою ($i + j = \text{const}$). 
У класичній задачі напрямок обходу кожної наступної діагоналі чергується (зигзаг): парні діагоналі обходяться знизу-вгору-вправо, а непарні — зверху-вниз-вліво.

Головна складність тут — коректна зміна напрямку руху при досягненні меж матриці, особливо на прямокутних (не квадратних) матрицях.

Для прямокутної матриці кількість діагоналей завжди `rows + columns - 1`; допустимі координати кожної діагоналі додатково обрізаються межами.

### a) Звичайний випадок: квадратна матриця $3 \times 3$

**Початкова матриця:**
```
1  2  3
4  5  6
7  8  9
```

Кожен елемент належить до діагоналі $d = i + j$. Діагоналі йдуть від $d = 0$ до $d = m + n - 2 = 4$.

| Діагональ ($d$) | Елементи ($i + j = d$) | Напрямок | Порядок обходу | Результат |
| :--- | :--- | :--- | :--- | :--- |
| **$d = 0$** (парна) | `(0,0)` | Знизу-Вгору | `[0][0]` (1) | `[1]` |
| **$d = 1$** (непарна)| `(0,1), (1,0)` | Згори-Вниз | `[0][1]` (2) $\rightarrow$ `[1][0]` (4) | `[1, 2, 4]` |
| **$d = 2$** (парна) | `(0,2), (1,1), (2,0)`| Знизу-Вгору | `[2][0]` (7) $\rightarrow$ `[1][1]` (5) $\rightarrow$ `[0][2]` (3) | `[1, 2, 4, 7, 5, 3]` |
| **$d = 3$** (непарна)| `(1,2), (2,1)` | Згори-Вниз | `[1][2]` (6) $\rightarrow$ `[2][1]` (8) | `[1, 2, 4, 7, 5, 3, 6, 8]` |
| **$d = 4$** (парна) | `(2,2)` | Знизу-Вгору | `[2][2]` (9) | `[1, 2, 4, 7, 5, 3, 6, 8, 9]` |

### b) Граничний випадок: прямокутна матриця $2 \times 4$

**Початкова матриця:**
```
1  2  3  4
5  6  7  8
```
Розміри: $m = 2, n = 4$. Діагоналі від $d = 0$ до $d = 5$.

- **$d = 0$** (парна, знизу-вгору): `[0][0]` (1) $\rightarrow$ `[1]`
- **$d = 1$** (непарна, згори-вниз): `[0][1]` (2) $\rightarrow$ `[1][0]` (5) $\rightarrow$ `[1, 2, 5]`
- **$d = 2$** (парна, знизу-вгору): `[1][1]` (6) $\rightarrow$ `[0][2]` (3) $\rightarrow$ `[1, 2, 5, 6, 3]`
- **$d = 3$** (непарна, згори-вниз): `[0][3]` (4) $\rightarrow$ `[1][2]` (7) $\rightarrow$ `[1, 2, 5, 6, 3, 4, 7]`
- **$d = 4$** (парна, знизу-вгору): `[1][3]` (8). Рядок 0 не містить елемента з $j = 4$, тому обхід закінчується на `8` $\rightarrow$ `[1, 2, 5, 6, 3, 4, 7, 8]`
- **$d = 5$** (непарна): порожня для діапазону індексів $\rightarrow$ завершення.

## Ключова ідея та шаблон коду

Є два способи реалізації:
1. **Симуляція руху з прапорцем напрямку**: Ми підтримуємо поточні координати `(row, col)` та напрямок `dir` (1 для вгору-вправо, -1 для вниз-вліво). При виході за межі ми коригуємо індекси та міняємо напрямок.
2. **Математичний підхід (Групування меж)**:
   Для кожної діагоналі $d \in [0, m + n - 2]$ ми розраховуємо межі індексу рядка $i$:
   - Мінімальний індекс рядка: $i_{\min} = \max(0, d - n + 1)$ (оскільки стовпець $j = d - i \le n - 1 \implies i \ge d - n + 1$).
   - Максимальний індекс рядка: $i_{\max} = \min(d, m - 1)$ (оскільки рядок $i \le m - 1$).
   - Якщо $d$ парне, йдемо по $i$ від $i_{\max}$ вниз до $i_{\min}$.
   - Якщо $d$ непарне, йдемо по $i$ від $i_{\min}$ вгору до $i_{\max}$.
   - Стовпець завжди визначається як $j = d - i$.

Математичний підхід набагато елегантніший, оскільки позбавляє нас від заплутаної логіки `if-else` на кутах матриці та автоматично працює для прямокутних матриць.

### Java-шаблон (Математичний підхід)
Доступ до файлу: [Medium_03_DiagonalTraverse.java](file:///C:/Users/kosti/Projects/google-practice/src/topic01_arrays_strings/practice/Medium_03_DiagonalTraverse.java)

```java
public int[] findDiagonalOrder(int[][] mat) {
    if (mat == null || mat.length == 0 || mat[0].length == 0) {
        return new int[0];
    }
    
    int m = mat.length;
    int n = mat[0].length;
    int[] result = new int[m * n];
    int idx = 0;
    
    // Кількість діагоналей дорівнює m + n - 1
    int totalDiagonals = m + n - 1;
    
    for (int d = 0; d < totalDiagonals; d++) {
        // Розраховуємо математичні межі для рядка i на діагоналі d
        int minRow = Math.max(0, d - n + 1);
        int maxRow = Math.min(d, m - 1);
        
        if (d % 2 == 0) {
            // Парна діагональ: знизу-вгору (індекс рядка зменшується)
            for (int i = maxRow; i >= minRow; i--) {
                result[idx++] = mat[i][d - i];
            }
        } else {
            // Непарна діагональ: зверху-вниз (індекс рядка зростає)
            for (int i = minRow; i <= maxRow; i++) {
                result[idx++] = mat[i][d - i];
            }
        }
    }
    
    return result;
}
```

### Аналіз складності:
- **Часова складність (Time Complexity):** $\mathcal{O}(M \times N)$, де $M$ — рядки, $N$ — стовпці. Кожен елемент відвідується рівно один раз.
- **Просторова складність (Space Complexity):** $\mathcal{O}(1)$ додаткової пам'яті (якщо не враховувати масив результату).

## Граничні випадки та підводні камені

Якщо використовувати **симуляційний підхід** із ручним коригуванням індексів, виникає критично важливий нюанс — **пріоритет перевірки меж на кутах**:

1. **Рух вгору-вправо (`row--, col++`)**:
   - При виході за межі ми можемо одночасно вийти за верхній рядок (`row < 0`) і за правий стовпець (`col >= n`) (наприклад, у верхньому правому куті матриці).
   - **Правило пріоритету:** Перевірка виходу за правий стовпець `if (col >= n)` повинна йти **першою**.
     - Якщо ми вийшли за правий стовпець, ми повинні зсунути рядок вниз на два кроки (`row += 2`) і стовпець вліво на один (`col--` або просто зафіксувати на `n - 1`), і змінити напрямок.
     - Якщо спочатку перевірити `if (row < 0)`, то ми просто збільшимо `col` (що призведе до `col = n`, тобто виходу за межі стовпців) і не опустимося на наступний рядок.

2. **Рух вниз-вліво (`row++, col--`)**:
   - Аналогічно, у лівому нижньому куті ми можемо одночасно вийти за нижній рядок (`row >= m`) та за лівий стовпець (`col < 0`).
   - **Правило пріоритету:** Перевірка виходу за нижній рядок `if (row >= m)` повинна бути **першою**.

Завдяки використанню математичного підходу (як у нашому шаблоні), ці помилки виключаються на рівні дизайну алгоритму.
