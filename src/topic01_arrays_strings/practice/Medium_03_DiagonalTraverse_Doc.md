# Medium_03_DiagonalTraverse — Довідка

## Техніка: Групування по діагоналях (i + j = const) + зміна напрямку на межі

**Суть:** усі елементи `mat[i][j]`, для яких сума індексів `i + j` однакова,
лежать на одній антидіагоналі. Обхід по зигзагу перебирає діагоналі по
порядку зростання `i + j` і на кожній діагоналі змінює напрямок проходу на
протилежний (парні діагоналі — знизу-вгору-вправо, непарні —
згори-вниз-вліво, або навпаки, залежно від конвенції). Найскладніша частина
— коректно обробити межу матриці: коли рух "вгору-вправо" впирається у
верхній рядок (`row < 0`) чи правий стовпець (`col >= n`), потрібно
математично перерахувати новий старт наступної діагоналі, і перевірка
виходу за стовпець має пріоритет над перевіркою виходу за рядок (або
навпаки — залежно від того, яка межа матриці вужча).

```java
for (int d = 0; d <= m + n - 2; d++) {
    // елементи діагоналі d: i + j == d, i в межах [max(0, d-n+1), min(d, m-1)]
    if (d % 2 == 0) {
        // обхід знизу-вгору (row спадає)
    } else {
        // обхід згори-вниз (row зростає)
    }
}
```

**Коли застосовувати:** будь-яка задача "обхід матриці по діагоналях" —
зигзаг-читання, групування елементів по діагоналях, суми/агрегати по
антидіагоналях, а також будь-який інший обхід із чергуванням напрямку на
межі (навіть якщо межа — не діагональ, а, наприклад, рядок).

**Складність:** O(M·N) час (кожен елемент відвідується рівно один раз),
O(1) додаткової пам'яті поза виходом (O(M+N) якщо потрібно згрупувати
елементи по діагоналях у списки).

## Drills

1. **Medium_03_01** — групування елементів по діагоналях (i + j = const) без зигзагу, просто список списків.
2. **Medium_03_02** — підрахунок кількості діагоналей, що містять більше одного елемента.
3. **Medium_03_03** — суми елементів по кожній антидіагоналі (`result[k]` = сума при i+j=k), пряме використання індексної математики.
4. **Medium_03_04** — той самий принцип "зміна напрямку на межі", але по рядках (боустрофедон): парні рядки зліва направо, непарні — справа наліво.
5. **Medium_03_05** — той самий зигзаг-обхід по діагоналях, що й в оригіналі, але на прямокутній (не квадратній) матриці, щоб перевірити коректність обробки межових випадків.

Розв'язуй у порядку 01 → 05, не підглядаючи в Medium_03_DiagonalTraverse.java.
