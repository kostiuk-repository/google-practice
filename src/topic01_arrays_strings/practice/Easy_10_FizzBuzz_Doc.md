# Easy_10_FizzBuzz — Довідка

## Техніка: Extensible Rule List (замість if-else chain)

**Суть:** замість жорсткого ланцюжка `if (i % 15 == 0) ... else if (i % 3 == 0) ...
else if (i % 5 == 0) ...`, правила зберігаються як список пар
"дільник → слово", впорядкований за зростанням дільника. Для кожного
числа проходимо цей список, і якщо число ділиться на дільник правила —
додаємо відповідне слово до результату. Якщо жодне правило не
спрацювало — повертаємо саме число як рядок. Додавання нового правила
(наприклад, "Bang" для 7) стає рядком коду в списку, а не новою гілкою
в умовному операторі.

```java
List<int[]> rules = ...; // або List<Map.Entry<Integer,String>>
int[][] divisorWord = {{3, 0}, {5, 1}}; // спрощено; реально пари (divisor, word)
for (int i = 1; i <= n; i++) {
    StringBuilder sb = new StringBuilder();
    for (var rule : rules) {
        if (i % rule.divisor == 0) sb.append(rule.word);
    }
    result.add(sb.length() > 0 ? sb.toString() : String.valueOf(i));
}
```

**Коли застосовувати:** будь-яка задача класифікації/маркування, де
кількість правил може зростати з часом (нові дільники, нові категорії,
нові фічі-флаги). Ключова ознака: якщо додавання ще однієї умови
змусило б переписувати наявний if-else ланцюжок — час винести правила
в структуру даних, яку прохід ітерує.

**Складність:** O(N * R) час, де N — довжина діапазону, R — кількість
правил (зазвичай мала константа); O(1) додаткової пам'яті на правило.

## Drills

3. **Easy_10_03** — агрегація в лічильники [fizzOnly, buzzOnly, fizzBuzz] замість списку рядків.
