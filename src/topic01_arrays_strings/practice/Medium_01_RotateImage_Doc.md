# Medium_01_RotateImage — Довідка

## Техніка: In-Place Matrix Transformation (Transpose + Reverse / Layer Swap)

**Суть:** обертання квадратної матриці на 90° in-place можна звести до
композиції двох простіших операцій — транспонування (`mat[i][j] ↔ mat[j][i]`)
і реверсу рядків (або порядку рядків). Альтернативний погляд на ту саму
задачу — пошарове обертання: для кожного "кільця" (layer) матриці
циклічно переставляються четвірки елементів (top → right → bottom → left →
top) без виділення додаткової пам'яті.

```java
// transpose
for (int i = 0; i < n; i++)
    for (int j = i + 1; j < n; j++) {
        int tmp = mat[i][j];
        mat[i][j] = mat[j][i];
        mat[j][i] = tmp;
    }
// reverse each row (clockwise) — or reverse row order (counter-clockwise)
for (int[] row : mat) reverseInPlace(row);
```

**Коли застосовувати:** будь-яка задача "оберни/відобрази матрицю без
виділення нової матриці" — 90°/180° повороти, дзеркальне відображення,
обертання лише певного шару (кільця). Ключова ознака: потрібна модифікація
`in-place`, і задачу можна розкласти на дві елементарні перестановки або на
цикл чотиристороннього обміну по шарах.

**Складність:** O(N²) час (N — розмір сторони матриці), O(1) додаткової
пам'яті — усі перестановки відбуваються в межах вхідної матриці.

## Drills

1. **Medium_01_01** — обертання на 90° проти годинникової стрілки (transpose + reverse порядку рядків замість reverse кожного рядка).
2. **Medium_01_02** — обертання на 180° (пряма перестановка протилежних кутів `mat[i][j] ↔ mat[n-1-i][n-1-j]`).
3. **Medium_01_03** — ізольоване транспонування квадратної матриці in-place, без другого кроку.
4. **Medium_01_04** — перевірка "чи має матриця обертальну симетрію" (обертання застосовується до копії й порівнюється з оригіналом).
5. **Medium_01_05** — обертання на 90° лише зовнішнього кільця (пошаровий чотиристоронній обмін для одного, найзовнішнього шару).

Розв'язуй у порядку 01 → 05, не підглядаючи в Medium_01_RotateImage.java.
