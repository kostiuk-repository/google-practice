# Easy_08_LargestNumberTwice — Довідка

## Суть теми / алгоритму простими словами
Техніка **Two-Tracker Single Pass (Двозначний трекінг за один прохід)** дозволяє знаходити найбільший (`largest`) та другий за величиною (`secondLargest`) елементи масиву паралельно за один прохід. 
Замість того, щоб спочатку знаходити максимум за один прохід, а потім шукати другий максимум за другий прохід, ми оновлюємо обидва показники каскадно: якщо поточний елемент більший за глобальний максимум, старий глобальний максимум стає другим максимумом, а поточний елемент записується як новий глобальний максимум. Якщо ж він менший за максимум, але більший за другий максимум, ми оновлюємо тільки його.

Базова задача: знайти, чи є найбільший елемент масиву щонайменше вдвічі більшим за будь-який інший елемент масиву. Якщо так — повернути його індекс, інакше повернути `-1`.

### a) Звичайний випадок: `nums = [3, 6, 1, 0]`
Ініціалізуємо трекери:
- `maxIndex = -1`
- `largest = Integer.MIN_VALUE`
- `secondLargest = Integer.MIN_VALUE`

| Крок (Індекс `i`) | Елемент `nums[i]` | Умова `nums[i] > largest` | Умова `nums[i] > secondLargest` | Новий `largest` (та `maxIndex`) | Новий `secondLargest` |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Старт** | — | — | — | -2147483648 | -2147483648 |
| 0 | 3 | `3 > MIN_VALUE` є `true` | — | 3 (`maxIndex = 0`) | -2147483648 |
| 1 | 6 | `6 > 3` є `true` | — | 6 (`maxIndex = 1`) | 3 (старий `largest`) |
| 2 | 1 | `1 > 6` є `false` | `1 > 3` є `false` | 6 | 3 |
| 3 | 0 | `0 > 6` є `false` | `0 > 3` є `false` | 6 | 3 |

**Після циклу:** Перевіряємо умову домінування: `largest >= 2 * secondLargest` -> `6 >= 2 * 3` -> `6 >= 6` (є `true`).
**Результат:** `maxIndex = 1`.

### b) Граничний випадок: `nums = [3, 4, 1]` (найбільший не перевершує інших удвічі)
| Крок (Індекс `i`) | Елемент `nums[i]` | Новий `largest` | Новий `secondLargest` |
| :--- | :--- | :--- | :--- |
| **Старт** | — | -2147483648 | -2147483648 |
| 0 | 3 | 3 | -2147483648 |
| 1 | 4 | 4 | 3 |
| 2 | 1 | 4 | 3 |

**Після циклу:** `largest >= 2 * secondLargest` -> `4 >= 2 * 3` -> `4 >= 6` (є `false`).
**Результат:** `-1`.

## Ключова ідея та шаблон коду (Core Concept & Code Template)

Домінування найбільшого елемента над **всіма** іншими елементами масиву є еквівалентним домінуванню над **другим за величиною** елементом. Тобто, якщо найбільший елемент $M_1 \ge 2 \cdot M_2$ (де $M_2$ — другий за величиною елемент), то він гарантовано більший або дорівнює подвоєному значенню будь-якого іншого елемента в масиві. Це дозволяє уникнути перевірки кожного елемента окремо. Ми шукаємо $M_1$ та $M_2$ за один прохід, зберігаючи також індекс максимуму.

### Базовий шаблон коду (Java)
```java
public int dominantIndex(int[] nums) {
    if (nums == null || nums.length < 2) {
        return (nums != null && nums.length == 1) ? 0 : -1;
    }
    
    int largest = Integer.MIN_VALUE;
    int secondLargest = Integer.MIN_VALUE;
    int maxIndex = -1;
    
    for (int i = 0; i < nums.length; i++) {
        int x = nums[i];
        // Якщо поточний елемент більший за найбільший
        if (x > largest) {
            // Старий максимум стає другим за величиною
            secondLargest = largest;
            // Поточний стає новим максимумом
            largest = x;
            maxIndex = i;
        } 
        // Якщо поточний елемент менший або дорівнює найбільшому, але більший за другий
        else if (x > secondLargest) {
            secondLargest = x;
        }
    }
    
    // Перевіряємо, чи найбільший хоча б вдвічі більший за другий найбільший
    if (largest >= 2 * secondLargest) {
        return maxIndex;
    }
    
    return -1;
}
```

### Аналіз складності (Complexity Analysis)
- **Time Complexity (Часова складність):** $O(N)$, де $N$ — довжина масиву `nums`. Здійснюється рівно один прохід.
- **Space Complexity (Просторова складність):** $O(1)$, оскільки потрібна фіксована кількість змінних незалежно від розміру масиву.

## Граничні випадки та підводні камені (Edge Cases & Common Pitfalls)

1. **Неправильна ініціалізація трекерів:** Якщо ініціалізувати `largest = 0` та `secondLargest = 0`, а масив містить від'ємні числа, то нулі залишаться переможцями, що є помилкою. Завжди використовуйте `Integer.MIN_VALUE` для ініціалізації пошуку максимуму.
2. **Ігнорування дублікатів:** Якщо вхідний масив містить однакові максимальні елементи, наприклад, `[6, 6, 2]`. Оскільки найбільший елемент має бути унікальним (за умовою деяких задач), поведінка трекерів при рівності `x == largest` має бути чітко визначена. 
3. **Неправильний каскадний запис:** Найчастіша помилка новачків:
   ```java
   if (x > largest) {
       largest = x; // Значення старого максимуму втрачено для secondLargest!
   }
   ```
   Порядок має бути суворим: спочатку присвоюємо `secondLargest = largest`, а потім оновлюємо `largest = x`.
