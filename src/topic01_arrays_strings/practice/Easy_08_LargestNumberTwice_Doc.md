# Easy_08_LargestNumberTwice — Довідка

## Техніка: Two-Tracker Single Pass (Largest / Second-Largest)

**Суть:** один прохід масивом, дві змінні-трекери — `largest` і
`secondLargest` — оновлюються разом на кожному кроці. Якщо поточний
елемент більший за `largest`, старий `largest` "спускається" в
`secondLargest`, а поточний елемент стає новим `largest`. Інакше, якщо
поточний елемент більший за `secondLargest` (але не за `largest`),
оновлюється лише `secondLargest`. Ідею легко узагальнити на три й
більше трекерів або замінити другий трекер на лічильник чи індекс.

```java
int largest = Integer.MIN_VALUE, secondLargest = Integer.MIN_VALUE;
for (int x : nums) {
    if (x > largest) {
        secondLargest = largest;
        largest = x;
    } else if (x > secondLargest) {
        secondLargest = x;
    }
}
```

**Коли застосовувати:** будь-яке завдання, де відповідь залежить від
одного або двох "найкращих" значень, а другий прохід масивом (щоб
порівняти все з максимумом) є зайвим. Ключова ознака: питання виду
"найбільший ЩОДО другого найбільшого" або "домінує над рештою".

**Складність:** O(N) час, O(1) додаткової пам'яті — незалежно від
кількості трекерів (два, три...), пам'ять залишається сталою.

## Drills

1. **Easy_08_01** — повернути саме друге за величиною значення (сам трекер — відповідь).
2. **Easy_08_02** — знайти індекс найменшого елемента, що не перевищує половини максимуму.
3. **Easy_08_03** — різниця між найбільшим і другим найбільшим значеннями.
4. **Easy_08_04** — трекер максимуму + лічильник входжень замість другого значення.
5. **Easy_08_05** — узагальнення до ТРЬОХ трекерів: топ-3 різних значення.

Розв'язуй у порядку 01 → 05, не підглядаючи в Easy_08_LargestNumberTwice.java.
