# Easy_06_LengthOfLastWord — Довідка

## Суть теми / алгоритму простими словами
Техніка **Ручного посимвольного сканування з кінця (Right-to-Left Custom Scanning)** використовується для виділення токенів (слів) без використання високорівневих методів, таких як `split()`, `trim()` чи регулярних виразів. 
Оскільки нам потрібне лише останнє слово, немає сенсу розбивати весь рядок на масив підрядків (що алокує багато зайвої пам'яті). Натомість ми починаємо перегляд з кінця рядка:
1. Пропускаємо всі кінцеві пробіли (якщо вони є).
2. Рахуємо довжину слова, доки не зустрінемо перший пробіл перед ним або початок рядка.

## Інженерний контекст: parse only what the query needs

Задача вчить важливій оптимізації: якщо потрібен лише хвіст структури, не обов’язково materialize всю структуру. `split()` створив би всі tokens, хоча відповідь залежить тільки від останнього.

### Де зустрічається

- отримання basename або останнього path segment;
- пошук останнього поля в delimiter-separated record;
- читання file trailer, suffix marker або останньої команди;
- trim-подібна нормалізація без створення проміжного рядка;
- reverse parsing там, де grammar локальна з кінця.

Це різновид **demand-driven parsing**: алгоритм читає мінімальний фрагмент input, достатній для відповіді. Для дуже довгих рядків або memory-sensitive code це зменшує allocations і GC pressure.

### Що перевіряє співбесіда

- чи розділите trailing separators і сам token на дві фази;
- чи коректно обробите рядок лише з пробілів;
- чи не вийдете за index `-1`;
- чи поясните, що два послідовні `while` разом залишаються $O(n)$;
- чи уточните, що в production «слово» може вимагати складнішого Unicode/tokenization контракту.

Це не «два проходи по рядку»: кожен індекс відвідується максимум один раз. Обидва цикли разом мають $O(N)$ часу.

### a) Звичайний випадок: `s = "   to the moon  "`
Індекс починається з кінця: `i = s.length() - 1 = 15`.

**Крок 1: Пропуск кінцевих розділювачів (пробілів)**
| Індекс `i` | Символ `s.charAt(i)` | Умова `== ' '` | Зміна індексу `i` |
| :--- | :--- | :--- | :--- |
| 15 | `' '` (пробіл) | `true` | `15 -> 14` |
| 14 | `' '` (пробіл) | `true` | `14 -> 13` |
| 13 | `'n'` | `false` | Зупинка першого циклу |

**Крок 2: Підрахунок символів останнього слова**
Початкова довжина `length = 0`.

| Індекс `i` | Символ `s.charAt(i)` | Умова `!= ' '` | Довжина `length` | Зміна індексу `i` |
| :--- | :--- | :--- | :--- | :--- |
| 13 | `'n'` | `true` | `0 + 1 = 1` | `13 -> 12` |
| 12 | `'o'` | `true` | `1 + 1 = 2` | `12 -> 11` |
| 11 | `'o'` | `true` | `2 + 1 = 3` | `11 -> 10` |
| 10 | `'m'` | `true` | `3 + 1 = 4` | `10 -> 9` |
| 9 | `' '` (пробіл) | `false` | Зупинка циклу | `9` |

**Результат:** `4`.

### b) Граничний випадок: `s = "     "` (тільки пробіли)
Індекс починається з `i = 4`.

**Крок 1: Пропуск кінцевих розділювачів**
Всі символи є пробілами, тому індекс `i` зменшується до `-1` і цикл завершується.

**Крок 2: Підрахунок символів останнього слова**
Оскільки `i = -1`, другий цикл не починається взагалі.

**Результат:** `0`.

## Ключова ідея та шаблон коду (Core Concept & Code Template)

Використання `String.trim().split(" ")` для пошуку останнього слова працює за час $O(N)$, але вимагає виділення $O(N)$ пам'яті для зберігання масиву всіх слів. Замість цього, використовуючи два послідовні цикли `while` з індексом `i` від кінця рядка, ми досягаємо $O(1)$ додаткової пам'яті. 
Перший цикл шукає кінець останнього слова, другий цикл відраховує його довжину від кінця до початку.

### Базовий шаблон коду (Java)
```java
public int lengthOfLastWord(String s) {
    // Починаємо з останнього індексу вхідного рядка
    int i = s.length() - 1;
    
    // Цикл 1: Пропускаємо всі кінцеві пробіли
    while (i >= 0 && s.charAt(i) == ' ') {
        i--;
    }
    
    // Змінна для підрахунку довжини слова
    int length = 0;
    
    // Цикл 2: Рахуємо літери слова, доки не зустрінемо наступний пробіл
    while (i >= 0 && s.charAt(i) != ' ') {
        length++;
        i--;
    }
    
    return length;
}
```

### Аналіз складності (Complexity Analysis)
- **Time Complexity (Часова складність):** $O(N)$, де $N$ — довжина рядка `s`. У найгіршому випадку (слово на початку або тільки пробіли) ми переглядаємо всі символи один раз.
- **Space Complexity (Просторова складність):** $O(1)$, оскільки ми використовуємо лише один індекс-вказівник та змінну довжини, не створюючи жодних додаткових об'єктів.

## Граничні випадки та підводні камені (Edge Cases & Common Pitfalls)

1. **Вихід за межі масиву (Index Out Of Bounds):** Завжди перевіряйте умову `i >= 0` першою всередині умов циклу. Якщо спочатку написати `s.charAt(i) == ' '`, а потім `i >= 0`, то при `i = -1` виникне помилка `StringIndexOutOfBoundsException`.
2. **Рядок без пробілів:** Наприклад, `s = "hello"`. Перший цикл не зробить жодної ітерації, а другий пройде по всьому рядку та успішно поверне `5` (умова `i >= 0` зупинить цикл).
3. **Використання бібліотечних методів:** Новачки часто пишуть `s.trim()`, що створює новий рядок. У великих циклах обробки це призводить до зниження продуктивності.
