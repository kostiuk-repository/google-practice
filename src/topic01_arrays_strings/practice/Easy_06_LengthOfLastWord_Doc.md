# Easy_06_LengthOfLastWord — Довідка

## Техніка: Ручний посимвольний скан з пропуском розділювачів (без split()/trim())

**Суть:** замість виклику `split()` чи `trim()`, які алокують масиви й
рядки, рядок сканується вручну індексом посимвольно. Спершу
пропускаються "порожні" розділювачі (пробіли, коми тощо), потім
рахуються/збираються символи "токена" (слова), доки знову не
зустрінеться розділювач або кінець рядка. Напрямок скану (з початку чи
з кінця) і тип розділювача — параметри конкретної задачі, сама схема
циклу незмінна.

```java
int i = s.length() - 1;
while (i >= 0 && s.charAt(i) == ' ') i--;   // пропустити розділювачі
int length = 0;
while (i >= 0 && s.charAt(i) != ' ') {      // порахувати символи токена
    length++;
    i--;
}
```

**Коли застосовувати:** парсинг рядків без бібліотечних методів
розбиття, коли важлива продуктивність (без зайвих алокацій) або коли
потрібна нестандартна поведінка (свій розділювач, скан з кінця,
трекінг максимуму серед токенів).

**Складність:** O(N) час (N — довжина рядка), O(1) додаткової пам'яті
(окрім випадків, коли треба повернути сам текст токена).

## Drills

1. **Easy_06_01** — довжина ПЕРШОГО слова (скан з початку, пропустити пробіли на початку).
2. **Easy_06_02** — порахувати кількість слів у рядку.
3. **Easy_06_03** — повернути текст останнього слова (не довжину, а сам підрядок).
4. **Easy_06_04** — той самий скан, але розділювач — кома замість пробілу.
5. **Easy_06_05** — довжина НАЙДОВШОГО слова (трекінг running max під час скану).

Розв'язуй у порядку 01 → 05, не підглядаючи в Easy_06_LengthOfLastWord.java.
