# Hard_01_FirstMissingPositive — Довідка

## Техніка: Cyclic Sort / Index Placement

**Суть:** масив використовується як власна імпліцитна hash-множина без
виділення додаткової пам'яті. Для кожного значення `x` у "цільовому"
діапазоні (наприклад, `1 <= x <= n`) обчислюється "домашній" індекс
(`x - 1`), і значення циклічно обмінюється (`swap`) в цей індекс, поки
на поточній позиції не опиниться значення, яке вже стоїть на своєму
місці, або значення поза діапазоном. Після цього проходу другий,
фінальний прохід перевіряє, на якому індексі `i` значення не дорівнює
очікуваному (`i + 1`) — це і є "відповідь" (перша пропущена позитивна,
дублікат тощо).

```java
int i = 0;
while (i < n) {
    int target = nums[i] - 1; // домашній індекс для значення nums[i]
    if (nums[i] > 0 && nums[i] <= n && nums[nums[i] - 1] != nums[i]) {
        int tmp = nums[target];
        nums[target] = nums[i];
        nums[i] = tmp;
    } else {
        i++;
    }
}
for (i = 0; i < n; i++) {
    if (nums[i] != i + 1) return i + 1; // або зібрати всі такі i
}
```

**Коли застосовувати:** задачі "знайти пропущене/дубльоване число(а) в
масиві розміру N з обмеженим діапазоном значень [1..N] або [0..N-1]" за
умови O(N) часу та O(1) додаткової пам'яті — коли HashSet заборонений
обмеженням складності.

**Складність:** O(N) час (кожен елемент переміщується на своє місце не
більше одного разу за амортизованим аналізом), O(1) додаткова пам'ять
(окрім, за потреби, вихідного списку/масиву результату).

## Drills

1. **Hard_01_01** — перевірка, чи масив містить усі числа від 1 до n (булевий переказ).
2. **Hard_01_02** — пошук першого дубліката за допомогою тієї ж техніки розміщення.
3. **Hard_01_03** — збір УСІХ пропущених позитивних чисел, а не лише першого.
4. **Hard_01_04** — зсув формули цільового індексу: діапазон [0, n-1] замість [1, n].
5. **Hard_01_05** — "Set Mismatch": одночасний пошук дубліката й пропущеного числа.

Розв'язуй у порядку 01 → 05, не підглядаючи в Hard_01_FirstMissingPositive.java.
