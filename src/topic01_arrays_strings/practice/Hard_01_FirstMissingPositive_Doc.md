# First Missing Positive — Довідка

## Суть теми / алгоритму простими словами
Техніка **Cyclic Sort** (або **Index Placement** / циклічне сортування) — це підхід, який дозволяє впорядкувати масив чисел за лінійний час $O(N)$ без використання додаткової пам'яті $O(1)$. 

Основна ідея полягає в тому, що якщо ми маємо масив розміром $N$, то перше пропущене позитивне число обов'язково лежить у діапазоні від $1$ до $N + 1$. Ми намагаємося помістити кожне число $x$, яке лежить у межах $[1, N]$, на його «правильне» місце — індекс $x - 1$. Для цього ми циклічно міняємо місцями елементи, доки кожне коректне число не опиниться на своєму місці. Після цього ми просто шукаємо перший індекс $i$, де значення не дорівнює $i + 1$.

### Опис базової задачі
Дано невідсортований масив цілих чисел `nums`. Потрібно знайти найменше пропущене позитивне ціле число за час $O(N)$ та з використанням додаткової пам'яті $O(1)$.

Доказ лінійного часу: кожен успішний swap ставить принаймні одне значення на його остаточне місце, отже таких swap не більше `n`.

### a) Звичайний випадок: `nums = [3, 4, -1, 1]` ($N = 4$)

Ми проходимо по масиву вказівником `i` від $0$ до $N-1$. Якщо число `nums[i]` знаходиться в діапазоні $[1, 4]$ і ще не стоїть на своєму місці (тобто `nums[nums[i] - 1] != nums[i]`), ми міняємо його місцями з елементом на цільовому індексі. Ми **не** збільшуємо `i`, поки на поточній позиції не опиниться число, яке не підходить для перестановки.

| Крок | Стан масиву `nums` | `i` | `nums[i]` | Цільовий індекс (`nums[i] - 1`) | Дія |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **1** | `[3, 4, -1, 1]` | 0 | 3 | $3 - 1 = 2$ | `nums[2]` це -1 (не 3). Міняємо місцями `nums[0]` та `nums[2]`. |
| **2** | `[-1, 4, 3, 1]` | 0 | -1 | — | -1 не позитивне. Переміщення неможливе. Збільшуємо `i` до 1. |
| **3** | `[-1, 4, 3, 1]` | 1 | 4 | $4 - 1 = 3$ | `nums[3]` це 1 (не 4). Міняємо місцями `nums[1]` та `nums[3]`. |
| **4** | `[-1, 1, 3, 4]` | 1 | 1 | $1 - 1 = 0$ | `nums[0]` це -1 (не 1). Міняємо місцями `nums[1]` та `nums[0]`. |
| **5** | `[1, -1, 3, 4]` | 1 | -1 | — | -1 не позитивне. Збільшуємо `i` до 2. |
| **6** | `[1, -1, 3, 4]` | 2 | 3 | $3 - 1 = 2$ | `nums[2]` вже дорівнює 3. Збільшуємо `i` до 3. |
| **7** | `[1, -1, 3, 4]` | 3 | 4 | $4 - 1 = 3$ | `nums[3]` вже дорівнює 4. Збільшуємо `i` до 4 (кінець циклу). |

**Фінальний прохід перевірки:**
- `i = 0`: `nums[0] == 1` — ок.
- `i = 1`: `nums[1] == -1` (має бути 2). Невідповідність знайдено!
- **Результат:** повертаємо **2**.

### b) Граничний випадок (з дублікатами): `nums = [1, 1]` ($N = 2$)

| Крок | Стан масиву `nums` | `i` | `nums[i]` | Цільовий індекс (`nums[i] - 1`) | Дія |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **1** | `[1, 1]` | 0 | 1 | $1 - 1 = 0$ | `nums[0]` вже дорівнює 1. Збільшуємо `i` до 1. |
| **2** | `[1, 1]` | 1 | 1 | $1 - 1 = 0$ | Цільовий елемент `nums[0]` вже містить 1. Спроба обміну призведе до нескінченного циклу. Пропускаємо, збільшуємо `i` до 2. |

**Фінальний прохід перевірки:**
- `i = 0`: `nums[0] == 1` — ок.
- `i = 1`: `nums[1] == 1` (має бути 2). Невідповідність!
- **Результат:** повертаємо **2**.

## Ключова ідея та шаблон коду (Core Concept & Code Template)

Масив виступає в ролі спрощеної хеш-таблиці. Оскільки ми обмежені пам'яттю $O(1)$, ми не можемо виділити новий масив чи `HashSet`. Проте ми знаємо:
1. Будь-яке число $x$, яке менше за 1 або більше за $N$, не може бути шуканою відповіддю.
2. Будь-яке число $x \in [1, N]$ має зберігатися на індексі $x - 1$.
3. Якщо ми переставимо всі такі числа на свої місця, то перший індекс `i`, де `nums[i] != i + 1`, вкаже на пропущене число `i + 1`. Якщо всі позиції заповнені правильно, то пропущене число — це `N + 1`.

### Базовий шаблон коду (Java)

```java
package topic01_arrays_strings.practice;

public class Hard_01_FirstMissingPositive {

    public int firstMissingPositive(int[] nums) {
        if (nums == null || nums.length == 0) {
            return 1;
        }
        
        int n = nums.length;
        int i = 0;
        
        // Крок 1: Циклічне сортування (розміщення елементів на свої місця)
        while (i < n) {
            int currentVal = nums[i];
            int targetIdx = currentVal - 1;
            
            // Перевіряємо умови для обміну:
            // 1. Число повинно бути в діапазоні [1, n]
            // 2. Елемент на цільовому індексі ще не дорівнює нашому числу (запобігає нескінченному циклу для дублікатів)
            if (currentVal > 0 && currentVal <= n && nums[targetIdx] != currentVal) {
                // Міняємо місцями nums[i] та nums[targetIdx]
                int temp = nums[targetIdx];
                nums[targetIdx] = currentVal;
                nums[i] = temp;
                // Не збільшуємо i, тому що після обміну на nums[i] 
                // опинилося нове значення, яке також треба перевірити
            } else {
                i++;
            }
        }
        
        // Крок 2: Знаходження першої невідповідності
        for (i = 0; i < n; i++) {
            if (nums[i] != i + 1) {
                return i + 1; // Пропущене число
            }
        }
        
        // Якщо всі числа від 1 до n на своїх місцях
        return n + 1;
    }
}
```

### Аналіз складності
- **Часова складність (Time Complexity):** $O(N)$. Хоча всередині циклу `while` є операції обміну без інкременту `i`, кожен обмін ставить щонайменше одне число на його фінальну позицію. Оскільки елемент може бути встановлений на правильне место щонайбільше один раз, сумарна кількість обмінів не перевищує $N$. Отже, амортизований час роботи становить $O(N)$.
- **Просторова складність (Space Complexity):** $O(1)$. Ми модифікуємо вхідний масив `in-place` і використовуємо лише кілька локальних змінних для індексів та обміну.

## Граничні випадки та підводні камені (Edge Cases & Common Pitfalls)

1. **Нескінченний цикл при дублікатах:**
   Якщо перевіряти лише `targetIdx != i`, то для масиву `[1, 1]` на індексі `i = 1` програма намагатиметься обміняти `nums[1]` з `nums[0]`, оскільки цільовий індекс для `1` — це `0`. Але `nums[0]` вже дорівнює `1`. Це призведе до нескінченного циклу обмінів. 
   *Вирішення:* Завжди перевіряйте `nums[targetIdx] != nums[i]` (число на цільовому місці не повинно дорівнювати поточному числу).

2. **Помилки Off-by-one (вихід за межі масиву):**
   Цільовий індекс для числа `x` дорівнює `x - 1`. Якщо `x` дорівнює `n`, його індекс `n - 1`. Дуже важливо дозволяти обробку значень, які точно дорівнюють `n` (`currentVal <= n`), але уникати виходу за межі масиву.

3. **Від'ємні числа та нуль:**
   Вони повинні ігноруватися при перестановках (`currentVal > 0`), оскільки для них немає валідного індексу в діапазоні $[0, N-1]$.

4. **Порожній масив або null:**
   Хоча зазвичай обмеження задач гарантують довжину хоча б 1, варто додавати перевірку на пустий масив, для якого першим пропущеним позитивним є `1`.

## Розбір Drills (Drills Breakdown)

### 1. Hard_01_01 — Contains All One To N
- **Мета:** Перевірити, чи масив містить абсолютно всі числа від 1 до $N$ без дублікатів і пропусків (будь-який інший елемент або дублікат робить відповідь `false`).
- **Відмінність від основної:** Ми повертаємо `boolean` замість числа.
- **Підказка:** Використовуйте циклічне сортування. Після сортування перевірте кожну позицію: якщо знайдено `nums[i] != i + 1`, одразу повертайте `false`. Якщо цикл завершився успішно — `true`.
- **Приклад:** 
  - Вхід: `[3, 1, 2]`. Після сортування: `[1, 2, 3]`. Усі `nums[i] == i + 1`. Вихід: `true`.
  - Вхід: `[1, 2, 4]`. 4 ігнорується, бо $4 > 3$. Після сортування: `[1, 2, 4]`. На індексі 2: `nums[2] = 4 != 3`. Вихід: `false`.

### 2. Hard_01_02 — First Duplicate Positive
- **Мета:** Знайти перше дубльоване позитивне число в діапазоні $[1, N]$ за його фінальною позицією після спроби впорядкування.
- **Відмінність від основної:** Ми шукаємо не пропущене число, а число, яке дублюється і лежить на «чужому» місці.
- **Підказка:** Виконайте циклічне сортування. Під час фінального проходу перевірте: якщо `nums[i] != i + 1`, то елемент `nums[i]` є дублікатом (оскільки його правильне місце `nums[i] - 1` вже зайняте копією цього числа). Поверніть `nums[i]`. Якщо дублікатів немає, поверніть `-1`.
- **Приклад:** 
  - Вхід: `[1, 2, 2, 3]`. Після сортування: `[1, 2, 3, 2]`. На індексі 3: `nums[3] = 2 != 4`. Вихід: `2`.

### 3. Hard_01_03 — All Missing Positives
- **Мета:** Знайти ВСІ пропущені позитивні числа в діапазоні від 1 до $N$.
- **Відмінність від основної:** Повертаємо список `List<Integer>` усіх пропущених чисел, а не лише першого.
- **Підказка:** Виконайте стандартне циклічне сортування. Під час другого проходу перевірте весь масив: для кожного індексу `i`, де `nums[i] != i + 1`, додайте значення `i + 1` у результуючий список.
- **Приклад:** 
  - Вхід: `[4, 3, 2, 7, 8, 2, 3, 1]`. Після сортування: `[1, 2, 3, 4, 3, 2, 7, 8]`. Невідповідності на індексі 4 (очікується 5) та індексі 5 (очікується 6). Вихід: `[5, 6]`.

### 4. Hard_01_04 — First Missing Non-Negative
- **Мета:** Знайти найменше пропущене невід'ємне число (0, 1, 2...).
- **Відмінність від основної:** Діапазон зміщується з $[1, N]$ на $[0, N-1]$.
- **Підказка:** Змініть формулу цільового індексу. Тепер число $x$ має стояти на індексі $x$ (тобто `targetIdx = nums[i]`). Умови обміну: `nums[i] >= 0 && nums[i] < n && nums[nums[i]] != nums[i]`. У фінальному проході шукаємо індекс, де `nums[i] != i`. Якщо такого немає, повертаємо `n`.
- **Приклад:** 
  - Вхід: `[0, 1, 3]`. Після сортування: `[0, 1, 3]` (3 ігнорується, бо $3 \ge 3$). На індексі 2: `nums[2] = 3 != 2`. Вихід: `2`.

### 5. Hard_01_05 — Find Missing And Duplicate
- **Мета:** Знайти пару чисел `[дублікат, пропущене]` в масиві, де одне з чисел від 1 до $N$ було перезаписано іншим.
- **Відмінність від основної:** Повертаємо масив з двох елементів.
- **Підказка:** Виконайте циклічне сортування. Під час перевірки знайдіть індекс `i`, на якому `nums[i] != i + 1`. Число `nums[i]` — це дублікат, а очікуване число `i + 1` — це пропущене число. Поверніть `new int[]{nums[i], i + 1}`.
- **Приклад:** 
  - Вхід: `[1, 2, 2, 4]`. Після сортування: `[1, 2, 2, 4]`. На індексі 2: `nums[2] = 2 != 3`. Вихід: `[2, 3]`.
