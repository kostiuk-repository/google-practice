# Medium_02_RottingOranges — Довідка

## Техніка: Multi-Source BFS (Level-by-Level Graph Infection)

**Суть:** симулювати процес поширення гнилі на сітці. Оскільки всі гнилі апельсини поширюють гниль одночасно, ми застосовуємо BFS з кількох джерел (multi-source BFS). До черги спочатку додаються ВСІ початково гнилі апельсини, а підрахунок хвилин ведеться пошарово (level-by-level).

```java
Queue<int[]> queue = new LinkedList<>();
for (int r = 0; r < rows; r++) {
    for (int c = 0; c < cols; c++) {
        if (grid[r][c] == 2) queue.add(new int[]{r, c});
    }
}
int minutes = 0;
while (!queue.isEmpty()) {
    // Обробка рівня...
}
```

**Коли застосовувати:** пошук найкоротшого часу поширення процесу (інфекція, пожежа, вода, чутки) з кількох початкових точок одночасно у незваженому графі/сітці.

**Складність:** O(M×N) час та пам'ять, оскільки кожна клітинка сітки відвідується скінченну кількість разів.

## Drills

1. **Medium_02_01** — підрахунок початкової кількості свіжих та гнилих апельсинів.
2. **Medium_02_02** — виконання одного кроку поширення гнилі (заливка на 1 рівень).
3. **Medium_02_03** — знаходження відстаней від одного заданого джерела інфекції до всіх інших клітинок.
4. **Medium_02_04** — знаходження максимальної відстані від будь-якої суші до найближчої води (multi-source BFS).
5. **Medium_02_05** — капстоун: поширення гнилі з урахуванням клітинок-прискорювачів (подвійна швидкість, зважений BFS / Dijkstra).
