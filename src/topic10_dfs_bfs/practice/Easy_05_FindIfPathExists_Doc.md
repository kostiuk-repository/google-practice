# Easy_05_FindIfPathExists — Довідка

## Техніка: Edge List → Adjacency List → DFS/BFS with Visited Set

**Суть:** маючи граф у вигляді плоского списку ребер (`int[][] edges`),
першим кроком завжди будуємо adjacency list — `List<List<Integer>>`,
де `i`-й елемент містить сусідів вузла `i`. Без цього кроку перевірка
"хто сусід вузла X" коштувала б O(E) щоразу; з adjacency list — O(1)
(просто читання списку). Далі — звичайний DFS або BFS від `source` з
множиною `visited`, що зупиняється, коли досягнуто `destination`, або
коли всі досяжні вузли вичерпано.

```java
List<List<Integer>> adj = new ArrayList<>();
for (int i = 0; i < n; i++) adj.add(new ArrayList<>());
for (int[] e : edges) {
    adj.get(e[0]).add(e[1]);
    adj.get(e[1]).add(e[0]); // неорієнтований граф
}

boolean[] visited = new boolean[n];
// DFS або BFS від source, перевіряючи visited перед відвідуванням
```

**Коли застосовувати:** будь-яка задача про зв'язність у графі,
заданому списком ребер — чи існує шлях, скільки компонент зв'язності,
чи можна дістатись за K кроків. Ключова ознака: вхідні дані — це
"плоский" список ребер, а не готова adjacency list чи сітка.

**Складність:** O(V + E) час на побудову adjacency list і один
DFS/BFS-прохід, O(V + E) пам'яті.

## Drills

1. **Easy_05_01** — лише побудова adjacency list зі списку ребер (без обходу).
2. **Easy_05_02** — перевірка існування шляху через BFS замість DFS.
3. **Easy_05_03** — підрахунок кількості компонент зв'язності в усьому графі.
4. **Easy_05_04** — пакетна обробка багатьох запитів (source, destination) за один прохід через component id.
5. **Easy_05_05** — капстоун: чи існує шлях довжиною не більше K ребер (BFS з обмеженням глибини).
