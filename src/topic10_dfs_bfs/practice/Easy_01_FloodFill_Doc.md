# Easy_01_FloodFill — Довідка

## Техніка: Grid DFS/BFS Flood Fill (Same-Value Connectivity)

**Суть:** починаємо з клітинки-джерела (`sr, sc`), запам'ятовуємо її
початкове значення (`originalColor`), і обходимо (DFS або BFS) усі
4-напрямково суміжні клітинки, які МАЮТЬ ТЕ САМЕ початкове значення.
Кожну відвідану клітинку перефарбовуємо (або позначаємо відвіданою) і
рекурсивно/через чергу продовжуємо обхід. Критично: перевіряти
`originalColor`, а не поточне значення клітинки, інакше після
перефарбування першої клітинки умова збігу з сусідами стане хибною
(або, якщо `newColor == originalColor`, обхід піде в нескінченну
рекурсію).

```java
void dfs(int[][] img, int r, int c, int originalColor, int newColor) {
    if (r < 0 || r >= img.length || c < 0 || c >= img[0].length) return;
    if (img[r][c] != originalColor) return;
    img[r][c] = newColor;
    dfs(img, r + 1, c, originalColor, newColor);
    dfs(img, r - 1, c, originalColor, newColor);
    dfs(img, r, c + 1, originalColor, newColor);
    dfs(img, r, c - 1, originalColor, newColor);
}
```

**Коли застосовувати:** будь-яка задача про "залиту фарбою" зв'язну
область у сітці — flood fill, підрахунок острівців, підрахунок розміру
регіону, периметр регіону. Ключова ознака: потрібно обійти всі
клітинки, зв'язані одна з одною через сусідство і спільну властивість
(однакове значення), і щось із ними зробити (порахувати, перефарбувати,
позначити).

**Складність:** O(M×N) час у гіршому випадку (кожна клітинка
відвідується не більше одного разу завдяки перевірці `originalColor` /
`visited`), O(M×N) пам'яті на стек рекурсії (DFS) або чергу (BFS).

## Drills

1. **Easy_01_01** — порахувати розмір зв'язної області (без перефарбування).
2. **Easy_01_02** — та сама техніка, але BFS через чергу замість рекурсивного DFS.
3. **Easy_01_03** — порахувати кількість ОКРЕМИХ зв'язних регіонів у всій сітці.
4. **Easy_01_04** — flood fill з 8-напрямковою (діагональною) суміжністю.
5. **Easy_01_05** — капстоун: flood fill одразу з кількох джерел за один прохід зі спільним visited-станом.

Розв'язуй у порядку 01 → 05, не підглядаючи в Easy_01_FloodFill.java.
