# Easy_04_MaxDepthNaryTree — Довідка

## Техніка: Recursive Depth via "1 + Combine(Children Results)"

**Суть:** глибина дерева з кореня `node` — це `0`, якщо `node == null`,
інакше `1 + max(глибина лівого піддерева, глибина правого піддерева)`.
Це канонічна форма DFS-рекурсії "знизу вгору": кожен виклик чекає
результатів від дітей і комбінує їх у власний результат. Змінюючи лише
функцію-комбінатор (`max` → `+`, або додаючи перевірку різниці), ту саму
рекурсію можна перевикористати для підрахунку вузлів, перевірки
збалансованості, пошуку глибини конкретного вузла тощо.

```java
int maxDepth(TreeNode node) {
    if (node == null) return 0;
    int left = maxDepth(node.left);
    int right = maxDepth(node.right);
    return 1 + Math.max(left, right);
}
```

**Коли застосовувати:** будь-яка задача "яка глибина/висота/кількість
вузлів дерева" або "чи збалансоване дерево" — ключова ознака: відповідь
для вузла обчислюється ЛИШЕ через результати його прямих дітей
(bottom-up рекурсія), без потреби знати щось про предків.

**Складність:** O(N) час (кожен вузол відвідується один раз), O(H)
пам'яті на стек рекурсії, де H — висота дерева (у гіршому випадку O(N)
для виродженого дерева).
