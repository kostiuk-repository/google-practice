# Hard_01_WordLadder — Довідка

## Техніка: BFS Word Transformation Graph (Shortest Path)

**Суть:** знайти найкоротший ланцюжок трансформації від `beginWord` до `endWord`. Кожен крок змінює рівно одну літеру, і кожне проміжне слово має бути у словнику. Задача зводиться до пошуку найкоротшого шляху у незваженому графі слів, що оптимально вирішується через BFS.

```java
Queue<String> queue = new LinkedList<>();
queue.add(beginWord);
Set<String> visited = new HashSet<>();
visited.add(beginWord);

int level = 1;
while (!queue.isEmpty()) {
    int size = queue.size();
    for (int i = 0; i < size; i++) {
        String word = queue.poll();
        if (word.equals(endWord)) return level;
        // Знайти сусідів та додати до черги...
    }
    level++;
}
```

**Коли застосовувати:** пошук мінімальної кількості кроків для переходу з одного стану в інший з обмеженим набором дозволених переходів.

**Складність:** O(N * M^2) час, де N — кількість слів у словнику, M — довжина слова. O(N * M) пам'ять для зберігання черги та visited-сету.

## Drills

1. **Hard_01_01** — перевірка, чи відрізняються два слова рівно на одну літеру.
2. **Hard_01_02** — швидкий пошук усіх сусідів слова за допомогою заміни символів та перевірки в Set.
3. **Hard_01_03** — базовий BFS для обчислення довжини найкоротшого шляху у графі слів.
4. **Hard_01_04** — Word Ladder з підтримкою джокерів (wildcard знаків), які підміняють будь-яку літеру.
5. **Hard_01_05** — капстоун: оптимізація Word Ladder за допомогою двонаправленого BFS (Bidirectional BFS).
