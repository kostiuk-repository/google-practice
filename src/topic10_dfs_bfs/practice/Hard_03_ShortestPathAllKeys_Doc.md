# Hard_03_ShortestPathAllKeys — Довідка

## Техніка: BFS State Space Search (Grid + Bitmask)

**Суть:** знайти найкоротший шлях для збору всіх ключів на сітці з перешкодами та воротами. Стан BFS визначається не лише координатами `(r, c)`, але й поточним набором зібраних ключів, представленим у вигляді бітової маски (`keyMask`). Відвідування однієї і тієї ж клітинки з різними масками ключів вважається різними станами.

```java
// Стан черги: {row, col, keyMask, steps}
Queue<int[]> queue = new LinkedList<>();
boolean[][][] visited = new boolean[rows][cols][1 << totalKeys];
```

**Коли застосовувати:** пошук найкоротшого шляху у лабіринтах, де прохідність залежить від накопиченого стану (ключі, інвентар, фази, заряд).

**Складність:** O(M×N * 2^K) час та пам'ять, де K — кількість ключів (зазвичай K <= 6, тому 2^K <= 64).

## Drills

1. **Hard_03_01** — пошук стартової позиції та підрахунок загальної кількості ключів на сітці.
2. **Hard_03_02** — перевірка можливості відкриття замка за допомогою бітової маски ключів.
3. **Hard_03_03** — BFS для пошуку найкоротшого шляху до єдиного ключа без дверей.
4. **Hard_03_04** — BFS для проходження лабіринту з дверима, маючи всі ключі від самого початку.
5. **Hard_03_05** — капстоун: пошук шляху, в якому наступання на пастку 'T' призводить до втрати всіх наявних ключів.

Розв'язуй у порядку 01 → 05, не підглядаючи в Hard_03_ShortestPathAllKeys.java.
