# Medium_01_CloneGraph — Довідка

## Техніка: Graph Deep Copy via Visited Mapping

**Суть:** створити глибоку копію зв'язного неорієнтованого/орієнтованого графу. Головна проблема — наявність циклів. Для запобігання нескінченній рекурсії ми використовуємо HashMap для відображення `OriginalNode -> ClonedNode`. Коли ми зустрічаємо вузол, який уже є в мапі, ми повертаємо його клон замість рекурсивного створення.

```java
Map<Node, Node> visited = new HashMap<>();

Node clone(Node node) {
    if (node == null) return null;
    if (visited.containsKey(node)) {
        return visited.get(node);
    }
    Node cloneNode = new Node(node.val);
    visited.put(node, cloneNode);
    for (Node neighbor : node.neighbors) {
        cloneNode.neighbors.add(clone(neighbor));
    }
    return cloneNode;
}
```

**Коли застосовувати:** глибоке копіювання довільних структур даних з посиланнями (циклічні списки, графи, сітки з перехресними посиланнями).

**Складність:** O(V + E) час, де V — кількість вершин, E — кількість ребер, оскільки кожен вузол і ребро обходяться один раз. O(V) пам'ять для HashMap.
