# Hard_02_LongestIncreasingPath — Довідка

## Техніка: Grid DFS with Memoization (DAG Longest Path)

**Суть:** знайти довжину найдовшого зростаючого шляху в матриці. Рух дозволений у 4 напрямках на клітинки з суворо більшим значенням. Оскільки переходи йдуть лише в один бік (вгору), граф переходів є DAG (спрямованим ациклічним графом). Застосовується DFS з мемоізацією: результат для кожної клітинки обчислюється один раз і зберігається в кеші `memo[r][c]`.

```java
int dfs(int[][] matrix, int r, int c, int[][] memo) {
    if (memo[r][c] != 0) return memo[r][c];
    int max = 1;
    for (int[] d : dirs) {
        int nr = r + d[0], nc = c + d[1];
        if (nr >= 0 && nr < rows && nc >= 0 && nc < cols && matrix[nr][nc] > matrix[r][c]) {
            max = Math.max(max, dfs(matrix, nr, nc, memo) + 1);
        }
    }
    memo[r][c] = max;
    return max;
}
```

**Коли застосовувати:** пошук екстремальних шляхів у сітках/графах, де переходи ациклічні (гарантують рух за зростанням/спаданням), а підзадачі сильно перекриваються.

**Складність:** O(M×N) час (кожна клітинка обробляється один раз завдяки memo), O(M×N) додаткова пам'ять для таблиці memo та стеку DFS.

## Drills

1. **Hard_02_01** — пошук найдовшого зростаючого шляху з фіксованої клітинки (без мемоізації).
2. **Hard_02_02** — пошук найдовшого зростаючого шляху в одновимірному масиві з кроком сусідства.
3. **Hard_02_03** — обчислення вихідних ступенів (out-degrees) клітинок для топологічного сортування.
4. **Hard_02_04** — найдовший шлях спадання в матриці (еквівалентність задачі).
5. **Hard_02_05** — капстоун: найдовший зростаючий шлях, в якому дозволено зробити один "стрибок" через клітинку меншої або рівної висоти.
