# Medium_03_PacificAtlantic — Довідка

## Техніка: Reverse Reachability Search (Flow from Borders)

**Суть:** знайти клітинки, з яких вода може стекти і в Тихий, і в Атлантичний океани. Замість запусків пошуку з кожної внутрішньої клітинки (що повільно), ми запускаємо зворотний пошук (вгору по схилу) від кордонів Тихого та Атлантичного океанів окремо. Клітинки, які досяжні з обох океанів, є відповіддю.

```java
void dfs(int[][] heights, boolean[][] reachable, int r, int c) {
    reachable[r][c] = true;
    for (int[] d : dirs) {
        int nr = r + d[0], nc = c + d[1];
        if (nr >= 0 && nr < rows && nc >= 0 && nc < cols && !reachable[nr][nc]) {
            if (heights[nr][nc] >= heights[r][c]) { // Тече вгору (зворотний напрямок)
                dfs(heights, reachable, nr, nc);
            }
        }
    }
}
```

**Коли застосовувати:** пошук точок перетину шляхів досяжності до двох або більше цілей (раковин, стоків, виходів) у сітці чи графі.

**Складність:** O(M×N) час на два обходи сітки. O(M×N) додаткової пам'яті для матриць досяжності.

## Drills

1. **Medium_03_01** — перевірка, чи може вода з конкретної клітинки стекти до Тихого океану.
2. **Medium_03_02** — обчислення матриці досяжності для заданого набору початкових клітинок кордону (зворотний хід).
3. **Medium_03_03** — знаходження точок перетину двох булевих матриць досяжності.
4. **Medium_03_04** — розв'язання задачі Pacific/Atlantic з урахуванням однієї непрохідної клітинки-дамби.
5. **Medium_03_05** — капстоун: розширення на 3 океани (Тихий, Атлантичний, Індійський) та знаходження клітинок, здатних текти в усі три.

Розв'язуй у порядку 01 → 05, не підглядаючи в Medium_03_PacificAtlantic.java.
