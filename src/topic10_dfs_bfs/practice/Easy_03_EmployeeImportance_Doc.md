# Easy_03_EmployeeImportance — Довідка

## Техніка: HashMap Lookup + DFS/BFS over an Implicit Tree/Graph

**Суть:** список співробітників — це, по суті, дерево (або граф)
"підпорядкування", закодоване через `id` та список `subordinates`
(ids). Замість лінійного пошуку співробітника за `id` на кожному кроці
(O(N) щоразу), спочатку будуємо `HashMap<Integer, Employee>` — це дає
O(1) доступ до будь-якого співробітника за id. Потім звичайний
DFS (або BFS) спускається по ребрах "subordinates", накопичуючи
потрібний агрегат (суму importance, максимум, глибину, кількість
тощо).

```java
Map<Integer, Employee> byId = new HashMap<>();
for (Employee e : employees) byId.put(e.id, e);

int dfs(int id) {
    Employee e = byId.get(id);
    int total = e.importance;
    for (int subId : e.subordinates) {
        total += dfs(subId);
    }
    return total;
}
```

**Коли застосовувати:** будь-яка задача, де сутності пов'язані через
"посилання за id" (співробітник → підлеглі, файл → підпапки, коментар
→ відповіді) і потрібно агрегувати щось по всьому піддереву/підграфу,
досяжному з заданої точки.

**Складність:** O(N) час (кожен співробітник відвідується один раз
завдяки HashMap-побудові дерева заздалегідь), O(N) пам'яті на
HashMap і стек рекурсії.
