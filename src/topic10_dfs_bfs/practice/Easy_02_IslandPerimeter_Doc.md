# Easy_02_IslandPerimeter — Довідка

## Техніка: Boundary-Edge Counting During Grid Traversal

**Суть:** периметр острова — це кількість "відкритих" ребер його
клітинок, тобто ребер, що межують з водою або виходять за межі сітки.
Є два еквівалентні способи це порахувати:

1. **Арифметичний трюк (без DFS):** для кожної клітинки-суші додати 4,
   а потім відняти 2 за кожного сухопутного сусіда праворуч/знизу
   (кожна така пара клітинок ховає 2 внутрішні, "закриті" ребра).
2. **DFS-обхід з підрахунком меж:** під час обходу острова кожен раз,
   коли сусід виходить за межі сітки АБО є водою, додати 1 до
   периметра.

```java
int dfsPerimeter(int[][] grid, int r, int c, boolean[][] visited) {
    if (r < 0 || r >= grid.length || c < 0 || c >= grid[0].length || grid[r][c] == 0) {
        return 1; // "відкрите" ребро
    }
    if (visited[r][c]) return 0;
    visited[r][c] = true;
    return dfsPerimeter(grid, r + 1, c, visited)
         + dfsPerimeter(grid, r - 1, c, visited)
         + dfsPerimeter(grid, r, c + 1, visited)
         + dfsPerimeter(grid, r, c - 1, visited);
}
```

**Коли застосовувати:** будь-яка задача, де потрібно порахувати не сам
розмір зв'язної області, а довжину її межі — периметр острова,
кількість "відкритих" сторін клітинок, підрахунок огорожі навколо
регіону.

**Складність:** O(M×N) час (кожна клітинка відвідується один раз
завдяки `visited`), O(M×N) пам'яті на стек рекурсії/visited-масив.
