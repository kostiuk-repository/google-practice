# Medium_05_SurroundedRegions — Довідка

## Техніка: Boundary DFS / Capture & Anti-Capture

**Суть:** захопити (замінити 'O' на 'X') всі оточені області на дошці. Область не вважається оточеною, якщо вона з'єднана з кордоном. Замість обходу всіх внутрішніх клітинок ми запускаємо DFS з кожної 'O' на кордоні та перефарбовуємо її та її зв'язних сусідів у тимчасовий символ 'S' (Safe). Після обходу кордонів ми переглядаємо всю сітку: усі решта 'O' замінюємо на 'X', а всі 'S' повертаємо в 'O'.

```java
// Крок 1: Заливка безпечних клітинок з кордонів
for (int r = 0; r < rows; r++) {
    dfs(board, r, 0);
    dfs(board, r, cols - 1);
}
// Крок 2: Фінальний перегляд
for (int r = 0; r < rows; r++) {
    for (int c = 0; c < cols; c++) {
        if (board[r][c] == 'O') board[r][c] = 'X';
        else if (board[r][c] == 'S') board[r][c] = 'O';
    }
}
```

**Коли застосовувати:** задачі на "захоплення" або ізоляцію внутрішніх областей сітки, що не дотичні до меж.

**Складність:** O(M×N) час (два проходи по сітці), O(M×N) пам'ять для стеку рекурсії у гіршому випадку.

## Drills

1. **Medium_05_01** — перевірка, чи є клітинка з символом 'O' граничною.
2. **Medium_05_02** — запуск DFS від граничної клітинки та позначення безпечних областей символом 'S'.
3. **Medium_05_03** — фінальний прохід по сітці: заміна залишків 'O' на 'X' та відновлення 'S' до 'O'.
4. **Medium_05_04** — одновимірний варіант задачі (масив символів, оточений стінами 'X').
5. **Medium_05_05** — капстоун: тривимірний варіант Surrounded Regions на кубічній сітці X x Y x Z.

Розв'язуй у порядку 01 → 05, не підглядаючи в Medium_05_SurroundedRegions.java.
