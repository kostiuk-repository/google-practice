# Medium_05_CountPrimes — Довідка

## Техніка: Оптимізоване решето Ератосфена

**Суть:** Оптимізація решета Ератосфена шляхом обробки тільки непарних чисел (що зменшує пам'ять вдвічі) та завершення зовнішнього циклу при `i * i >= N`.

```java
if (n <= 2) return 0;
boolean[] isNotPrime = new boolean[n];
int count = 1; // 2 is prime
for (int i = 3; i < n; i += 2) {
    if (!isNotPrime[i]) {
        count++;
        if ((long) i * i < n) {
            for (int j = i * i; j < n; j += 2 * i) {
                isNotPrime[j] = true;
            }
        }
    }
}
return count;
```

**Коли застосовувати:** Підрахунок та генерація простих чисел для великих N (до 10^7 - 10^8).

**Складність:** O(N log log N) час, O(N/2) додаткова пам'ять.

## Drills

1. **Medium_05_01** — генерація списку простих чисел до N.
2. **Medium_05_02** — сегментоване решето для підрахунку простих чисел у діапазоні [L, R].
3. **Medium_05_03** — розклад парного числа на суму двох простих (гіпотеза Гольдбаха).
4. **Medium_05_04** — оптимізація пам'яті решета за допомогою java.util.BitSet.
5. **Medium_05_05** — знаходження суми простих дільників для кожного числа в діапазоні від 1 до N.
