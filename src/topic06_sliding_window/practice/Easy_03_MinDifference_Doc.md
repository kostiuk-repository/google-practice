# Easy_03_MinDifference — Довідка

## Техніка: Sort + Fixed-Size Window

**Суть:** якщо задачу можна звести до порівняння "найбільший мінус
найменший" в наборі елементів, сортування перетворює довільний вибір
підмножини на вибір ПОСЛІДОВНОГО вікна у відсортованому масиві —
тоді найкраще вікно розміру `k` завжди складається з `k` сусідніх
(після сортування) елементів, і його можна знайти одним проходом
вікна фіксованого розміру.

```java
Arrays.sort(nums);
int best = Integer.MAX_VALUE;
for (int i = 0; i + k - 1 < nums.length; i++) {
    best = Math.min(best, nums[i + k - 1] - nums[i]);
}
```

**Коли застосовувати:** потрібно мінімізувати розкид (max-min) серед
підмножини фіксованого розміру k, обраної з масиву — сортування завжди
є першим кроком, бо воно гарантує, що оптимальна підмножина складається
із сусідніх елементів.

**Складність:** O(N log N) на сортування + O(N) на прохід вікна;
O(1) додаткової пам'яті (не рахуючи сортування).
