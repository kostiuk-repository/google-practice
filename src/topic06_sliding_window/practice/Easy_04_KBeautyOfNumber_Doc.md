# Easy_04_KBeautyOfNumber — Довідка

## Техніка: Fixed-Size Window Over a Digit String

**Суть:** число конвертується у рядок цифр, і по ньому ковзає вікно
фіксованої довжини `k`. На кожному кроці вирізаємо підрядок довжини
`k`, парсимо його в число і перевіряємо якусь умову (тут — чи ділить
воно вихідне число без остачі, і чи не дорівнює нулю).

```java
String s = String.valueOf(num);
int count = 0;
for (int i = 0; i + k <= s.length(); i++) {
    int val = Integer.parseInt(s.substring(i, i + k));
    if (val != 0 && num % val == 0) count++;
}
```

**Коли застосовувати:** задача про "підрядки/підпослідовності цифр
фіксованої довжини" числа або рядка, де для кожного вікна потрібно
обчислити якусь властивість і зібрати статистику (кількість, суму,
максимум) по всіх вікнах.

**Складність:** O(D * k) де D — кількість цифр числа (через
`substring` + `parseInt` на кожному кроці); O(1) додаткової пам'яті
поза виходом.

## Drills

1. **Easy_04_01** — вирізати всі вікна довжини k без жодної перевірки (сирі значення).
2. **Easy_04_02** — змінена умова: рахувати вікна-палindromи замість дільників.
3. **Easy_04_03** — змінена агрегація: сума значень-дільників замість їх кількості.
4. **Easy_04_04** — трекінг максимуму: найбільший дільник-підрядок (argmax).
5. **Easy_04_05** — капстоун: пошук найменшого k, при якому k-beauty досягає порогу.

Розв'язуй у порядку 01 → 05, не підглядаючи в Easy_04_KBeautyOfNumber.java.
