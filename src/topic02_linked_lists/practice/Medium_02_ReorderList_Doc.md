# Medium_02_ReorderList — Довідка

## Техніка: Пошук середини, Реверс та Інтерлівінг (Split, Reverse, Merge)

**Суть:** розв'язання складних задач реорганізації списків шляхом розбиття їх на підзадачі:
1. Знаходження середини списку за допомогою швидкого/повільного вказівників (`fast`/`slow`).
2. Розділення списку на дві незалежні половини.
3. Реверс другої половини списку.
4. Почергове злиття (інтерлівінг) першої половини та реверсованої другої половини.

```java
// 1. Пошук середини
ListNode slow = head, fast = head;
while (fast != null && fast.next != null) {
    slow = slow.next;
    fast = fast.next.next;
}

// 2. Розділення
ListNode second = slow.next;
slow.next = null;

// 3. Реверс
ListNode prev = null;
ListNode curr = second;
while (curr != null) {
    ListNode tmp = curr.next;
    curr.next = prev;
    prev = curr;
    curr = tmp;
}
second = prev;

// 4. Інтерлівінг
ListNode first = head;
while (second != null) {
    ListNode tmp1 = first.next;
    ListNode tmp2 = second.next;
    
    first.next = second;
    second.next = tmp1;
    
    first = tmp1;
    second = tmp2;
}
```

**Коли застосовувати:** симетричне перевпорядкування списку з обох кінців, перевірка паліндромів, специфічне групування елементів за один прохід по пам'яті без створення копій вузлів.

**Складність:** O(N) час, O(1) додаткової пам'яті.

## Drills

1. **Medium_02_01** — розділення списку навпіл по середині та повернення голови другої половини.
2. **Medium_02_02** — ітеративний реверс зв'язаного списку in-place.
3. **Medium_02_03** — почергове злиття (інтерлівінг) двох окремих списків.
4. **Medium_02_04** — реверс лише другої половини списку зі збереженням першої.
5. **Medium_02_05** — капстоун: реверс першої половини та її інтерлівінг із другою половиною списку.

Розв'язуй у порядку 01 → 05, не підглядаючи в Medium_02_ReorderList.java.
