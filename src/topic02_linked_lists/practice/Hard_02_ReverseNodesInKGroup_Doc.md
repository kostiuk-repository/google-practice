# Hard_02_ReverseNodesInKGroup — Довідка

## Техніка: Локальний реверс блоками фіксованої довжини (Segmented Linked List Reversal)

**Суть:** розбиття списку на блоки довжиною $K$. Для кожного блоку перевіряється наявність достатньої кількості елементів. Якщо елементів достатньо, здійснюється реверс цього блоку in-place, а його межі правильно підключаються до попередніх та наступних частин списку. Якщо елементів менше ніж $K$, блок залишається без змін (за стандартною умовою LeetCode).

```java
ListNode dummy = new ListNode(0, head);
ListNode groupPrev = dummy;

while (true) {
    ListNode kth = getKth(groupPrev, k);
    if (kth == null) break;
    
    ListNode groupNext = kth.next;
    
    // Реверс поточної групи
    ListNode prev = kth.next;
    ListNode curr = groupPrev.next;
    while (curr != groupNext) {
        ListNode tmp = curr.next;
        curr.next = prev;
        prev = curr;
        curr = tmp;
    }
    
    // Перепідключення голови та хвоста групи
    ListNode tmp = groupPrev.next;
    groupPrev.next = kth;
    groupPrev = tmp;
}
return dummy.next;
```

**Коли застосовувати:** реверсування зв'язаних списків порціями, групування/перестановка елементів через рівні проміжки, реалізація складних перекомпонувань даних з обмеженням на додаткову пам'ять.

**Складність:** O(N) час, O(1) додаткової пам'яті.

## Drills

1. **Hard_02_01** — перевірка наявності принаймні $K$ вузлів у поточному списку.
2. **Hard_02_02** — ізольований реверс перших $K$ елементів списку.
3. **Hard_02_03** — реверс списку блоками розміру 2 (використання шаблону K-group).
4. **Hard_02_04** — реверс блоками розміру $K$ з обов'язковим реверсом останнього неповного блоку.
5. **Hard_02_05** — капстоун: реверс лише непарних за порядком $K$-груп (1-ша, 3-тя тощо), парні залишаються без змін.

Розв'язуй у порядку 01 → 05, не підглядаючи в Hard_02_ReverseNodesInKGroup.java.
