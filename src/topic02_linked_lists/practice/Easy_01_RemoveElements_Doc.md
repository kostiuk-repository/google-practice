# Easy_01_RemoveElements — Довідка

## Техніка: Dummy Sentinel + Prev/Curr Traversal

**Суть:** щоб видаляти вузли з однозв'язного списку за умовою (включно з
головою), створюємо фіктивний вузол `dummy`, який вказує на `head`.
Потім рухаємось двома вказівниками: `prev` (останній вузол, що
залишається) та `curr` (вузол, що перевіряється). Якщо `curr`
задовольняє умову видалення — `prev.next = curr.next`; інакше —
`prev = curr`. У обох випадках `curr = curr.next`.

```java
ListNode dummy = new ListNode(0);
dummy.next = head;
ListNode prev = dummy, curr = head;
while (curr != null) {
    if (shouldRemove(curr)) {
        prev.next = curr.next;
    } else {
        prev = curr;
    }
    curr = curr.next;
}
return dummy.next;
```

**Коли застосовувати:** будь-яке завдання "видали вузли за умовою" — за
значенням, за позицією, за діапазоном, за динамічною умовою (порівняння
із попереднім вузлом). Ключова ознака: голова списку сама може
підпадати під видалення, тож без `dummy` довелось би окремо обробляти
цей edge case.

**Складність:** O(N) час, один прохід; O(1) додаткової пам'яті (окрім
самого `dummy`-вузла).

## Drills

1. **Easy_01_01** — видалення вузла за 0-indexed позицією замість значення.
2. **Easy_01_02** — видалення лише ПЕРШОГО входження значення (ранній вихід).
3. **Easy_01_03** — узагальнення умови до діапазону [low, high].
4. **Easy_01_04** — видалення всіх входжень + підрахунок видалених через out-параметр.
5. **Easy_01_05** — динамічна умова: видалення послідовних дублікатів (порівняння з попереднім вузлом).

Розв'язуй у порядку 01 → 05, не підглядаючи в Easy_01_RemoveElements.java.
