# Easy_09_BinaryToInteger — Довідка

## Техніка: Left-To-Right Streaming Accumulate

**Суть:** коли найстарший розряд знаходиться на голові списку, число
можна побудувати за один прохід зліва направо без обчислення степенів
двійки заздалегідь: на кожному кроці зсуваємо накопичене значення на
один розряд і додаємо новий біт.

```java
int result = 0;
ListNode curr = head;
while (curr != null) {
    result = result * 2 + curr.val; // еквівалент (result << 1) | curr.val
    curr = curr.next;
}
return result;
```

**Коли застосовувати:** побудова числа з послідовності розрядів (будь-
якої основи числення), потокова перевірка подільності (модульна
арифметика без побудови повного числа), синхронна обробка двох
паралельних послідовностей розрядів (побітові операції). Ключова
ознака: розряди йдуть у порядку "від старшого до молодшого", тож
накопичення working зліва направо без потреби у зворотному проході чи
обчисленні довжини заздалегідь.

**Складність:** O(N) час, O(1) додаткової пам'яті (окрім акумулятора).

## Drills

1. **Easy_09_01** — узагальнення основи числення: результат*10 + digit замість результат*2 + bit.
2. **Easy_09_02** — дзеркальна конвенція: молодший біт на голові списку (потребує степенів двійки замість простого зсуву).
3. **Easy_09_03** — потокова перевірка подільності на 3 через модульний акумулятор замість побудови значення.
4. **Easy_09_04** — синхронна обробка ДВОХ списків одночасно для обчислення XOR.
5. **Easy_09_05** — капстоун: акумулятор з редукцією за модулем на кожному кроці (захист від переповнення).

Розв'язуй у порядку 01 → 05, не підглядаючи в Easy_09_BinaryToInteger.java.
