# Easy_06_RemoveDuplicates — Довідка

## Техніка: Single-Pointer Adjacent Comparison (Sorted List)

**Суть:** оскільки список відсортований, усі однакові значення стоять
ПОРЯД. Достатньо одного вказівника `curr`: порівнюємо `curr.val` з
`curr.next.val`. Якщо рівні — "перестрибуємо" дублікат
(`curr.next = curr.next.next`), інакше просуваємо `curr` далі.

```java
ListNode curr = head;
while (curr != null && curr.next != null) {
    if (curr.val == curr.next.val) {
        curr.next = curr.next.next;
    } else {
        curr = curr.next;
    }
}
return head;
```

**Коли застосовувати:** видалення сусідніх дублікатів у ВІДСОРТОВАНОМУ
списку (де дублікати гарантовано згруповані). Якщо потрібно видалити
ВЕСЬ дубльований блок (а не залишити один екземпляр), або якщо голова
списку сама може бути видалена — знадобиться `dummy` + `prev`, як у
класичній задачі "Remove Duplicates II".

**Складність:** O(N) час, один прохід; O(1) додаткової пам'яті.

## Drills

1. **Easy_06_01** — дедуплікація + підрахунок кількості унікальних значень.
2. **Easy_06_02** — видалення ВСЬОГО блоку дублікатів (а не залишення одного екземпляра); потребує `dummy` + `prev`.
3. **Easy_06_03** — той самий скан, але на НЕвідсортованому списку (лише сусідні збіги).
4. **Easy_06_04** — перевірка наявності дублікатів без мутації списку.
5. **Easy_06_05** — капстоун: заміна кожного блоку дублікатів на вузол із КІЛЬКІСТЮ входжень.
