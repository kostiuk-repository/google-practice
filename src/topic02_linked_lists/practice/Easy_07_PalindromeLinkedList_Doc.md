# Easy_07_PalindromeLinkedList — Довідка

## Техніка: Find Middle -> Reverse Second Half -> Compare

**Суть:** три послідовні кроки, кожен із яких — окрема відома техніка:
1. Знайти середину списку через slow/fast вказівники (як у
   Easy_04_MiddleOfLinkedList).
2. Розвернути другу половину списку на місці (класичний in-place
   reverse).
3. Порівняти значення першої половини з розвернутою другою,
   вказівник-за-вказівником.

```java
ListNode slow = head, fast = head;
while (fast != null && fast.next != null) {
    slow = slow.next; fast = fast.next.next;
}
ListNode secondHalf = reverse(slow);
ListNode p1 = head, p2 = secondHalf;
boolean isPalindrome = true;
while (p2 != null) {
    if (p1.val != p2.val) { isPalindrome = false; break; }
    p1 = p1.next; p2 = p2.next;
}
```

**Коли застосовувати:** будь-яка задача "перевір симетричність
однозв'язного списку" за умови O(1) пам'яті — без копіювання значень у
масив чи стек. Ключова ознака: непарна довжина вимагає окремо
обробити/пропустити істинний середній елемент.

**Складність:** O(N) час (пошук середини — O(N/2), розворот — O(N/2),
порівняння — O(N/2)); O(1) додаткової пам'яті.

## Drills

1. **Easy_07_01** — ізольований in-place розворот списку (крок 2 сам по собі).
2. **Easy_07_02** — порівняння двох ВЖЕ розділених половин (крок 3 сам по собі).
3. **Easy_07_03** — повна перевірка для непарної довжини з коректним пропуском середини.
4. **Easy_07_04** — видалення середнього вузла перед перевіркою (комбінація з видаленням).
5. **Easy_07_05** — капстоун: перевірка + відновлення оригінального порядку списку після виконання.
