# Easy_10_MergeNodes — Довідка

## Техніка: Злиття та накопичення між роздільниками (Zero-Segment Traversal & Accumulation)

**Суть:** прохід по списку з накопиченням проміжних значень (наприклад, суми, добутку або максимуму) між вузлами-роздільниками (зазвичай нулями) та записом результату безпосередньо в існуючі вузли (in-place) або створення нового компактного списку.

```java
ListNode dummy = new ListNode(0);
ListNode tail = dummy;
ListNode curr = head.next;
int sum = 0;

while (curr != null) {
    if (curr.val == 0) {
        tail.next = new ListNode(sum);
        tail = tail.next;
        sum = 0;
    } else {
        sum += curr.val;
    }
    curr = curr.next;
}
return dummy.next;
```

**Коли застосовувати:** групування або агрегування послідовностей вузлів, розділених маркерами (нулі, спеціальні значення, зміна знаку). Дозволяє ефективно обробляти потокові дані та виконувати злиття за один прохід.

**Складність:** O(N) час, O(1) додаткової пам'яті (якщо виконується in-place модифікація) або O(M) додаткової пам'яті (де M — кількість сегментів, якщо створюється новий список).

## Drills

1. **Easy_10_01** — просте обчислення суми всіх елементів списку без урахування роздільників.
2. **Easy_10_02** — фільтрація вузлів: видалення всіх нульових вузлів зі списку зі збереженням решти елементів.
3. **Easy_10_03** — злиття фіксованих пар елементів, розділених нулями.
4. **Easy_10_04** — обчислення префіксної суми (running sum) для елементів у межах кожного сегмента між нулями.
5. **Easy_10_05** — капстоун: злиття сегментів шляхом пошуку максимального елемента замість суми в кожному сегменті.

Розв'язуй у порядку 01 → 05, не підглядаючи в Easy_10_MergeNodes.java.
