# Easy_08_IntersectionOfTwoLists — Довідка

## Техніка: Switch-Heads Two-Pointer Equalization

**Суть:** два вказівники `pA` і `pB` йдуть кожен своїм списком. Коли
`pA` доходить до `null`, він перестрибує на `headB`; коли `pB` доходить
до `null`, він перестрибує на `headA`. Обидва вказівники за N+M кроків
проходять СУМАРНО однакову відстань (`lenA + lenB`), тож вони або
зустрінуться рівно у вузлі перетину, або обидва одночасно стануть
`null` (якщо перетину немає).

```java
ListNode pA = headA, pB = headB;
while (pA != pB) {
    pA = (pA == null) ? headB : pA.next;
    pB = (pB == null) ? headA : pB.next;
}
return pA; // вузол перетину або null
```

**Коли застосовувати:** пошук спільного вузла (хвоста) двох списків без
додаткової пам'яті. Еквівалентний за ідеєю підхід — вирівняти довжини
наперед (порахувати обидві довжини, просунути довший список на
різницю), що дає той самий результат за трохи інший "маршрут".

**Складність:** O(N + M) час, O(1) додаткової пам'яті.
