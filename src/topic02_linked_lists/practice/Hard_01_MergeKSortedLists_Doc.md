# Hard_01_MergeKSortedLists — Довідка

## Техніка: Багатостороннє злиття за допомогою купи / Черги з пріоритетом (Min-Heap / K-Way Merge)

**Суть:** злиття $K$ відсортованих списків у один відсортований список. Замість лінійного порівняння голів усіх списків на кожному кроці, ми використовуємо мінімальну купу (Min-Heap), де зберігаються поточні голови списків. Найменший елемент вилучається за $O(\log K)$, а на його місце додається наступний елемент з того самого списку.

```java
if (lists == null || lists.length == 0) return null;

PriorityQueue<ListNode> pq = new PriorityQueue<>((a, b) -> Integer.compare(a.val, b.val));

for (ListNode list : lists) {
    if (list != null) {
        pq.add(list);
    }
}

ListNode dummy = new ListNode(0);
ListNode curr = dummy;

while (!pq.isEmpty()) {
    ListNode node = pq.poll();
    curr.next = node;
    curr = curr.next;
    
    if (node.next != null) {
        pq.add(node.next);
    }
}
return dummy.next;
```

**Коли застосовувати:** об'єднання багатьох відсортованих джерел даних (файли логів, потоки транзакцій), злиття зовнішніх файлів (External Merge Sort), робота з великою кількістю впорядкованих послідовностей.

**Складність:** $O(N \log K)$ час, де $N$ — загальна кількість вузлів, $K$ — кількість списків. $O(K)$ додаткової пам'яті для збереження елементів у черзі з пріоритетом.

## Drills

1. **Hard_01_01** — злиття трьох відсортованих списків через порівняння.
2. **Hard_01_02** — злиття K списків, кожен з яких містить щонайбільше один вузол.
3. **Hard_01_03** — вилучення та повернення мінімального елемента з масиву голів K списків.
4. **Hard_01_04** — злиття K відсортованих списків за допомогою стратегії "Розділяй і володарюй" без купи.
5. **Hard_01_05** — капстоун: злиття K списків, представлених як списки динамічних масивів (`List<List<Integer>>`).

Розв'язуй у порядку 01 → 05, не підглядаючи в Hard_01_MergeKSortedLists.java.
